## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes  
**Focus:** Frameworks enabling multi-agent systems, tool use, and persistent execution.

### 1. **CrewAI** (Trending)
*   **Status:** Actively maintained, version ~0.28.x.
*   **Hot:** Increased focus on **"crew" orchestration** with hierarchical task delegation (Manager -> Agents -> Tasks). Newer releases emphasize **inter-crew communication** and sharing context/outputs between specialized crews.
*   **New:** Growing ecosystem of **"tools"** contributed by the community (e.g., specialized web search, data extraction, niche API wrappers). Integration with **OpenAI-compatible local endpoints** (LM Studio, Ollama) is a common user modification.
*   **Pattern:** Users often combine CrewAI with a **vector database** (Chroma, LanceDB) for crew-level memory, moving beyond simple conversation history.

### 2. **LangGraph** / **LangChain**
*   **Status:** LangGraph is the strategic focus for stateful, cyclic multi-agent workflows. LangChain remains for simpler chains.
*   **Hot:** **"Checkpointing"** and **persistence** are key themes. The ability to save, interrupt, and resume complex agent graphs is a major differentiator for long-running tasks.
*   **New:** Increased documentation and examples for **"human-in-the-loop"** nodes within graphs, enabling approval steps or clarifications. There's also a push towards better **debugging and visualization** of graph execution flows.
*   **Pattern:** Used for building **autonomous research agents** that plan, execute web/search tasks, synthesize, and loop until a condition is met.

### 3. **AutoGen** (Microsoft)
*   **Status:** Steady development, with a strong research/academic user base.
*   **Hot:** **Code execution security** and **sandboxing** remain critical discussion points. The framework's strength is in **conversational patterns** between highly capable agents (e.g., a Planner, a Coder, a Critic).
*   **New:** Work on **"group chat"** management with more sophisticated moderation and speaker selection policies. Exploration of **cost optimization** strategies across multiple LLM calls.
*   **Pattern:** Frequently employed for **complex problem-solving** where iterative debate and refinement between agents yields better results than a single agent.

### 4. **Emerging / Niche**
*   **Smol Agents:** Gaining attention for a **minimalist, programmatic approach**. Appeals to developers who want fine-grained control without high-level abstractions.
*   **Model Context Protocol (MCP):** Not a framework itself, but a **standard** gaining traction. It allows agents/tools to expose a uniform interface for context, enabling better interoperability between different agent systems. Frameworks are beginning to add MCP server/client support.
*   **Local-First Runtimes:** Growth in projects like **`cudaclaw`** (GPU-resident agents) and **`flux-runtime`** (self-assembling VM) reflects a trend towards **efficiency and specialized hardware** use, moving away from pure API-based chat loops.

### **Specific, Factual Observation for Fleet Context:**
A discernible split is forming in the open-source agent ecosystem. One path focuses on **high-level orchestration** (CrewAI, LangGraph) for business logic and workflow management. A parallel path focuses on **low-level runtime efficiency** and hardware specialization (e.g., `cudaclaw`'s GPU residency, `flux-runtime`'s bytecode). The fleet's architecture, with `plato-torch` for training rooms and `cudaclaw` as a runtime, appears to straddle this split—aiming for orchestrated training (`plato-torch`) while investing in efficient execution (`cudaclaw`). The emerging **Model Context Protocol (MCP)** could be a relevant integration point for fleet tools to become accessible to other frameworks.

**Recommendation (P1 - Safe Channel):** Monitor the adoption curve of **MCP**. If it becomes a *de facto* standard for tool exposure, ensuring fleet components (like `holodeck-rust` NPCs or `constraint-theory-core` engines) can serve an MCP endpoint could increase their utility and testability by external agents.
