## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes  
**Focus:** Frameworks for building, orchestrating, and deploying autonomous AI agents.

### 1. **CrewAI** (crewai/crewai)
- **Status:** Actively maintained, ~15k stars.
- **Recent Trend:** Increased focus on **inter‑agent communication** and **shared context memory**. The `Crew` abstraction (Manager, Agents, Tasks) remains core.
- **New:** Experimental support for **dynamic task graphs** (tasks created at runtime based on previous outputs) and tighter integration with **local LLMs** via Ollama.
- **Observation:** The framework is being used more for **multi‑stage research and content‑generation pipelines** than for real‑time reactive systems.

### 2. **AutoGen** (microsoft/autogen)
- **Status:** Steady development, ~26k stars.
- **Recent Trend:** Emphasis on **code‑based, customizable agent workflows** over high‑level automation. The `GroupChat` and `AssistantAgent`/`UserProxyAgent` patterns are well‑established.
- **New:** Improved **tool‑call reliability** and extended support for **vision‑capable models** (GPT‑4V, LLaVA). Some community efforts to port AutoGen agents to **edge devices** (Jetson, Raspberry Pi).
- **Observation:** Used heavily in academic and enterprise prototyping, but the complexity of orchestration sometimes leads to brittle, long‑running sessions.

### 3. **LangGraph** (langchain‑ai/langgraph)
- **Status:** Rapid adoption, ~13k stars.
- **Recent Trend:** Becoming the **de‑facto standard for building stateful, cyclic agent workflows**. The graph‑based paradigm (nodes, edges, state) is being adopted by other frameworks.
- **New:** **Checkpointing** and **human‑in‑the‑loop** integrations are now stable. Increased examples of **hierarchical graphs** (sub‑graphs for sub‑tasks).
- **Observation:** The learning curve is steeper than CrewAI or AutoGen, but it offers finer control over agent state and flow—making it suitable for **long‑running, persistent agent systems**.

### 4. **Emerging / Niche Frameworks**
- **SmolAgents** (smol‑ai/agents): Minimalist, focused on **single‑purpose, small‑footprint agents**. Gaining traction for **edge deployment**.
- **Aider** (paul‑gauthier/aider): AI‑pair‑programmer that edits code in a local git repo. Not a multi‑agent framework, but cited in agent‑tooling discussions.
- **Open‑Interpreter** (open‑interpreter/open‑interpreter): Allows LLMs to run code locally. Often used as a **tool‑execution layer** within other agent frameworks.

### 5. **Patterns & Gaps (April 2026)**
- **Pattern:** Movement toward **hybrid orchestration**—using LangGraph for core state‑machine logic, but wrapping CrewAI‑style crews or AutoGen agents as nodes.
- **Pattern:** Increased attention to **agent‑to‑agent validation** (e.g., having a second agent review the first’s output) to reduce hallucinations.
- **Gap:** Most frameworks assume **cloud‑based LLM APIs**. Lightweight, self‑hosted runtime for **GPU‑resident agents** (like `cudaclaw` in our fleet) is still a niche.
- **Gap:** Few frameworks natively support **inter‑agent sentiment or personality persistence** (like `holodeck‑rust` NPCs).

### 6. **Fleet Relevance**
- Our `plato‑torch` training rooms could be seen as a **specialized agent‑orchestration environment** focused on **training‑data generation**.
- The `deadband protocol` (P0/P1/P2) is a **constraint‑based planning pattern** not yet widely mirrored in open‑source frameworks.
- The `flux‑runtime` (self‑assembling bytecode VM) might offer a lower‑level, more portable agent runtime than current Python‑based frameworks.

**Specific Factual Observation:**  
In April 2026, the open‑source agent‑framework landscape is consolidating around **state‑ful, graph‑based orchestration** (LangGraph) and **high‑level crew/task abstractions** (CrewAI). There is measurable interest in **edge‑deployable, small‑footprint agents**, but most production examples still rely on cloud LLM APIs. Our fleet’s architecture—particularly the GPU‑resident runtime (`cudaclaw`) and sentiment‑aware simulation (`holodeck‑rust`)—occupies a distinct niche not yet fully addressed by mainstream frameworks.
