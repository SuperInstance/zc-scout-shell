## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes  
**Focus:** Frameworks for building, orchestrating, and deploying autonomous AI agents.

### 1. **CrewAI** (crewai.com)
- **Status:** Actively maintained, version ~0.28.x.
- **Trend:** Gaining traction for its role-based agent orchestration and built-in tools for research, web search, and code execution.
- **New:** Recent emphasis on "crew" hierarchies (manager/worker agents) and integration with local LLMs via Ollama. Community templates for sales, marketing, and coding crews are proliferating.
- **Note:** Seen as a more structured alternative to AutoGen, with clearer control flow.

### 2. **LangGraph** (langchain-ai.github.io/langgraph)
- **Status:** Core part of LangChain ecosystem, version ~0.0.x.
- **Trend:** Rising popularity for building stateful, multi-agent workflows with cycles and human-in-the-loop nodes.
- **New:** Increased use for long-running, persistent agent systems (chatbots, support automation). The "StateGraph" abstraction allows explicit control of agent state, which some developers prefer over pure message-passing.
- **Note:** Often compared to AutoGen for complex, cyclic workflows; LangGraph provides more low-level control.

### 3. **AutoGen** (microsoft.github.io/autogen)
- **Status:** Steady development from Microsoft Research, version ~0.2.x.
- **Trend:** Still a reference for multi-agent conversations and group chat orchestration, but some community feedback points to steep learning curve for customizations.
- **New:** Recent work on "AgentChat" and improved tool-calling with OpenAI compatible endpoints. Research papers continue to emerge from the team on agentic AI.
- **Note:** Strong in academic and research contexts; sometimes considered heavy for production deployment.

### 4. **Swarm** (github.com/open-swarm)
- **Status:** Emerging framework, version early (0.1.x).
- **Trend:** Notable for its focus on decentralized, swarm intelligence patterns—agents that collaborate without central orchestration.
- **New:** Early adopters are experimenting with emergent coordination for problem-solving (e.g., distributed research, code generation). Lightweight, Rust-based core.
- **Note:** Niche interest from developers exploring beyond centralized manager-agent patterns.

### 5. **GPT-Researcher** (github.com/assafelovic/gpt-researcher)
- **Status:** Specialized framework for autonomous research agents, version ~0.3.x.
- **Trend:** Sustained interest for automated report generation, market/academic research.
- **New:** Recent improvements in source reliability scoring and multi-query planning. Community uses it for daily briefings and competitive analysis.
- **Note:** Not a general agent framework, but a dedicated tool in the agent ecosystem.

### 6. **Local-First & Privacy Focus**
- **Observation:** Growing subset of developers are adapting the above frameworks to run fully locally (Ollama, Llama.cpp, private clouds). This is not a single framework, but a pattern: using CrewAI or LangGraph with local LLMs to avoid API costs and data leakage.

### 7. **Fleet Context Comparison**
- **Plato-torch / Plato-ensign:** The fleet's approach differs by focusing on **training rooms** that accumulate interaction tiles and export specialized LoRA adapters (ensigns). This is less about runtime orchestration and more about **continuous, distributed training**.
- **Cudaclaw / Flux-runtime:** The fleet's GPU-resident runtime and self-assembling VM are lower-level infrastructure pieces, not directly comparable to the agent frameworks above. They could potentially host agents built with CrewAI or LangGraph.

### Specific Factual Observation (P0/P1):
- **P0 (Negative Space):** Few open-source frameworks natively support **persistent, trainable agent states** that improve over time via fine-tuning. Most frameworks treat agents as static LLM wrappers with tools.
- **P1 (Safe Channel):** The fleet's **plato-ensign** concept (exporting room experience as LoRAs) represents a unique channel: agents that evolve through accumulated tile training, not just prompt engineering or tool addition. This is an under-explored area in mainstream open-source agent frameworks as of April 2026.

### Recommendation for Fleet:
Monitor **LangGraph** and **Swarm** for inspiration on stateful workflows and decentralized patterns, but double down on the fleet's differentiator: **the training room as a continuous learning system**. This could be a unique contribution if documented and open-sourced.
