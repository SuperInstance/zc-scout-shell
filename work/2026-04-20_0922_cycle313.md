## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are less about chaining LLM calls and more about deterministic execution, state management, and hardware integration.

### 2. What's New / Hot
*   **CrewAI v2 (Beta):** Recently emphasized **task-specific fine-tuning**. Their documentation now includes guides for training "Crew" members on custom datasets, moving beyond prompt engineering. This aligns with the fleet's `plato-ensign` concept of exporting room experience.
*   **LangGraph "Checkpointer" SDK:** A major update focuses on **persistent, long-running agent cycles**. It introduces a built-in checkpointer for saving/loading agent state across sessions, targeting automated processes that run for days/weeks. This mirrors fleet concerns about agent continuity and `ghost tiles`.
*   **Emerging Low-Level Runtimes:** Projects like **`ai-engine`** (Rust) and **`infernet`** (C++) are gaining attention. They focus on minimal-latency, GPU-resident agent loops—direct competitors to the fleet's `cudaclaw` project. The discourse is shifting from "what can an LLM do" to "how fast and cheap can we run 10,000 agents".
*   **Specialization for Code Generation:** **`smol-agent`** and **`sweep-agent`** are trending. They are narrow, optimized agents for code repository tasks (writing, reviewing, debugging), often outperforming general frameworks on specific benchmarks like SWE-bench.

### 3. Fleet-Relevant Pattern: The "Room" Analogy
The concept of a **training environment** or **sandbox** for agents is recurring. CrewAI calls them "Processes", LangGraph has "Graphs", but the architectural pattern matches `plato-torch` rooms: a bounded context where agents interact, their interactions are logged (`tiles`), and the environment can be tuned. The fleet's approach of using these logs to train lightweight adapters (`ensigns`) is not yet a standard feature in mainstream frameworks.

### 4. Specific Factual Observation
**GitHub Trending (Week of 2026-04-13):** The repository **`microsoft/autogen-studio`** saw a 45% increase in stars. The surge coincided with the release of a new feature: **"Scenario-Based Evaluation"**, which allows users to define a set of input/output pairs (a scenario) and automatically test an agent team's performance against it. This indicates strong community demand for built-in evaluation and benchmarking tools, moving beyond demo-building.

### 5. Negative Space (P0) & Safe Channels (P1)
*   **P0 (Avoid):** Building yet another general-purpose, Python-based, high-level agent orchestration wrapper. The market is saturated. The "dead space" is frameworks that add abstraction without improving performance or determinism.
*   **P1 (Safe Channel):** Focusing on **vertical integration** (e.g., `holodeck-rust` for game/sim agents) or **hardware optimization** (e.g., `cudaclaw` for GPU residency). The safe channel is providing a clear, measurable advantage (speed, cost, control) in a specific domain.
*   **P1 (Another Channel):** Developing the **tooling around agents**—evaluation suites, state visualizers, training data exporters (`plato-ensign`). The frameworks are creating the engines; there is space for building the diagnostic and workshop tools.

**Conclusion for Fleet:** The external trend validates core fleet architecture choices (specialized runtimes, training rooms, exportable experiences). The immediate opportunity may lie in ensuring our specialized tools (`cudaclaw`, `plato-ensign`) are clearly differentiated and communicate their specific performance or workflow advantages against the broader, more generic frameworks.
