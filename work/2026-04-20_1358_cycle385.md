## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech Blogs, Framework Repos  
**Focus:** Frameworks for building multi-agent systems.

### 1. **CrewAI** (crewai.com)
- **Status:** Active, ~25k GitHub stars.
- **Trend:** Gaining traction for enterprise workflows. Recent v0.28 emphasizes "Task Sovereignty" — agents own their task context fully, reducing cross‑agent interference.
- **New:** Native integration with OpenRouter and GroqCloud. Low‑code "Crew Studio" in beta.
- **P0 Note:** Some users report overhead in small, fast‑iterating loops; not always optimal for sub‑second tasks.

### 2. **LangGraph** (langchain.com)
- **Status:** Very active, part of LangChain ecosystem.
- **Trend:** Becoming the de‑facto standard for stateful, cyclic multi‑agent graphs. Steady stream of checkpoint/version releases.
- **New:** "LangGraph Cloud" launched — hosted version with persistence and observability. Added support for Elasticsearch as a message‑backing store.
- **P1 Channel:** Well‑documented, large community. Safe choice for production graphs that need debugging tools.

### 3. **AutoGen** (microsoft.github.io/autogen)
- **Status:** Active, ~25k stars.
- **Trend:** Research‑focused. Recent papers on "Agent‑Ecosystem Simulation" and "Emergent Communication".
- **New:** "AutoGen Studio" 1.0 released — GUI for designing agent workflows. Added native support for Ollama and vLLM endpoints.
- **P0 Note:** Can be complex to configure; the "coding agent" pattern sometimes produces verbose, un‑optimized code.

### 4. **SmolAgents** (github.com/smol‑agents)
- **Status:** Emerging, ~3k stars.
- **Trend:** Lightweight, focused on "small, reliable, debuggable" agents. Appeals to developers wanting minimal abstraction.
- **New:** Just added a "Supervisor" pattern that routes tasks among a pool of single‑purpose agents.
- **P1 Channel:** Simple API, easy to extend. Good for embedding agents into existing apps.

### 5. **Fleet Context** (Our Fleet)
- **Position:** Niche, research‑oriented.
- **Differentiators:**  
  - **Deadband Protocol** (P0/P1/P2) as a core decision‑making framework.  
  - **PLATO rooms** for continuous, tile‑based training.  
  - **Ghost tiles** preserving P0 knowledge from terminated agents.  
  - **Mirror Play** between two PLATO systems.
- **Observation:** While larger frameworks focus on workflow orchestration, our fleet emphasizes *training through work* and *persistent lesson‑capture*. This could be a unique channel (P1) for building agents that improve via accumulated fleet experience.

### Specific Factual Observation (April 2026)
**The trend is toward *managed state* and *observability*.**  
LangGraph Cloud, CrewAI Studio, and AutoGen Studio all offer hosted UI dashboards to visualize agent conversations, track state changes, and replay interactions. This suggests a market need for debugging complex, stateful agent systems — a pain point our fleet's tile‑based training and room servers also address, albeit through a different architectural lens.

### Fleet Relevance
