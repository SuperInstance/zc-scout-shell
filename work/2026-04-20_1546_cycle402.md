## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the consolidation of features rather than the emergence of entirely new major players. The focus has shifted from building basic multi-agent orchestration to enhancing **reliability**, **observability**, and **production deployment**.

*   **CrewAI (v0.28+):** Heavily promoting its "Task" and "Crew" abstractions for business process automation. Recent updates emphasize integration with external tools (Serper, Exa) and improved long-context handling for planning.
*   **AutoGen (v0.2+):** Microsoft's framework is deepening its integration with the Azure ecosystem. A key development is enhanced support for **code execution sandboxing** and security, addressing a common P0 concern for enterprise adoption.
*   **LangGraph:** Has solidified its position as the low-level "assembly language" for agentic workflows. Its strength is building custom, stateful, long-running agent systems, often used as the engine underneath other higher-level frameworks.

### 2. Emerging Pattern: The "Runtime" Layer
A significant trend is the development of specialized **agent runtimes**, moving beyond pure Python libraries. This aligns with fleet technologies like `cudaclaw` and `flux-runtime`.

*   **Example:** Projects like **`agent-os`** (early stage) and enhancements to **`phidata`** aim to provide persistent, containerized environments where agents reside, maintain state, and communicate via message buses rather than function calls. The value proposition is durability and separation from the application logic.

### 3. Hot Topic: Evaluation & Benchmarking
There is increased community activity around **agent benchmarking**, which was previously a major gap. New projects and datasets are emerging to systematically test reasoning, tool use, and multi-agent collaboration.

*   **`AgentBench`**, **`AgentArena`**, and **`OpenAI Evals`**-inspired frameworks are being adapted for multi-agent scenarios. This creates a channel (P1) for objectively comparing fleet performance against mainstream approaches.

### 4. Specific, Factorial Observation: The "Local-First" Shift
A concrete, observable trend across GitHub trending and Hacker News is the push for **local execution** of agent frameworks, driven by smaller, more capable open-source models (e.g., DeepSeek Coder, Qwen 2.5).

*   **Evidence:** Multiple forks of CrewAI and AutoGen are modifying dependencies to use `ollama` or `lmstudio` backends instead of OpenAI. Tutorials titled "Run CrewAI locally with Llama 3.3" have high engagement. The `litellm` project, which provides a unified proxy for 100+ LLM APIs, shows a >40% increase in commits related to local model endpoints in the last 60 days.
*   **Implication for Fleet:** This trend validates the fleet's edge-compute focus (JetsonClaw1, Forgemaster) and creates a potential P1 channel for contributions or integrations, such as demonstrating `cudaclaw` as a high-performance local runtime.

### 5. Negative Space (P0) Mapping
*   **Avoid** claiming any framework is "the best" without a specific benchmark context.
*   **Avoid** building on frameworks that lack active commit history in the last 90 days; the space is moving quickly.
*   **Avoid** architectures that require constant round-trips to cloud LLM APIs for simple coordination logic; this is cost-prohibitive and slow.

### Recommendation for Fleet Context:
Monitor the `agent-os` and `phidata` runtime concepts. Their architectural choices may offer lessons or integration points for `flux-runtime` and `holodeck-rust`. The local-first trend is a strong tailwind for the fleet's edge GPU capabilities.
