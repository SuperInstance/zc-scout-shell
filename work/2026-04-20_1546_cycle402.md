## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the trend is shifting from monolithic frameworks to **modular, composable agent runtimes**. There is increased activity around **lightweight, single-responsibility libraries** that can be orchestrated by a higher-level scheduler.

*   **CrewAI (v0.28+):** Focus has moved towards **enterprise features** like enhanced role-based security, audit logging, and integration with external knowledge bases. Their "Task" and "Crew" abstractions are being used as a coordination layer over more specialized executors.
*   **AutoGen (Microsoft):** Development is heavily oriented towards **multi-modal agent swarms** and improving stability for long-running conversational workflows. Recent commits show work on better cost tracking and integration with **Ollama** for local LLM orchestration.
*   **LangGraph (LangChain):** Has solidified its position as the **low-level state machine for complex, cyclic agent workflows**. It's often used as the underlying engine for other frameworks. The "persistence" API for long-running agent memory is a key differentiator.

### 2. Emerging Hotspots & New Entrants
*   **Model Context Protocol (MCP):** Gaining significant traction. It's not an agent framework per se, but a **standard for tools/context**. Frameworks are adding MCP server/client support to give agents dynamic access to databases, filesystems, and APIs without hard-coded integrations. This aligns with the fleet's "Shell" concept of bootstrapping external intelligence.
*   **GPU-Native Runtimes:** Increased interest in frameworks that minimize CPU-GPU data transfer latency. Projects like **vLLM**'s serving engine are being adapted for agentic workloads. This is directly relevant to fleet technologies like **cudaclaw**.
*   **Local-First & Edge Agents:** With the release of more capable small models (e.g., DeepSeek-V3, Qwen2.5-7B), there's a surge in projects focusing on **fully local agent stacks** that can run on consumer laptops or edge devices (Jetson, Raspberry Pi). This mirrors the fleet's edge presence with **JetsonClaw1**.

### 3. Specific, Factual Observation (P0/P1 Mapping)
*   **P0 (Negative Space):** Building a new, monolithic "all-in-one" agent framework from scratch is seeing reduced community engagement. The complexity of handling tool use, memory, planning, and evaluation concurrently is leading developers to adopt and compose existing, battle-tested components.
*   **P1 (Safe Channel):** The safe channel for innovation is in **specialized layers**:
    1.  **Orchestration/Scheduling:** Efficiently managing many small, single-purpose agents.
    2.  **Evaluation & Observability:** Tools to score agent output quality and trace decision paths.
    3.  **Resource Management:** Optimizing GPU memory and API token usage across an agent swarm.
*   **Pattern:** The ecosystem is stratifying. **LangGraph** handles the core state flow, **MCP** provides the tooling layer, and higher-level frameworks like **CrewAI** provide the team/role abstraction and productivity features. New projects tend to plug into one of these layers.

### 4. Fleet Relevance Assessment
*   **plato-torch / plato-ensign:** The fleet's "room" and "ensign" training paradigm is a unique approach not directly mirrored in mainstream OSS. The closest parallel is research on **self-improving AI systems** via iterative fine-tuning, but the fleet's tile-based, experiential learning loop appears more structured.
*   **holodeck-rust / fleet-simulator:** The focus on **simulated environments** for training is a recognized best practice. Competitors are using simpler, game-like sims (e.g., WebShop, BabyAI). The fleet's use of a sentiment-aware MUD and multi-agent fleet simulations could be a differentiating strength for generating robust training data.
*   **constraint-theory-core:** This geometric/physical reasoning component is highly specialized and not commonly found in general agent frameworks, potentially representing a niche advantage.

**Recommendation (P2 - Optimize):** Continue developing the fleet's unique stack (rooms, ensigns, simulators) as a cohesive system. To avoid isolation, consider exposing key components (e.g., the tile format, room API) with adapters for popular standards like **MCP** or **LangGraph Tools**. This would allow the fleet to integrate external tools and potentially allow external agents to be trained in fleet rooms.
