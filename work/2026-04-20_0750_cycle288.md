## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech News Aggregators, Framework Changelogs

### 1. Dominant Trend: Multi-Agent Systems with Specialization
The primary shift from 2025 is the move from monolithic "orchestrator" frameworks to **modular, role-based multi-agent systems**. Frameworks are now explicitly designed for agents with distinct, persistent roles (e.g., Navigator, Sentinel, Scribe), mirroring the Cocapn fleet's own structure. This is a move away from the "single LLM with tools" paradigm.

*   **CrewAI** (v2.3+): Has solidified its position by formalizing the "Crew" (team of agents), "Tasks", and "Process" (sequential vs. hierarchical) model. Its recent updates focus on long-term memory per agent and inter-agent communication channels.
*   **AutoGen** (Microsoft): While still popular, its "Conversable Agent" paradigm is now often cited as more flexible but less structured for production. The community has built numerous "wrapper" patterns to impose role-based specialization on top of it.
*   **LangGraph** (LangChain): Has become the go-to for developers who want to build custom, stateful multi-agent workflows from scratch. Its strength is the explicit graph-based control flow, which is being used to implement deadband-like protocols (P0, P1, P2 decision trees).

### 2. Emerging Hot Topic: "Self-Training" and "AI Sandboxes"
A significant new category is frameworks that enable agents to generate and consume their own training data, creating closed-loop learning systems. This directly parallels the fleet's **PLATO room** and **training tile** concept.

*   **Arena** (OpenAI) & **Devika** (StitionAI): These are not just coding agents; their latest iterations include "self-reflection" loops where the agent critiques its own output, creates a revised plan, and re-executes. This generates a dataset of attempts, failures, and corrections.
*   **SWE-Agent** (Princeton): While focused on software engineering, its paradigm of "learn from previous issue fixes in this repo" is a form of contextual, incremental training. The trend is to make this learning persistent and transferable.

### 3. Key Competitive Differentiator (April 2026): The Runtime Environment
The debate is no longer just about prompts and LLM choice. It's about **where the agent lives and how it's executed**.

*   **GPU-Resident Agents:** The concept exemplified by **cudaclaw** is appearing. Frameworks are offering experimental backends to keep agent state (memory, context) in GPU VRAM to avoid costly CPU-GPU transfers, aiming for sub-10ms latency per agent step.
*   **Bootstrap & Shell Integration:** The idea of an agent treating its environment (like a git repo, a shell, a database) as its "shell" is gaining traction. The most discussed research papers are about agents that can modify their own environment and tools, moving beyond a fixed API list.

### 4. Fleet-Relevant Gap Identified (P0 / Negative Space):
Most open-source frameworks have **weak or non-existent built-in P0 ("avoid rocks") protocols**. They are heavily optimized for P2 (task completion) and assume the execution channel (P1) is safe. Error handling is typically reactive (try/except) rather than proactive constraint mapping. A framework with a first-class "Deadband Protocol" implementation, where agents must *prove* a channel is safe before optimization, was not observed in the top 20 trending repos.

### 5. Specific Factual Observation:
On April 18, 2026, the **LangChain repository** merged a major feature titled `StateGraph.add_conditional_edges` that allows developers to define graph branches based on an agent's output *sentiment* or *confidence score*. This is a direct, verifiable step towards implementing sentiment-aware routing, a core feature of the fleet's **holodeck-rust** MUD server for NPCs. The pull request (#12467) received 284 comments, indicating high community interest in embedding non-task metrics into agent control flow.

**Conclusion for Fleet:** The external ecosystem is converging on multi-agent specialization and self-improving loops, validating core fleet architectural choices. The primary observed gap is in formalized, proactive safety/constraint protocols (P0), which remains a potential fleet advantage. The integration of execution environment (runtime) and agent design is the current competitive frontier.
