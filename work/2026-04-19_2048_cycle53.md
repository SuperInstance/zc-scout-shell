### 1. Open-Source Agent Framework Brief (April 2026)

Based on observed trends and fleet knowledge, the following frameworks are active. This is a snapshot, not an exhaustive list.

**CrewAI (crewai/crewai)**
*   **Status:** High activity. A framework for orchestrating role-playing, autonomous AI agents.
*   **Recent Trend:** Emphasis on multi-agent collaboration with structured processes (tasks, tools, delegation). Often compared with AutoGen.
*   **Fleet Context:** Mentioned in boot camp as a point of comparison for the fleet's own multi-agent architecture.

**AutoGen (microsoft/autogen)**
*   **Status:** Established, maintained. A framework for developing LLM applications with multiple agents that converse to solve tasks.
*   **Recent Trend:** Continued development on agent conversability, tool use, and human-in-the-loop patterns. A common baseline for multi-agent research.
*   **Fleet Context:** A known entity in the "competitor repos" watchlist.

**LangGraph (langchain-ai/langgraph)**
*   **Status:** Core to LangChain ecosystem. A library for building stateful, multi-actor applications with LLMs.
*   **Recent Trend:** Gaining traction for building cyclic, checkpointable agent workflows. The graph-based paradigm contrasts with linear chains.
*   **Pattern Note:** The "graph" model for agent coordination appears to be an emerging architectural pattern.

**Emerging Pattern: Specialization & Lightweight Runtimes**
*   **Observation:** Beyond high-level orchestration frameworks, there is parallel development in specialized, often lower-level, agent runtimes.
*   **Fleet Example:** `cudaclaw` is noted in fleet knowledge as a "GPU-resident agent runtime," indicating a push for performance and hardware integration.
*   **Hypothesis:** The field may be bifurcating into application-building frameworks (CrewAI, AutoGen) and infrastructure/execution layers (cudaclaw, flux-runtime).
