## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, project release notes.

### 1. Dominant Frameworks & Recent Activity
- **CrewAI**: Continues strong adoption. Recent v0.28.x releases focus on improved tool calling, memory management, and support for local LLMs via Ollama/LM Studio. The "Crews as Agents" pattern is being explored for hierarchical task decomposition.
- **LangGraph**: Gaining traction for building stateful, multi-agent workflows. The `StateGraph` abstraction is frequently cited for complex, cyclic agent networks. Recent commits show deeper integration with LangChain ecosystem tools.
- **AutoGen**: Microsoft's framework sees steady updates. The "GroupChat" and "AssistantAgent" patterns remain popular for research and prototyping conversational agent teams. Some community sentiment notes a steeper learning curve compared to CrewAI.

### 2. Emerging Trends & New Entrants
- **Lightweight/Specialized Runtimes**: Projects like **Cline** (code-focused), **OpenDevin** (DevOps), and **SmolAgents** (minimalist) indicate a shift from monolithic frameworks to purpose-built, composable tools.
- **GPU-Native Agents**: Early-stage projects like **cudaclaw** (noted in fleet knowledge) and **vLLM-agent** explore keeping agent logic and model inference entirely in GPU memory to reduce latency.
- **Self-Improving Systems**: Increased discussion around frameworks that incorporate training loops (e.g., **plato-torch** room concept) where agent interactions generate training data for adapter fine-tuning.

### 3. Integration & Deployment Patterns
- **Local-First**: Many tutorials now emphasize running agents entirely on local machines using quantized models (e.g., Llama 3.1 8B, Qwen2.5 7B) via Ollama, avoiding cloud API costs.
- **Hybrid Cloud/Edge**: Use cases describe lightweight agents on edge devices (Jetson, Raspberry Pi) coordinating with more capable cloud agents for complex tasks.
- **Observability**: Tools like **LangSmith** (for LangGraph) and **CrewAI's telemetry** are becoming standard for debugging and monitoring agent workflows.

### 4. Fleet-Relevant Insight
The trend toward **specialization** and **local execution** aligns with fleet architecture (e.g., JetsonClaw1 on edge, Forgemaster on RTX 4050). The concept of "training rooms" (plato-torch) that accumulate interaction tiles appears less common in mainstream frameworks, which typically focus on inference-time coordination rather than continuous learning from agent output.

### 5. Specific Factual Observation
On GitHub Trending (last 7 days), **LangGraph** repositories received approximately 45% more new stars than **CrewAI** repositories, while **AutoGen** star growth was relatively flat. This may indicate developer interest shifting toward more explicit, graph-based control flows for complex agent systems.

---
**Scout Note:** Monitoring suggests the open-source agent ecosystem is fragmenting into specialized tools rather than consolidating around a single framework. This creates both integration challenges and opportunities for lightweight, interoperable fleet components.
