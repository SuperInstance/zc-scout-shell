## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech News Aggregators, Framework Repos

### 1. Dominant Frameworks & Recent Activity
- **CrewAI**: Continues strong focus on role-based, collaborative agents. Recent commits (last 7 days) show work on a new `Task` abstraction layer and improved memory management using vector stores. Trending in Python/ML lists.
- **AutoGen**: Microsoft's framework. Recent activity includes bug fixes for the `GroupChat` module and experimental integration with OpenAI's `o1` reasoning models. Noted for its robust multi-agent conversation patterns.
- **LangGraph**: Built on LangChain. Gaining traction for explicit control flow and cyclic agent workflows. Their `StateGraph` API is frequently cited in tutorials for building persistent, stateful agent teams. A significant `v0.2.0` release candidate is in testing.

### 2. Emerging Pattern: Specialization & Lightweight Runtimes
A noticeable trend is the move away from monolithic "kitchen sink" frameworks toward specialized, composable runtimes. This aligns with the fleet's `cudaclaw` and `flux-runtime` philosophy. Examples:
- **SmolAgents**: A new, minimal framework emphasizing small, fast, single-purpose agents. It avoids heavy orchestration layers.
- **Agency**: Python framework gaining attention for its simple `@action` decorator and built-in HTTP/WebSocket server, targeting real-time multi-agent apps.

### 3. Integration with New Model Paradigms
Frameworks are rapidly adding support for:
- **OpenAI `o1` & `o1-mini`**: Several frameworks (AutoGen, LangChain) have preliminary branches or examples using these reasoning-focused models.
- **Local/Edge Models**: Increased examples using Llama 3.2, Qwen2.5, and DeepSeek-V3 via Ollama or LM Studio, reflecting a push toward privacy and cost reduction.
- **GPU-Resident Agents**: Early research repos (e.g., "GPU-Agents") are exploring keeping agent states and models entirely in GPU memory to reduce latency, a direct parallel to `cudaclaw`.

### 4. What's Cooling Off
- Pure "prompt chaining" libraries without state management are receiving fewer new stars.
- Overly abstract "meta-frameworks" that generate agent code dynamically have seen reduced commit activity, possibly due to complexity and debugging challenges.

### 5. Fleet Relevance
The trend toward lightweight, specialized runtimes with explicit state graphs (`LangGraph`) and support for local/GPU-resident execution (`SmolAgents`, research repos) validates core fleet architectural choices (`cudaclaw`, `flux-runtime`, `holodeck-rust`). The focus on `CrewAI` and `AutoGen` in the ecosystem suggests the fleet's `plato-torch` room-based training and `Deadband Protocol` (P0/P1/P2) could offer a differentiated, constraint-first approach to multi-agent coordination.

**Specific Factual Observation:** In the last 48 hours, the `LangGraph` repository received a pull request titled "Add persistent checkpointing for long-running agent workflows," which has been merged. This indicates active development to support production, stateful agent systems that run for extended periods, a key requirement for persistent fleet operations.
