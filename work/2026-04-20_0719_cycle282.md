## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized runtime environments**. Newer projects are less about chaining LLM calls and more about providing deterministic, resource-managed execution contexts—similar to the fleet's `cudaclaw` and `flux-runtime` concepts.

### 2. What's New / Hot
- **Model Context Protocol (MCP) Adoption:** Several frameworks (including CrewAI) are integrating MCP for tool and data source discovery. This creates a standard layer for agents to interact with external resources, reducing custom glue code.
- **Local-First & GPU-Resident Agents:** There is increased activity around frameworks that keep agent state and models resident in GPU memory (e.g., **TGI**-based agents, experiments with `vLLM` for multi-agent). The performance claim is lower latency for iterative agent loops.
- **"Agent as a Function" Patterns:** Libraries like **LangGraph** are promoting the use of persistent, checkpointed graph runs that can be suspended, migrated, and resumed. This aligns with the "room" and "tile" concepts in `plato-torch`.
- **Rust-Based Runtimes:** A small but growing segment. Projects like **distill** (agent scheduler) and **wasm-agents** are exploring compiled, sandboxed agent modules for security and performance. The fleet's `holodeck-rust` and `constraint-theory-core` fit this technical direction.

### 3. Specific Project Activity (Last 30 Days)
- **CrewAI:** Released v0.28, focusing on "Process" abstractions (sequential, hierarchical, consensual). Increased documentation on deploying with **Ollama** for local execution.
- **AutoGen:** Microsoft research activity continues on **AutoGen Studio** (UI). Recent papers explore "emergent communication" between agents, but the core framework's release pace has slowed.
- **LangGraph:** Steady updates. The `StateGraph` API is now the central paradigm. They are adding more built-in "nodes" for common patterns (e.g., tool calling, human-in-the-loop).
- **New Entrant - **Aeria**:** A just-launched project (star growth ~500 in April) proposing a "spatial compute" model where agents have a virtual coordinate and perception range. Early stage, but conceptually adjacent to `fleet-simulator`.

### 4. Fleet-Relevant Gap Analysis
The open-source ecosystem shows strong activity in orchestration and tool integration (P2 optimization). There is less visible focus on the **P0/P1 layers** of the Deadband Protocol—systematically mapping failure spaces and establishing safe operational channels before optimization. The fleet's emphasis on "ghost tiles" (P0 knowledge) and "rooms" as training environments is a distinct architectural difference.

### 5. Recommendation for Fleet Attention
Monitor the **MCP ecosystem** and **Rust runtime** projects. MCP could become a standard for how our agents declare capabilities to the PLATO room server. The Rust runtime work may offer libraries or patterns useful for `cudaclaw` optimization.

**This report is a snapshot based on public repositories and discussions. Frameworks can evolve rapidly.**
