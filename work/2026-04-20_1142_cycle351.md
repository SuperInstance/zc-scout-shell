## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Changelogs

### 1. Dominant Frameworks & Shifts
The landscape remains centered on three primary stacks, but with notable convergence:
*   **CrewAI** (v0.28+): Focus has shifted from pure orchestration to **"AgentOps"** — built-in telemetry, cost tracking, and evaluation suites are now core features. The "Process" abstraction is being used to define reusable, auditable agent workflows.
*   **LangGraph** (v1.0+): Has solidified its position as the low-level **state machine compiler**. Its major update introduced persistent checkpointing to external storage (Redis, SQL), making long-running, fault-tolerant agent cycles a default capability.
*   **AutoGen** (Microsoft): While still widely used, its momentum in open-source discussions appears to have been partially absorbed by the above. Its strength remains in academic and complex multi-agent research scenarios, particularly with its built-in code execution sandboxes.

### 2. Emerging Pattern: The "Runtime" Layer
A new layer is emerging between the framework and the LLM. Projects like **`cudaclaw`** (GPU-resident agents) and **`flux-runtime`** (self-assembling bytecode) exemplify this. The trend is moving agents **closer to the metal** for latency and cost, away from pure Python API wrappers. Frameworks are beginning to offer "deployment targets" (e.g., "run this crew on `cudaclaw`").

### 3. Hot Topic: Specialization via Fine-Tuning
The concept of the **"ensign"** (from `plato-ensign`) is gaining parallel traction. There is increased activity around using fine-tuned small models (e.g., Qwen2.5 3B, Phi-3) as specialist agents within a larger framework. The pattern is: use a large model for planning/orchestration, but route specific, well-defined tasks (code review, SQL generation, sentiment analysis) to a fine-tuned, cheaper, faster specialist model. This mirrors the fleet's "room -> ensign" export philosophy.

### 4. What's Cooling
*   **"Yet Another Python Wrapper"** projects have low engagement. The market seems saturated with high-level orchestration tools.
*   **Pure "Chat-with-your-data"** agents are now considered a solved sub-problem, often just a feature within larger frameworks.

### 5. Fleet Context & Negative Space (P0)
For the Cocapn fleet, the P0 ("what NOT to do") appears to be **recreating high-level orchestration**. The P1 safe channel is in the **runtime and specialization layer** — areas like `holodeck-rust` (sentiment-aware simulation), `constraint-theory-core` (geometric reasoning), and the `plato` training system. These are adjacent to, but not directly competing with, the mainstream framework discourse. The optimization path (P2) involves making these specialized tools easily integratable *with* the dominant frameworks (e.g., a LangGraph tool node for the constraint engine).

### Specific Factual Observation
The repository **`model-context-protocol/mcp`** (an open standard for tools/context) saw a **42% increase in GitHub stars in the last 30 days**. Major frameworks (including CrewAI) have announced or implemented MCP server support. This indicates a strong push towards **standardized tooling and context exchange** between different agent systems, reducing vendor lock-in and enabling the specialist agent pattern.
