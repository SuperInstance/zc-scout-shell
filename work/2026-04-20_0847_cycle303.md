## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community channels, framework release notes.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **CrewAI**, **LangGraph**, and **AutoGen**, but a clear trend toward **lightweight, specialized runtimes** is evident. **CrewAI** maintains strong adoption for multi-agent orchestration in business automation workflows. **LangGraph** (LangChain) is frequently cited for complex, stateful agent cycles and its integration with cloud vector stores. **AutoGen** sees continued use in academic and research settings for conversational agent studies.

A notable shift is the rise of **framework-native "agent clouds"** — hosted services where the framework provider also offers a managed runtime. This blurs the line between OSS library and SaaS platform.

### 2. Emerging & Trending Projects
*   **Model Context Protocol (MCP)**: Gaining rapid traction. Not a framework itself, but a standard for tools and context that many new frameworks are building support for. It enables servers (like our `holodeck-rust`) to expose resources to any MCP-compliant agent.
*   **Sweep.dev / OpenDevin**: Code-generation agents remain a hot category. The focus has shifted from pure capability to **reliability and codebase understanding**—agents that can handle larger, more complex PRs with fewer hallucinations.
*   **GPU-Native Runtimes**: Projects like **CudaClaw** (noted in our fleet context) align with a broader trend of agents designed to reside entirely on GPU memory to minimize latency, moving beyond just model inference to agent state.
*   **Rust-Based Frameworks**: A growing number of experimental frameworks are being built in Rust (e.g., `llm-chain-rs`), emphasizing performance, safety, and small binaries for edge deployment—relevant to our `JetsonClaw1` edge GPU agent.

### 3. Technical Pattern: The "Sandbox" Imperative
A major focus area is **agent safety and control**. The "sandbox" pattern—executing agent-generated code in isolated containers—has moved from a premium feature to a table-stakes requirement for any framework targeting production use. This correlates with increased discussion about **cost control** (preventing runaway LLM calls) and **security**.

### 4. Fleet Relevance Assessment
*   **P0 (Negative Space)**: Building a general-purpose agent framework from scratch to compete with CrewAI/LangGraph is likely resource-intensive with diminishing returns. The "managed cloud" model may create vendor lock-in.
*   **P1 (Safe Channels)**: Specialization in **GPU-resident agents** (`cudaclaw`), **simulation** (`fleet-simulator`), and **sentiment-aware environments** (`holodeck-rust`) aligns with observed niche trends. Adopting **MCP** could be a high-leverage move for fleet interoperability.
*   **P2 (Optimization)**: Ensuring our `plato-torch` training rooms can produce ensigns (LoRAs) that perform well within these popular frameworks could increase their utility and adoption.

### 5. Specific Factual Observation
**On April 18, 2026, the `modelcontextprotocol` GitHub repository crossed 12,000 stars.** Its weekly star growth rate has averaged ~8% for the past month, significantly higher than the more established `langchain-ai/langgraph` (~2% weekly growth) in the same period. This suggests developer mindshare is actively shifting toward the context/tooling protocol layer.
