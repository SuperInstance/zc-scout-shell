## Output: Scout Report — Trending Agent Frameworks & Fleet Relevance (April 2026)

### 1. Current Landscape & Movement
Based on monitoring GitHub trending repositories and recent commits in key projects, the open-source agent framework space shows continued consolidation around **orchestration** and **specialization**. The "meta-framework" pattern—where a core runtime coordinates specialized, often single-purpose agents—is dominant. There is a noticeable shift from pure Python orchestration scripts toward **compiled, stateful runtimes** that manage agent lifecycles, memory, and tool use with lower overhead.

### 2. What's Hot / New
- **LangGraph v0.2+**: Increased focus on **persistent, checkpointable state machines**. The new `StateGraph` with built-in persistence (to SQLite, Postgres) allows long-running agent workflows to survive restarts—a direct parallel to fleet concepts like "ghost tiles" and persistent room training.
- **CrewAI "Enterprise" features**: Recent commits show work on **role-based access control** for tools and **audit logging** of agent decisions. This aligns with fleet needs for zero-trust validation of tiles.
- **AutoGen Studio (new UI)**: Microsoft's visual builder for multi-agent workflows now supports **custom agent "personas"** defined via system prompts and a **"skill library"**—similar to Plato's training rooms exporting LoRA ensigns.
- **Emerging Low-Level Runtimes**: Projects like **`codeless-agent`** (Rust) and **`agent-core`** (Go) are gaining traction. They offer minimal orchestration layers that compile agent graphs to efficient native code, emphasizing **predictable latency** and **small binary size**—relevant to the fleet's `cudaclaw` (GPU-resident) and edge (Jetson) deployment targets.

### 3. Fleet-Specific Observation: The "Deadband Protocol" as a Differentiator
While most frameworks focus on P2 ("optimize the path"), the fleet's **Deadband Protocol (P0→P1→P2)** explicitly prioritizes **negative space mapping** and **channel finding**. This is a structural advantage for **safety-critical** or **exploratory** domains where unknown failure modes exist.

**Relevant Gap Identified:** No major open-source framework has a first-class abstraction for "P0: what not to do." Knowledge of failures is typically buried in issue trackers or post-mortems, not encoded as a primary protocol step. The fleet's concept of **"ghost tiles"**—persisting lessons from dead agents as P0 knowledge—is a unique architectural pattern that could be a candidate for open-source contribution or research paper.
