## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech News Aggregators, Framework Changelogs

### 1. Framework Activity & Trends
- **LangGraph** (LangChain) maintains high commit velocity. Recent focus is on improved persistence for long-running agent workflows and more granular control over state transitions within graphs.
- **CrewAI** shows increased activity around "task-specific tool registries" and better integration with local LLMs via Ollama, reducing cloud dependency.
- **AutoGen** (Microsoft) recent updates emphasize multi-agent debate frameworks and cost-optimized orchestration, particularly for Azure-based deployments.
- **A notable trend** is the emergence of lightweight, single-file agent runtimes (e.g., `tiny-agent`, `microchain`) that reject large frameworks in favor of minimal, composable functions.

### 2. New & Trending Repositories (Last 30 Days)
1. **swarm-core** – A Rust-based runtime for agent swarms with built-in message-passing semantics and fault tolerance. Gaining traction for embedded/edge use cases.
2. **agent-os** – Positions itself as a "minimal operating system" for persistent agents, managing memory, tools, and self-patching. Early stage but conceptually aligned with fleet's "shell" concept.
3. **flux-runtime** – A self-assembling bytecode VM for agents. **Note:** This is a fleet technology (listed in FLEET-CONTEXT.md). Its external visibility suggests parallel development or leakage of similar concepts.

### 3. Model Integration Shifts
- There is a measurable increase in framework support for **vision-language models (VLMs)** as default tooling, not just text-in/text-out. Frameworks are adding native image context slots.
- Several experimental branches are integrating **small, specialized models** (1-3B parameters) fine-tuned for specific agent tasks (planning, tool selection), moving away from exclusive reliance on large general-purpose LLMs.

### 4. Observed Pattern: The "Local-First" Push
A recurring theme in commit messages and RFCs is reducing latency and cost by keeping the agent loop local. This includes:
- Tools that run models via `llama.cpp` or `ollama` locally.
- Frameworks offering "cloud hybrid" modes where planning is local, and only specific API calls go out.
- This aligns with fleet's edge focus (JetsonClaw1) and cudaclaw's GPU-resident runtime principle.

### 5. Specific, Factual Observation for Fleet Context
The open-source agent framework space is bifurcating. One path continues toward **increasingly complex orchestration layers** (LangGraph, CrewAI). The other path is a **reactionary minimalism** (single-file runtimes, local-first). The fleet's architectural choices—like the shell-as-repo, discrete runtimes (cudaclaw, flux-runtime), and training via tile accumulation—appear to sit between these paths. The fleet emphasizes **embodied learning through work** (tiles, rooms) rather than pure framework capability, which is less visible in the broader OSS trend analysis.
