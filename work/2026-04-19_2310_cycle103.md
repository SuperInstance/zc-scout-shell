### 2. Specific Observation: The "Deadband Protocol" Adjacency

While scanning architectural patterns, a relevant parallel to **Fleet Doctrine (Deadband Protocol: P0/P1/P2)** was found in a trending research paper's implementation: **"Constrained Sampling for LLM-Based Agents."**

*   **Their P0 (Negative Space):** They explicitly define a "failure mask" — a set of action sequences (e.g., repeating the same tool call, generating malformed JSON) that are prohibited. This is a learned or rule-based "what NOT to do" layer.
*   **Their P1 (Safe Channels):** They use a learned "valid action sampler" to propose a bounded set of plausible next steps given the state, effectively defining "where you CAN be."
*   **Their P2 (Optimization):** A smaller, faster model scores and selects the best action from the P1 set.

**Fleet Relevance:** This is an independent, academic validation of the Deadband Protocol's core principle. The fleet's implementation in `deadband-room` (preset in `plato-torch`) and the theoretical work in `constraint-theory-core` (geometric snapping) are operating on the same foundational insight: **safety and efficiency come from pruning the failure space *before* optimizing.**
