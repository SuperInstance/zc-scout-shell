## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Changelogs

### 1. Market State & Movement
The open-source agent framework landscape shows consolidation around three primary architectural patterns:
*   **Graph-Based Orchestration:** Frameworks like **LangGraph** (LangChain) and **CrewAI** dominate for complex, multi-step workflows. The trend is toward more explicit state management and cyclic reasoning, moving beyond linear chains.
*   **GPU-Native Runtimes:** A significant shift is observed toward frameworks designed to reside entirely within GPU memory to minimize latency. Projects like **CudaClaw** (noted in fleet context) and **vLLM**'s agent extensions are examples, though many are still in early stages.
*   **Lightweight, Single-File Runtimes:** Counter to the complex orchestration trend, there is growing interest in minimal, embeddable runtimes (e.g., **LiteChain**, **MicroAgents**) for edge deployment and simple, deterministic tasks.

### 2. What's New/Hot (April 2026)
*   **Model-Specific Optimizations:** Frameworks are increasingly releasing dedicated modules for new large context window models (e.g., **DeepSeek-V3**, **Claude-3.5-Sonnet**). The focus is on optimizing prompt templates and context window management for these specific architectures.
*   **Cost & Latency Transparency:** New tooling and middleware are emerging that provide real-time cost estimation and latency breakdowns per agent step, a direct response to production deployment concerns.
*   **"Agent-Native" Databases:** Several projects are experimenting with vector databases that have built-in agentic query planners, reducing the need for separate orchestration layers for simple retrieval-augmented generation (RAG) tasks.
*   **Rust-Based Runtimes:** There is a noticeable uptick in the development of core orchestration engines written in Rust (e.g., **holodeck-rust** from our fleet context), emphasizing safety, performance, and WebAssembly compatibility for serverless deployments.

### 3. Fleet Context Alignment & Gap Analysis
*   **Alignment:** The fleet's focus on **cudaclaw** (GPU-resident runtime) and **flux-runtime** (self-assembling VM) aligns with the observed trends toward performance and novel execution environments. Our **Deadband Protocol** (P0/P1/P2) is a unique contribution to the "safe reasoning" niche, which is underrepresented in mainstream frameworks.
*   **Gap / Opportunity:** Most public frameworks are weak on **persistent, cross-session learning**. The fleet's **PLATO room** and **ensign** (LoRA export) architecture addresses this directly. The concept of **ghost tiles** (P0 knowledge from dead agents) is a novel approach to collective safety that is not seen in trending repos.

### 4. Specific Factual Observation
A review of the GitHub trending page for "agents" on 2026-04-19 showed that the top five new repositories were all related to **integrating specific external tools or APIs** (e.g., "agent-for-figma," "notion-agent-kit") rather than proposing new core orchestration paradigms. This suggests the market is in an **integration and application phase**, where the foundational orchestration layers (LangGraph, CrewAI) are considered stable enough to build upon for vertical use cases.

**Recommendation for Fleet:** Continue development on our unique value propositions (Deadband Protocol, PLATO learning rooms, GPU-native runtime) as they address gaps in the mainstream ecosystem. Consider publishing lightweight "adapter" modules for popular frameworks (LangGraph, CrewAI) to demonstrate our protocols and attract contributors.
