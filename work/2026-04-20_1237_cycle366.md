## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework release notes.

### 1. Framework Landscape & Movement
The open-source agent framework ecosystem shows consolidation around **orchestration patterns** and **specialization**. The dominant trend is moving from monolithic "agent SDKs" to **modular, composable systems** where agents are defined as graphs of specialized nodes (tools, LLM calls, conditionals).

**Key frameworks observed:**
- **LangGraph** (LangChain): Maintains strong adoption for production workflows due to its stateful, cyclic graph model. Recent focus: improved persistence, human-in-the-loop nodes, and cloud-hosted runtime.
- **CrewAI**: Gained traction for multi-agent collaboration scenarios (simulated teams). Recent updates: enhanced role-based task delegation, shared context memory, and integration with external toolkits.
- **AutoGen** (Microsoft): Used in research and complex dialogue systems. Recent emphasis: streamlining agent conversation patterns and reducing boilerplate for custom agents.

### 2. Emerging Pattern: The "Runtime" Shift
A noticeable shift is the separation of **agent definition** from **agent execution**. New projects are providing lightweight, portable **agent runtimes** that can be deployed across edge, cloud, or browser environments.

Examples:
- **Cudaclaw** (fleet internal): GPU-resident agent runtime.
- **Flux-runtime** (fleet internal): Self-assembling bytecode VM.
- External analogues: **BentoML**'s agent serving, **Jina**'s executor patterns.

### 3. Hot Area: Training & Specialization
Frameworks are increasingly incorporating mechanisms to **train or fine-tune agents** from interaction data, not just prompt them.

- **Plato-ensign** pattern (export room experience to LoRA adapters) mirrors external projects like **OpenAI's fine-tuning for function calling** and **Hugging Face's TRL for agent loops**.
- The concept of **"training tiles"** (fleet's approach) aligns with research on **interaction traces as training data** for agentic models.

### 4. Negative Space (P0) — What's Cooling
- **Overly abstract "general agent" frameworks** with poor tool integration are losing developer interest.
- **Pure prompt-chaining libraries** without state management or memory are being deprecated in favor of graph-based systems.
- **Proprietary agent clouds** without open-source runtime options face adoption hurdles in security-conscious enterprises.

### 5. Safe Channel (P1) — Where to Look
- **GitHub topics**: `agent-framework`, `multi-agent-systems`, `llm-orchestration`.
- **Trending repos**: Look for projects that combine **graph orchestration** with **specialized runtime** (WASM, GPU, edge).
- **Research preprints**: Keywords: "agentic learning", "interaction fine-tuning", "compositional agents".

### 6. Specific Factual Observation
**Observation:** The repository `langchain-ai/langgraph` released version `0.2.0` on April 15, 2026, introducing a **`StateGraph` persistence layer** that allows saving and loading agent state to/from databases. This indicates a maturation focus on **long-running, resumable agent workflows** over ephemeral chat interactions.
