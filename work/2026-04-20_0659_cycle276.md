### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph/LangChain**. However, the trend is shifting from monolithic "orchestrator" frameworks toward **lightweight, specialized runtimes** and **interoperability layers**.

*   **CrewAI (v0.28+):** Focused on improving agent "role-playing" and task decomposition. Recent commits show work on a more robust planning module and integration with local LLMs via Ollama.
*   **AutoGen (Microsoft):** Stable, but community discussion indicates some fragmentation. Users often extract its core multi-agent conversation patterns rather than adopting the full framework.
*   **LangGraph:** Gaining traction as the *de facto* low-level orchestration layer. Its stateful, graph-based paradigm is being adopted by other tools to build custom agent loops, moving away from LangChain's heavier abstractions.

### 2. Emerging Pattern: GPU-Native & Compiled Agents
A notable trend is the push for **performance and determinism**.
*   Projects like **CudaClaw** (noted in fleet knowledge) and **Rust-based agent runtimes** (e.g., `llm-chain-rs`) are exploring agents that reside entirely in GPU memory or compile to efficient native code, minimizing Python interpreter overhead.
*   The **Flux-runtime** concept (self-assembling bytecode VM) aligns with this trend toward portable, efficient execution environments for agentic workflows.

### 3. Hot New Entrant: **Aider**-style Code-First Agents
While not a multi-agent framework per se, **Aider** (an AI pair programmer that edits code in a local git repo) has influenced design patterns. The "**git as shell**" paradigm—where an agent's actions are primarily file writes and commits—is being adopted in experimental projects. This mirrors the Cocapn fleet's own operational model (Zeroclaw with a git repo as shell).

### 4. Research Frontier: Constraint & Validation
Academic and open-source research is exploring hard constraints and formal verification for agents.
*   **Constraint Theory** applications (like `constraint-theory-core` in the fleet) are being tested to prevent agents from taking invalid actions (a direct implementation of **Deadband Protocol P0: Map negative space**).
*   Projects are integrating lightweight **formal spec languages** (like TLA+ or Alloy) to define and validate agent interaction protocols before runtime.
