## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized runtime layers**.

*   **CrewAI** continues to emphasize role-based agent teams with shared context and task delegation. Recent commits (last 7 days) show increased focus on integration with local LLMs via Ollama and LM Studio, reducing cloud dependency.
*   **AutoGen** is pushing its "Conversable Agent" paradigm harder, with recent development on more sophisticated tool-calling and code execution workflows. There is visible activity around multi-modal agent capabilities in experimental branches.
*   **LangGraph** is gaining traction as the preferred low-level orchestration layer *underneath* other frameworks. Its stateful, graph-based paradigm is being adopted by teams building custom, complex agent loops, making it a foundational dependency rather than a standalone user-facing tool.

### 2. Emerging Pattern: The "GPU-Resident" Runtime
This aligns with fleet technology **cudaclaw**. A new category is emerging, focused on minimizing latency by keeping the agent's state, context, and even model weights pinned in GPU memory.

*   **Example:** Projects like **`aici`** (Artificial Intelligence Control Interface) by Microsoft and **`llm-chain`** in Rust ecosystems are experimenting with pre-allocated GPU buffers for prompt templates and intermediate KV caches to achieve sub-millisecond token-to-token latency in agentic loops.
*   **Observation:** The hot topic is not just parallel function calling, but **persistent, stateful agents** that avoid the cost of serialization/deserialization and context re-loading between tool calls. This is a P1 "safe channel" for research.

### 3. Integration with "Model Routers"
Frameworks are increasingly agnostic to the underlying LLM. The integration point is shifting to **intelligent routing layers** that choose between multiple models (e.g., GPT-4, Claude 3, local Llama) based on cost, latency, and task type.

*   **Specific Project:** **`OpenRouter`** and **`LiteLLM`** are becoming common plumbing in agent stacks. A pattern observed in several trending repos is: `LangGraph` (orchestration) -> `LiteLLM` (routing/load balancing) -> `vLLM`/`TGI` (inference).
*   **This creates a P0 "negative space":** Hardcoding API calls to a single model provider (e.g., `openai.ChatCompletion.create`) is now considered an anti-pattern in new, forward-looking projects.

### 4. Research Frontier: Long-Horizon Planning & Self-Correction
The current limitation of most frameworks is their reactivity. The "hot" research, visible in arXiv pre-prints and conference workshop submissions, is on integrating **search-based planning algorithms** (Monte Carlo Tree Search, A*) with LLM-based evaluation functions to enable agents to plan multi-step sequences, simulate outcomes, and self-correct before execution.

*   **Fleet Relevance:** This directly connects to the **Deadband Protocol's P0/P1/P2** phased approach. External research is trying to formalize the "map negative space" (P0) step algorithmically.
*   **Example Library:** **`Tree-of-Thoughts`** implementations are being generalized and packaged as a pluggable module for existing frameworks.

### 5. Concrete, Factual Observation for Fleet Context
**As of April 2026, the most active pull requests and discussion threads in the `CrewAI` repository are concerning the `crewai-kit` extension system and improving the `CacheHandler` to work with vector databases for long-term agent memory across sessions.** This indicates the community's immediate practical need is for **extensibility and memory persistence**, not necessarily net-new agent paradigms.
