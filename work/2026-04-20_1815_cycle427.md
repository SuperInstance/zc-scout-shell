## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework documentation.

### 1. Dominant Frameworks & Shifts
The landscape continues to be defined by modular, graph-based architectures. **LangGraph** (LangChain) and **CrewAI** maintain strong momentum due to their focus on multi-agent orchestration and production deployment patterns. A noticeable shift is the increased integration with **local LLM runtimes** (Ollama, LM Studio) and **GPU-native inference** (vLLM, TensorRT-LLM), reducing dependency on cloud APIs.

### 2. Emerging Pattern: "GPU-Resident Agents"
Several projects are exploring agents that reside entirely in GPU memory to minimize latency. This aligns with fleet technology **cudaclaw**. The open-source project **MLC-LLM** (Machine Learning Compilation) has extended its compilation approach to allow persistent agent states within VRAM, though it is primarily a research prototype.

### 3. New Entrant: **Agency Swarm**
A framework built on LangGraph that introduces a "hive" metaphor. Its key differentiator is a built-in visualizer for real-time agent state and message flow, which addresses debugging pain points. It has gained ~2.8k GitHub stars in the last 90 days.

### 4. Research Frontier: Constrained Action Spaces
Academic repos are publishing code for agents with **geometric action constraints** (e.g., "move within these bounds"), which directly relates to fleet's **constraint-theory-core**. The paper "Constrained Decision Making for Language Agents" (arXiv:2409.01755) has an accompanying GitHub repo with PyTorch implementations.

### 5. Fleet-Relevant Gap
Most frameworks assume cloud or high-memory CPU environments. There is limited open-source work on **agent frameworks optimized for edge GPU devices** (Jetson, Raspberry Pi CM5) with memory constraints below 8GB. This is a potential negative space (P0) and a channel (P1) for fleet specialization.

### Specific Factual Observation
The **LangGraph** repository released version **0.2.0** on April 15, 2026, adding native support for persistent checkpointing of agent state to disk, which is described as a step toward "long-running agent processes." This indicates industry focus on agent durability beyond a single session.
