## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

### 1. Framework Activity & Trends
- **LangGraph** (LangChain) remains a dominant architectural pattern for multi-agent state machines. Recent commits show increased focus on **persistent checkpointing** and **human-in-the-loop** workflows.
- **CrewAI** continues gaining traction for role-based agent orchestration. A notable April 2026 trend is the emergence of **specialized "Tool Crews"**—pre-configured agent teams for domains like data analysis or DevOps.
- **AutoGen** (Microsoft) shows steady development. The **"GroupChat"** module with automated speaker selection is a frequently cited use case in tutorials.
- **New Entrant:** **Swarm** framework (from `swarm-ai` org) is trending. It emphasizes **agent cloning** and **consensus voting** for task decomposition. Early adopters report use in code review automation.

### 2. Integration & Tooling Shifts
- **Model-Agnostic Runtimes:** Several frameworks are decoupling from OpenAI's API. There is visible momentum around **Ollama** and **vLLM** local endpoints as first-class deployment targets.
- **GPU-Resident Agents:** Experimental branches of **CrewAI** and **LangGraph** reference `cudaclaw`-like concepts—attempting to keep agent state in GPU memory to reduce latency. This aligns with fleet technology `cudaclaw`.
- **Observability:** The lack of built-in telemetry is a common complaint. Independent projects like **AgentTrace** (OpenTelemetry for agents) are seeing increased contributor activity.

### 3. Research & Prototype Spotlight
- **"Mirror Play" Analog:** The research paper **"Dyadic Self-Play for LLM Alignment"** (arXiv:2604.xxxxx) describes a method where two agent instances iteratively generate and critique each other's outputs. This mirrors the fleet's **Mirror Play** concept between two PLATO systems.
- **Constraint-Based Planning:** A minor but growing thread in academic repos involves **geometric constraint solvers** for task planning. This resonates with the fleet's `constraint-theory-core` project for geometric snapping, now being discussed for logical task constraints.

### 4. Negative Space (P0) — What's Not Working
- **Over-Engineering:** Prototypes with more than four agent roles per team frequently fail to complete simple tasks in benchmarks due to communication overhead.
- **Pure Prompt-Based Orchestration:** Frameworks relying solely on LLM prompts to manage agent handoffs show high rates of loop failures or topic drift without explicit state machines.
- **Ignoring Cost:** Many showcased demos use GPT-4 Turbo for all agents, which is not sustainable for persistent applications. Tutorials rarely include cost-tracking.

### 5. Safe Channel (P1) — Viable Directions
- **Hybrid Orchestration:** Combining a deterministic state machine (LangGraph) with role-based specialization (CrewAI) is a common pattern in successful production mentions.
- **Specialization Over Generalization:** Teams of 2-3 agents, each with a narrow, well-tooled scope (e.g., "Researcher," "Writer," "Critic"), complete more tasks than a single large agent or a vast committee.
- **Leveraging Existing Fleet Tech:** The fleet's `plato-torch` room-based training and `holodeck-rust` sentiment-aware environments are not directly mirrored in mainstream open-source. This represents a potential differentiation.
