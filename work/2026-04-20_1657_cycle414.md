## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech Blogs, Framework Changelogs

### 1. Framework Maturation & Specialization
The landscape has shifted from general-purpose "orchestrator" frameworks toward specialized runtimes. Key trends:

*   **CrewAI** (v0.28+): Focus on role-based collaboration. Recent updates emphasize "process-driven" workflows with explicit `Process` classes (sequential, hierarchical). It's gaining traction for business automation where human-like team structures are intuitive.
*   **LangGraph** (v0.0.40+): Solidifying as the low-level "assembly language" for stateful, cyclic multi-agent systems. Its `StateGraph` and persistence features are being used as a backend for higher-level frameworks. The checkpointing system is a noted differentiator for long-running, recoverable agentic workflows.
*   **AutoGen** (v0.2+): Pivoting toward "conversable agents" with enhanced tool-calling and code execution. Its strength remains in research and rapid prototyping of conversational patterns, but faces criticism for production deployment complexity.

### 2. The Rise of the "GPU-Resident" and "Bootstrap" Paradigm
A new category is emerging, directly relevant to fleet architecture (cudaclaw, flux-runtime):

*   **GPU-Resident Agents:** Frameworks like **CudaClaw** (concept) and **vLLM**-based agent runtimes aim to minimize CPU-GPU latency by keeping agent state and model context in GPU memory. This is considered "hot" for latency-sensitive applications.
*   **Bootstrap/Self-Assembling Systems:** Inspired by projects like **flux-runtime**, there is increased research interest in agents that can modify their own code or architecture based on runtime constraints. This is mostly in early research (arXiv papers) but is a frequent topic in agent design discussions.

### 3. Integration with Inference Servers
A clear pattern is the tight coupling of agent frameworks with high-performance inference servers, moving away from direct OpenAI API calls.

*   **Common Stack:** `Agent Framework` + **LM Studio** / **ollama** / **TGI** (Text Generation Inference) for local model serving.
*   **Result:** This enables cheaper, faster, and more controllable iterations, supporting the trend toward smaller, fine-tuned "ensign" models (cf. plato-ensign).

### 4. What's NOT Trending (P0: Negative Space)
*   **Monolithic "AGI-in-a-box" projects** have lost significant momentum. The community skepticism toward overhyped, undifferentiated "AI assistant" repos has increased.
*   **Pure prompt-chaining libraries** without state management are now considered a solved base layer, not a framework.
*   **Framures** (Framework + Future) that promise radical new paradigms without working code see rapid disengagement.

### 5. Specific, Factual Observation for Fleet Context
**Observation:** The most active forks and contributions to **LangGraph** are in repositories implementing **simulation environments** (e.g., multi-agent coding benchmarks, synthetic user simulators). This aligns with the fleet's `fleet-simulator` project. The community is investing in tools to generate high-quality, structured interaction data for training and evaluating specialized agents, which validates the fleet's "tile-based training" architectural premise.
