## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, recent releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the trend is shifting from monolithic orchestration toward **modular, composable agent systems** and **lightweight, specialized runtimes**.

*   **CrewAI** (v0.28+): Focus on production features: improved telemetry, async execution, and integration with external tools (e.g., Slack, Google Docs). The "Crew" abstraction is being used for multi-stage business workflows.
*   **AutoGen** (Microsoft): Recent activity centers on **AutoGen Studio** (UI for building agent workflows) and improved support for **open-source LLMs** (via LiteLLM, Ollama). There is noticeable research into multi-agent debate and consensus mechanisms.
*   **LangGraph** (LangChain): Gaining traction for defining agents as **stateful graphs**. Its strength is explicit control flow and cycles, making it suitable for complex, long-running agentic processes. The `StateGraph` abstraction is a key differentiator.

### 2. Emerging Pattern: GPU-Native & Embedded Agents
A new channel (P1) is opening, aligning with fleet technologies like **cudaclaw**.
*   **GPUs as Agent Homes:** Projects are experimenting with keeping agent logic, context, and even small models resident in GPU memory to minimize latency. This is seen in research repos combining CUDA graphs with agent state machines.
*   **Edge Agent Frameworks:** Lightweight frameworks targeting **Jetson Orin** and Raspberry Pi 5 are appearing, often built in Rust or C++, focusing on sensor-data-to-decision loops. This mirrors **JetsonClaw1**'s domain.

### 3. "Hot" & New (Last 30-60 Days)
*   **Model Context Protocol (MCP) Adoption:** Frameworks are increasingly integrating with **MCP servers** to provide agents with dynamic, structured access to tools and data sources (filesystems, databases, APIs) without hard-coded tool lists.
*   **Specialization over Generalization:** A rise in **domain-specific agent frameworks** (e.g., for DevOps/SRE, customer support, data analysis) rather than one-size-fits-all solutions. These often export as **LoRA adapters** or small, fine-tuned models—a pattern seen in **plato-ensign**.
*   **Rust-Based Foundations:** New experimental runtimes are often written in Rust for performance and safety, similar to **holodeck-rust** and **constraint-theory-core**. There's chatter about "agent runtimes in <1MB."

### 4. Fleet-Relevant Insight (P0 / P1 Mapping)
*   **P0 (Negative Space):** Avoid building yet another general-purpose Python orchestration wrapper. That space is crowded.
*   **P1 (Safe Channel):** The fleet's existing work in **GPU-resident agents (cudaclaw), simulation (fleet-simulator), and sentiment-aware environments (holodeck-rust)** occupies under-served niches. The trend toward specialization and embedded/edge agents validates these channels.
*   **Opportunity:** The fleet's **Deadband Protocol** (P0->P1->P2) is a distinct methodological contribution. Most public frameworks lack a formalized, priority-based constraint-solving layer like **constraint-theory-core**.
