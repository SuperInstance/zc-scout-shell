### 1. Open-Source Agent Framework Brief (April 2026)

Based on monitoring GitHub and recent releases, the landscape shows continued evolution from monolithic frameworks toward specialized, modular runtimes. The trend aligns with fleet architecture principles (e.g., cudaclaw, flux-runtime).

**Notable Developments:**
*   **CrewAI & LangGraph:** These remain popular for orchestrating multi-agent workflows. Recent commits (last 7 days) show increased focus on **interoperability**—adding adapters for external tooling and model endpoints. This suggests a move away from closed ecosystems.
*   **AutoGen (Microsoft):** Activity has shifted toward **research branches** exploring agentic behavior in constrained, simulated environments. This mirrors aspects of the fleet's `fleet-simulator` and `holodeck-rust` projects.
*   **Emerging Pattern - GPU-Native Agents:** Several new, smaller repos (e.g., `infernet-rs`, `llama-edge`) are experimenting with agents that reside primarily in GPU memory, minimizing CPU-GPU data transfer. This is a direct parallel to the fleet's `cudaclaw` project.
*   **Hot Topic - Self-Assembling Systems:** There is growing research interest (evident in arXiv preprints) in systems where agents or their components can reconfigure their own communication and execution pathways at runtime. The fleet's `flux-runtime` (self-assembling bytecode VM) is positioned within this research vector.

**P0 (Negative Space):** Monolithic frameworks that require full model loading per agent instance are becoming less viable for high-throughput or edge deployment. Tight coupling to a single model provider (e.g., only OpenAI) is also a common constraint.

**P1 (Safe Channels):** Modular architectures that separate planning, execution, and memory. Frameworks that support multiple, swappable model backends and have efficient state management show more sustained growth.
