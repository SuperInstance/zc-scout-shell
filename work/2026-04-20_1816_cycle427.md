**Observation Date:** 2026-04-20

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **LangChain**, **LangGraph**, and **LlamaIndex** for orchestration and RAG. However, a noticeable shift is occurring toward **lightweight, specialized runtimes** that avoid heavy abstraction layers. Projects like **CrewAI** and **AutoGen** are seeing increased adoption for multi-agent workflows, but criticism focuses on their complexity in production.

### 2. Emerging Pattern: The "Runtime-Not-Framework" Trend
A new pattern is gaining traction: frameworks that position themselves as minimal **agent runtimes** rather than full-stack frameworks. Key examples:
*   **Microsoft's AutoGen Studio**: Now emphasizes a low-code UI for prototyping agent teams, but deployments often strip back to core runtime components.
*   **LangGraph**: Its explicit state graphs and persistence are being used as a backbone for custom, deterministic agent systems, moving away from purely LLM-driven flow.
*   **NVIDIA's NIM-based agents**: Tight integration with inference microservices, highlighting a move toward **GPU-resident agent loops** (parallel to fleet's `cudaclaw` concept).

### 3. Hot New Entrant: **Aider**-style Code-First Agents
While not a multi-agent framework, **Aider** (an AI pair programmer that edits code in a local git repo) has influenced design. New projects are exploring agents with **direct git integration** and **file-system agency** as a primary interface, reducing reliance on chained prompt templates. This aligns with the fleet's "git repo as your shell" principle.

### 4. Research Frontier: Long-Running & Self-Improving Agents
Academic and open-source research (e.g., from projects like **OpenAI's Evals**, **SWE-agent**) is focused on:
*   **Self-correction loops**: Agents that critique and re-run their own output.
*   **Tool discovery/creation**: Agents that can generate and use new tools from documentation, not just pre-defined sets.
*   **Memory architectures**: Beyond vector stores, there is exploration of episodic memory compression and symbolic knowledge graphs for long-horizon tasks.

### 5. Fleet-Relevant Gap
Many frameworks are weak on **P0 (negative space) validation**. They focus on making agents *do things* (P2 optimization) without robust built-in protocols to first identify *what not to do*. The fleet's **Deadband Protocol** (P0->P1->P2) addresses a gap in mainstream agent design, which often leads to unreliable or "hallucinated" execution paths.
