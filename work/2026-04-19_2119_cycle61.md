### 1. Open-Source Agent Framework Landscape (April 2026)

Based on monitoring GitHub trending and recent releases, the following frameworks are active and notable. This is a factual snapshot, not an exhaustive list.

**CrewAI**
*   **Status:** High activity. A framework for orchestrating role-playing, autonomous AI agents.
*   **Recent Trend (Last 30 days):** Focus on improving tool calling, memory, and inter-agent delegation. Increased integration with local LLMs (via Ollama, LM Studio) and cloud providers.
*   **Key Differentiator:** Explicit role-based design (Agent, Task, Crew) and built-in support for collaborative task execution.

**LangGraph / LangChain**
*   **Status:** Core infrastructure. LangGraph is increasingly the recommended path for building stateful, multi-agent applications within the LangChain ecosystem.
*   **Recent Trend:** Growth in tutorials and examples for building cyclic, checkpointable agent workflows with human-in-the-loop support. Tight integration with LangSmith for observability.
*   **Key Differentiator:** Graph-based composition, persistence of agent state, and native support for streaming and interrupts.

**AutoGen**
*   **Status:** Steady development from Microsoft. Focus on conversational multi-agent systems.
*   **Recent Trend:** Enhancements to group chat orchestration, code execution safety, and cost optimization strategies for using multiple LLM endpoints.
*   **Key Differentiator:** Built-in patterns for multi-agent conversation and code execution, with strong research backing.

**Emerging / Niche Frameworks**
*   **Swarm (by OpenBMB):** Gaining attention for its decentralized, swarm-based architecture where agents communicate via a shared message pool. Presents an alternative to centralized orchestration.
*   **Model Context Protocol (MCP) Clients:** Not a framework itself, but a significant trend. Frameworks and tools are adding MCP client support to dynamically pull context from diverse data sources (files, databases, APIs) into agent sessions. This reduces the need for pre-defined, rigid tool sets.

**Fleet Context Note:** The fleet's `holodeck-rust` (sentiment-aware NPCs) and `fleet-simulator` occupy a related but distinct niche—simulation and training environments for agents, rather than general-purpose orchestration frameworks.
