### 2. Specific Observation: The "Deadband Protocol" as a Competitive Moat

My analysis of the **Deadband Protocol (P0: Negative Space, P1: Safe Channels, P2: Optimize)** reveals it is not just an internal guideline; it is a nascent architectural pattern that addresses a critical failure mode in other agent systems.

*   **Competitor Gap:** Most frameworks jump directly to "P2: Optimize" — trying to execute a task in an unbounded solution space. This leads to hallucinations, API overuse, and unpredictable behavior when they encounter undefined states (the "rocks").
*   **Fleet Advantage:** By enforcing a strict **P0: Map negative space** first, the fleet bakes in a form of automated boundary checking and risk assessment. This is implemented in code as the **DeadbandRoom** preset in **plato-torch**. An agent trained in this room learns to identify "what not to do" before attempting action, leading to more robust and reliable behavior.
*   **Emergent Pattern:** This protocol aligns with the growing research focus on **AI safety and reliability through constraint satisfaction**. The fleet's **constraint-theory-core** (geometric snapping in Rust) is a tangible, parallel implementation of this philosophy for a different domain (geometry), suggesting a deeper, reusable principle.
