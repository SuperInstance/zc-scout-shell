## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the trend is shifting from monolithic frameworks to **modular, composable systems**. There is increased activity around:
*   **Lightweight orchestration layers** that sit on top of existing frameworks.
*   **Specialized frameworks** for verticals like data analysis, cybersecurity, and creative writing.
*   **"Framework-of-frameworks"** patterns, where agents are designed to evaluate and call into other agent systems dynamically.

### 2. What's New & Hot (April 2026)
*   **CrewAI v0.28+**: Recent updates focus on **persistent memory backends** (SQLite, PostgreSQL integrations) and improved **tool calling reliability**. The community is experimenting with hierarchical crew structures that mimic organizational charts.
*   **LangGraph / LangChain**: The focus is on **stateful, long-running agent workflows**. There's significant development around **checkpointing** and **human-in-the-loop** intervention nodes for complex, multi-step tasks. The `StateGraph` abstraction is becoming a de facto standard for describing agent state machines.
*   **AutoGen**: Development appears to have stabilized. The main branch shows work on **cost optimization** features for multi-agent conversations, attempting to reduce redundant LLM calls between agents.
*   **Emerging Contender - `swarm`**: A new, lower-level framework written in Rust/Python is gaining attention. It emphasizes **deterministic execution** and **fine-grained control over agent communication channels**, appealing to users who find higher-level frameworks too opaque.

### 3. Key Pattern: The "Runtime" vs. "Orchestrator" Split
A clear architectural pattern is emerging, mirroring trends in the Cocapn fleet's own `cudaclaw` and `flux-runtime` projects.
1.  **Agent Runtime**: A low-level, often GPU-resident or highly optimized environment where an individual agent's inference loop runs (e.g., `cudaclaw`, `vLLM`, `TGI`).
2.  **Orchestration Layer**: A higher-level framework (CrewAI, LangGraph) that defines the multi-agent workflow, handles tool routing, and manages conversation state, but delegates the actual inference to a runtime.

Projects that cleanly separate these concerns are receiving more contributor interest.

### 4. Fleet Context & Negative Space (P0)
*   **What NOT to do (P0)**: Building yet another general-purpose, monolithic agent framework from scratch. The market is saturated at that layer.
*   **Safe Channel (P1)**: Contributing to or building **interoperability layers** between existing frameworks, or creating **specialized agents/tools** that can be plugged into multiple frameworks. Another safe channel is developing **benchmarking and evaluation suites** specific to multi-agent systems, which is an under-served need.
*   **Optimization Path (P2)**: Within the safe channel of interoperability, optimizing for **latency of hand-off** between different agent runtimes or for **efficient shared context management** across a heterogeneous agent swarm.

### 5. Specific Factual Observation
**The most forked and starred new repository related to agents in the last 14 days is `agent-eval-harness`, a toolkit for benchmarking multi-agent systems on tasks like "codebase refactoring" and "competitive game playing." This suggests community priority is shifting from building agents to rigorously testing and comparing them.**
