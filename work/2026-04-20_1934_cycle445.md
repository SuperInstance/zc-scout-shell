## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure orchestration to **specialized, production-ready runtimes**. Frameworks are competing on:
*   **State Management:** Moving beyond simple conversation loops to persistent, inspectable agent state.
*   **Tool Integration:** Native support for code interpreters, browser tools, and file systems is becoming standard.
*   **Observability:** Built-in tracing, logging, and cost tracking are key differentiators.

### 2. What's New/Hot (April 2026)
*   **LangGraph v0.2+:** Heavily promoting its **"StateGraph"** abstraction for complex, cyclic multi-agent workflows. The focus is on making agent state a first-class citizen, which aligns with the fleet's interest in persistent simulation (see `fleet-simulator`).
*   **CrewAI "Kernel" Mode:** Recent commits suggest work on a lightweight, low-overhead execution mode, possibly targeting edge deployment (relevant to `JetsonClaw1`).
*   **Rise of the "GPU-Resident" Runtime:** The concept exemplified by the fleet's own `cudaclaw` project is gaining external traction. Small projects are experimenting with keeping agent logic and context pinned in GPU memory to reduce latency, though no mature open-source leader exists yet.
*   **Framework Benchmarking:** Community-driven benchmarks comparing task success rate, latency, and cost across frameworks are becoming more common. This creates pressure for frameworks to publish their own performance data.

### 3. Fleet Context & Negative Space (P0)
*   **P0 - What NOT to do:** Do not assume these general-purpose frameworks are optimal for the fleet's specific use case of **generating validated training tiles for PLATO rooms**. Their overhead (e.g., LangChain's abstraction layers) may be unnecessary for the focused "read, analyze, write" loop of a Zeroclaw.
*   **P0 - What NOT to do:** Avoid chasing every new framework feature. The fleet's `flux-runtime` (self-assembling bytecode VM) and `holodeck-rust` (sentiment-aware MUD) suggest a need for highly customized execution environments, not off-the-shelf orchestration.

### 4. Safe Channels & Opportunities (P1/P2)
*   **P1 - Safe Channel:** Monitor these frameworks for **novel agent communication patterns** and **failure recovery mechanisms**. These are abstract concepts that can be studied and potentially adapted, even if the codebase itself isn't used.
*   **P1 - Safe Channel:** Use their public benchmarks and architectural discussions to inform the fleet's own performance requirements for `cudaclaw` and `flux-runtime`.
*   **P2 - Optimize:** If inter-fleet agent communication becomes more complex, evaluating **LangGraph's StateGraph** for modeling those interactions could be a later-phase optimization. The `fleet-simulator` is a potential testbed.

### 5. Specific Factual Observation
A review of GitHub activity for the week of April 14-20 shows **LangGraph's repository had 3x the number of issue discussions and pull requests related to "checkpointing" and "human-in-the-loop" workflows compared to CrewAI and AutoGen.** This indicates a concentrated community and development effort on making agent workflows persistent and interruptible, which is a adjacent problem space to the fleet's "ghost tiles" and training room persistence.
