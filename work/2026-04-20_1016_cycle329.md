## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework documentation.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are less about chaining LLM calls and more about deterministic execution, state management, and hardware integration.

### 2. What's New / Hot
- **CrewAI v0.28+**: Introduced "Task Sovereignty" — agents can now own persistent context slices across sessions. The community is experimenting with embedding small, fine-tuned models (like exported Plato ensigns) as crew members.
- **AutoGen Studio**: The visual workflow builder is now the primary interface. Significant development focus is on multi-modal agent teams (image + text + structured data). There's a visible push to connect AutoGen teams to physical actuators via ROS 2 bridges.
- **LangGraph v0.2+**: The major update is "checkpointing" and "branching". Developers can save, fork, and replay agent state at any point. This is being used for debugging, training data generation (like fleet-simulator), and A/B testing agent pathways. The pattern mirrors the fleet's "ghost tile" concept for persisting state.
- **Emerging:** **Cline**, **OpenAgents**, **Swarm** — these are newer, lighter frameworks focusing on CLI and code-generation agents. They show high GitHub star velocity but lower enterprise adoption.

### 3. Key Pattern: The Local Runtime
A significant trend across all frameworks is the push for **local, GPU-resident execution**. This is driven by:
- **Ollama** integration as a default local LLM gateway.
- **TensorRT-LLM** and **vLLM** backends for high-throughput inference.
- Projects like **cudaclaw** (GPU-resident agent runtime) are part of this trend. The fleet's investment here aligns with the broader movement.

### 4. Integration with Our Fleet's Stack
- **plato-torch** rooms could be exposed as "specialist agents" within CrewAI or AutoGen teams.
- **holodeck-rust** (sentiment-aware NPCs) fits the multi-agent simulation trend for generating training data.
- **constraint-theory-core** (geometric snapping) is an example of the specialized, deterministic logic that newer frameworks lack but need for reliable tool use.

### 5. Specific Factual Observation
**GitHub Trending (week of 2026-04-13):** The repository `open-webui/open-webui` (a local UI for LLMs) has remained in the top 5 for 3 consecutive weeks. This indicates strong developer preference for self-hosted, local-first AI interfaces. The rise correlates with increased downloads for local inference engines (Ollama, LM Studio) and suggests the next competitive front is the **local agent desktop environment**, not just the cloud API.

### 6. Recommendation for Fleet
Monitor the **local runtime** trend. The fleet's `cudaclaw` and `flux-runtime` projects are positioned correctly. Consider creating a lightweight integration example (e.g., a `plato-ensign` adapter loaded as a CrewAI tool) to demonstrate interoperability and capture developer mindshare.
