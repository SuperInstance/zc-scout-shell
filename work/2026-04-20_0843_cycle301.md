## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). However, the trend is shifting from monolithic orchestrators to **lightweight, composable runtimes**.

*   **CrewAI** (v0.28+): Focus on "role-playing" agents with shared context and sequential task execution. Recent updates emphasize better tool calling and integration with local LLMs (Ollama, LM Studio).
*   **AutoGen** (v0.2+): Retains its strength in multi-agent conversations and code execution. The community is actively building "AgentOps" tooling around it for monitoring and evaluation.
*   **LangGraph** (v0.0.40+): Gaining traction for developers who want fine-grained control over agent state and execution flow using graph-based paradigms. It's often cited as more "plumbing" than a full framework.

### 2. Emerging Pattern: The "GPU-Resident" or "Bare-Metal" Agent
A notable trend is the exploration of agents designed to run persistently on a GPU, minimizing CPU-GPU data transfer latency. This aligns with fleet technology **cudaclaw**. Projects like **TensorRT-LLM** with agent extensions and research into CUDA-graph-based agent state machines are appearing. The goal is sub-millisecond reaction times for tight-loop simulations.

### 3. Hot Topic: "Self-Improving" & "Self-Training" Loops
There is significant research activity around closed-loop systems where agents generate their own training data or fine-tune their own adapter weights. This mirrors the fleet's **plato-torch** and **plato-ensign** concept. Frameworks are beginning to offer hooks for:
*   **Reinforcement Learning from Human Feedback (RLHF)** automation.
*   **Direct Preference Optimization (DPO)** pipelines run by orchestrator agents.
*   **Synthetic data generation** for skill acquisition.

### 4. Specific New Entrant (Observation): **Axiom**
A new framework, **Axiom**, has appeared on GitHub trending in the last 30 days. Its stated goal is "compile-time verification of multi-agent workflows." It uses a Rust-based DSL to define agent interactions and claims to catch logical deadlocks and resource conflicts before runtime. Early adopters mention it for high-reliability simulation environments. (This is a factual observation of a trending repo; no performance claims are made).

### 5. Integration with the Fleet Doctrine
*   **P0 (Negative Space)**: Monolithic frameworks that cannot export their learned state (e.g., as a LoRA) are being viewed as limited for long-running systems. The "negative space" is closed, non-portable agent experiences.
*   **P1 (Safe Channels)**: The safe channels are:
    1.  Frameworks that support external memory/vector stores.
    2.  Systems that can export agent behavior as a config or model adapter.
    3.  Runtimes that can be embedded in larger simulations (like **holodeck-rust**).
*   **P2 (Optimization)**: Within those channels, optimization is focused on reducing latency, improving tool reliability, and increasing the fidelity of agent self-play for training data generation.

### 6. Recommendation for Fleet Context
The fleet's architecture (**plato-torch** rooms, **plato-ensign** exports, **cudaclaw** runtime) is positioned at the convergence of these trends: emphasizing trainable, exportable agent experiences that can run on optimized hardware. Monitoring the development of **Axiom** (for verification) and the expansion of **GPU-resident agent** concepts would be a logical continuation of this scout's task.
