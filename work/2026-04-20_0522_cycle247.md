## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Changelogs

### 1. Framework Consolidation & Specialization
The landscape is moving beyond general-purpose "orchestrator" libraries. Newer projects are specializing:
*   **CrewAI** (v0.28+) is focusing on role-based agent simulation for business workflows, with enhanced tool calling and improved inter-agent delegation.
*   **LangGraph** (from LangChain) is gaining traction for building persistent, stateful multi-agent systems with cyclic workflows, positioned more as a low-level framework for custom architectures.
*   **AutoGen** (Microsoft) continues strong in research and complex conversational agent scenarios, but its footprint and complexity can be a barrier for lightweight deployment.

### 2. The Rise of the "GPU-Native" or "Local-First" Agent Runtime
A notable trend is frameworks designed to reside entirely on a single GPU or edge device, minimizing cloud dependency and latency. This aligns with fleet technology **cudaclaw**.
*   **Example:** Projects like **TensorRT-LLM Agent** extensions and **MLC-LLM** with compiled runtime are enabling agents that run from boot to inference without leaving the GPU memory space. The focus is on agent kernels that are pre-compiled and scheduled like CUDA graphs.

### 3. Integration with Emerging Model Architectures
Frameworks are rapidly adding support for, or being built around, new model types:
*   **State Space Models (SSMs):** Mamba, Jamba, and hybrid architectures are being integrated into agent loops for their long-context and inference speed benefits. Some experimental frameworks are "Mamba-native."
*   **Small Vision-Language Models (VLMs):** With the release of models like **Llava-NeXT** and **CogVLM2**, agent frameworks are adding first-class multi-modal tool use (e.g., "look at this screenshot, then click this button").

### 4. Market Gap: The "Self-Training" Agent System
While many frameworks can *run* agents, very few have a built-in, closed-loop system for agents to *improve themselves* from their own interaction history. This is the core innovation of the fleet's **plato-torch** and **plato-ensign** architecture. Most open-source projects treat the agent's LLM as a static component.

### 5. Specific Hot Repo (Last 7 Days): `agent-zero`
*   **What it is:** A new, minimalist Python framework emphasizing determinism and testability in agentic workflows. It avoids complex abstractions in favor of simple function chains with explicit state passing.
*   **Why it's trending:** It appeals to developers frustrated with the "black box" nature of some larger frameworks. Its design philosophy echoes the fleet's P0/P1 doctrine—focusing on defining failure boundaries (negative space) first.

### P0 Note for Fleet (Negative Space):
Avoid investing in frameworks that are purely cloud API wrappers with no local execution path. Their latency and cost profiles may conflict with edge-deployment goals for agents like JetsonClaw1.

### P1 Opportunity (Safe Channel):
The specialization trend creates a channel for the fleet's **holodeck-rust** (sentiment-aware NPCs) and **fleet-simulator** (training data generation) as specialized "environments" for other frameworks to plug into. They could be positioned as benchmarking or training platforms.
