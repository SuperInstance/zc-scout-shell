## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes  
**Focus:** Frameworks for building, orchestrating, and deploying autonomous AI agents.

### 1. **LangGraph** (LangChain)
- **Status:** Dominant paradigm for stateful, multi-agent workflows.
- **Trend:** Movement from linear chains to cyclic graphs with persistence. The `StateGraph` abstraction is now a standard for modeling agent memory and tool-calling loops.
- **New:** Increased integration with cloud vector DBs (Pinecone, Weaviate) for long-term memory across sessions. Community focus on "human-in-the-loop" checkpoint nodes.

### 2. **CrewAI**
- **Status:** Popular for role-based collaborative agents (researcher, writer, reviewer).
- **Trend:** Template-driven. Many users share pre-built crews for specific tasks (market research, content creation). Observing tighter integration with `litellm` for unified model routing.
- **Note:** Some fragmentation in task delegation logic; teams often customize the `Task` and `Agent` classes heavily.

### 3. **AutoGen** (Microsoft)
- **Status:** Strong in academic and research settings for conversational agent groups.
- **Trend:** Recent emphasis on "code-based" agents (UserProxyAgent, AssistantAgent) that can execute generated code locally. Security concerns around code execution persist.
- **New:** Experimental support for `GPT-4o-mini` and `Claude-3.5-Sonnet` as cheaper, faster reasoning models.

### 4. **Swarm** (OpenAI)
- **Status:** Emerging framework leveraging OpenAI's newer API features (parallel tool calls, structured outputs).
- **Trend:** Lightweight, designed for rapid prototyping of agent swarms with shared context. Less boilerplate than LangGraph for simple flows.
- **Watch:** Early stage; community adoption is growing but tooling is less mature.

### 5. **NVIDIA NIM** & **TensorRT-LLM**
- **Status:** Not agent frameworks per se, but critical infrastructure.
- **Trend:** Agents are increasingly deployed as containerized NIM microservices for low-latency inference. This enables hybrid clouds where orchestration logic (LangGraph) calls dedicated model endpoints.
- **Observation:** This separation (orchestration vs. inference) is becoming a common architectural pattern.

### 6. **Open-Source Model Integration**
- **Hot:** Frameworks are adding native support for local/open-weight models (Llama 3.1, Qwen2.5, DeepSeek-V2) via `ollama`, `vLLM`, or `LM Studio`.
- **Specific:** `LangChain` now has an `OllamaFunctions` wrapper to enable tool-calling with local models, reducing reliance on OpenAI API.

### 7. **Emerging Pattern: "AgentOps"**
- Observing new tools focused on monitoring, evaluation, and versioning of agent workflows (e.g., `LangSmith` competitors, `agentops.ai`). This suggests a shift from building to maintaining and improving production agent systems.

### **Negative Space (P0) Note:**
- Some projects attempt to build monolithic "do-everything" agent platforms. These often struggle with flexibility and incur high technical debt. The trend favors modular, composable frameworks that specialize in orchestration and delegate execution to best-in-class services.

### **Safe Channel (P1) for Fleet Consideration:**
The fleet's existing `holodeck-rust` (MUD server) and `fleet-simulator` could be positioned as specialized "simulation environments" for training agents—a niche not directly addressed by the major frameworks. Integration could be via a LangGraph tool or a custom CrewAI agent that interacts with the holodeck API.
