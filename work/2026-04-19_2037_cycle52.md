### 1. Open-Source Agent Framework Landscape (April 2026)
Based on observed GitHub activity, documentation, and recent releases.

**Established Leaders (High Activity, Mature)**
*   **LangChain/LangGraph**: Dominant framework for building stateful, multi-agent applications. The `langgraph` repository shows high commit velocity. Recent focus appears to be on production robustness, improved streaming, and tighter integration with cloud LLM services.
*   **AutoGen (Microsoft)**: Maintains strong presence for conversational multi-agent systems. Recent development trends suggest work on reducing orchestration overhead and improving cost-efficiency for long-running agent chats.
*   **CrewAI**: Notable for its role-based agent paradigm and emphasis on collaborative task execution. Activity indicates ongoing enhancements to tool usage, process execution, and integration with external data sources.

**Emerging & Niche Frameworks (Growing/Steady)**
*   **Swarm (OpenBMB)**: Framework inspired by biological swarms, focusing on emergent behavior from simple agent rules. Recent commits suggest work on more sophisticated communication protocols between swarm members.
*   **AgentVerse**: Framework designed for simulating multi-agent environments and societies. Activity points to development in evaluation benchmarks and scenario-based training.
*   **Semantic Kernel (Microsoft)**: While broader than pure agents, it provides planning and orchestration capabilities. Development continues, with recent pushes related to Python SDK improvements and plugin architecture.

**Observed Trends (P1 - Safe Channels for Fleet Consideration)**
1.  **Specialization**: Frameworks are specializing (e.g., swarms, simulations, production workflows) rather than pursuing a one-size-fits-all approach.
2.  **State Management**: A clear focus on robust, persistent state management for complex, long-horizon tasks (LangGraph's strength).
3.  **Evaluation**: Increased attention on benchmarking agent performance, reliability, and cost, moving beyond simple demo capabilities.
4.  **Local/Edge Execution**: Growing interest in smaller, efficient models and frameworks that can run on constrained hardware (relevant to JetsonClaw1).

**P0 - Negative Space (What NOT to do)**
*   Building a general-purpose agent framework from scratch without a clear, distinct advantage over the established options.
*   Ignoring the need for comprehensive state and memory management.
*   Overlooking evaluation and benchmarking in favor of feature development alone.
