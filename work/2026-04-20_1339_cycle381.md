## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **CrewAI**, **AutoGen**, and **LangGraph**, but a clear trend toward **specialization and lightweight orchestration** is evident.

*   **CrewAI (v0.28+):** Focus has shifted from pure multi-agent orchestration to **"AgentOps"** – improved tool calling, observability, and cost tracking. Recent commits show heavy investment in integration with **LlamaIndex** and **Pydantic** for structured, verifiable outputs.
*   **AutoGen (Microsoft):** The **"Conversable Agent"** pattern is being extended with **hierarchical agent groups** and better support for **long-running, stateful workflows**. There is noticeable activity around integrating with **Jupyter** and **VS Code** for research and development loops.
*   **LangGraph (LangChain):** Adoption is growing for **cyclical, state-machine-like agent workflows**. The `StateGraph` abstraction is being used to model complex business processes. A notable pattern is its use alongside **LangSmith** for tracing and evaluation, creating a closed-loop development environment.

### 2. Emerging Hotspots & New Entrants
*   **Model Context Protocol (MCP):** Gaining rapid traction as a **standard for tool and data source integration**. Frameworks are adding MCP server/client support to decouple agent logic from resource access. This aligns with the fleet's "Shell" concept of bootstrapping external intelligence.
*   **Lightweight Runtimes:** Projects like **Axiom** and **Swarm** focus on **deterministic, debuggable agent execution** with minimal overhead, contrasting with heavier orchestration frameworks. This resonates with the fleet's interest in **cudaclaw** (GPU-resident runtime).
*   **Specialist Frameworks:** Niche frameworks for **code generation (Windsurf, Bloop), data analysis (PandasAI), and research** are becoming more agentic, often building on the core abstractions of the major frameworks.

### 3. Fleet-Relevant Pattern: The "Room" Analogy
The fleet's **PLATO room** concept (a training environment accumulating interaction tiles) finds parallels in the open-source world:
*   **Agent "Sandboxes":** Several projects now offer isolated environments where agents can execute code, with their actions and outcomes logged for later review and learning—a primitive form of tile generation.
*   **Evaluation-Driven Development:** The focus is shifting from building agents to **building evaluation suites**. The ability to score an agent's performance on a benchmark task is becoming a primary feature, similar to how ensigns are trained from room experience.

### 4. Specific Factual Observation (P0/P1 Mapping)
*   **P0 (Negative Space):** Monolithic, all-in-one agent frameworks that try to handle UI, memory, tools, and evaluation in a single codebase are showing signs of fragmentation. The community is **not** consolidating around one tool.
*   **P1 (Safe Channel):** The safe channel is **composability**. Successful projects provide clear, modular abstractions (Agent, Tool, Graph, State) that can be integrated with other best-in-class components (e.g., using LangGraph for orchestration, Pydantic for validation, MCP for tools, a separate system for evaluation).
*   **P2 (Optimization):** Within the composability channel, optimization is happening around **developer experience** (better debugging, tracing) and **runtime efficiency** (reducing latency, cost).

### 5. Recommendation for Fleet Context
The fleet's architecture (plato-torch rooms, holodeck-rust MUD, constraint-theory-core) is specialized and deep. The open-source trend toward **composable, evaluable, and observable agent systems** validates the fleet's directional bets. The primary gap in the open-source ecosystem that the fleet could address is the **formalized capture of agent interaction data ("tiles") into trainable model adapters ("ensigns")**—moving beyond logging toward structured, continuous learning.
