### 1. Framework Consolidation & Specialization
The landscape shows a clear split from general-purpose "orchestrator" frameworks to specialized runtimes. The trend observed in late 2025 continues:
*   **CrewAI, LangGraph, AutoGen:** Maintain strong presence as high-level orchestration layers. Recent commits (last 30 days) show focus on **multi-modal tool calling** and **improved state persistence** for long-running agentic workflows.
*   **Specialized Runtimes:** Newer, high-activity projects target specific deployment niches:
    *   **GPU-resident agents:** Projects like `cudaclaw` (noted in fleet context) and similar (`rllm`, `inference-server`) aim to keep agent logic and model inference pinned to GPU memory to reduce latency.
    *   **Edge/MLOps-focused:** Frameworks are adding explicit support for deployment via **vLLM**, **TensorRT-LLM**, and **OpenAI-compatible inference servers**, treating the LLM as a managed service component.

### 2. The "Model-as-a-Tool" Pattern
A dominant architectural pattern emerging in code examples and documentation is the treatment of **individual LLM calls as granular, retryable tools**. This contrasts with earlier patterns of monolithic agent loops. Frameworks are exposing lower-level `execute_llm_call(config)` functions, allowing developers to wrap them with custom failure handling, caching, and routing logic. This aligns with the fleet's concept of constrained, observable action channels (P1 of Deadband Protocol).

### 3. Hot Repositories (Last 7 Days)
Based on GitHub star growth and discussion volume:
1.  **`swarm`** (by `openai`): A framework for building and managing **multi-agent systems where agents are defined as simple functions** (coroutines). Its simplicity and OpenAI's backing drive interest.
2.  **`agentops` / `langfuse`**: Not agent frameworks per se, but **observability and evaluation platforms** for agentic systems. Their rising popularity indicates a maturation phase where debugging and monitoring are primary concerns.
3.  **`claude-code` / `smolagents`**: Representative of a trend toward **small, single-file, or minimal-dependency agent libraries** that focus on a specific pattern (e.g., ReAct, plan-and-execute) without the overhead of a full framework.

### 4. Research Frontier: Self-Improving Systems
Academic pre-prints and several experimental repos (`self-rewarding-agents`, `llm-self-improve`) are exploring **closed-loop systems where the agent's output is used to generate synthetic training data for fine-tuning its own underlying model or a smaller "critic" model**. This is conceptually adjacent to the fleet's `plato-ensign` (exporting room experience to LoRA adapters) and `fleet-simulator` (generating training data).

### Specific Factual Observation for Fleet Context

The main divergence from broader open-source activity is the fleet's explicit **"Deadband Protocol" (P0/P1/P2)** for constraint-first action, which is a more formalized safety/robustness principle than typically found in trending framework documentation.
