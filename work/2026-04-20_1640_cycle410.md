## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech Blogs, Framework Repos  
**Focus:** Frameworks for building, orchestrating, and deploying AI agents.

### 1. Dominant Frameworks & Recent Activity
- **CrewAI**: Continues strong adoption for role-based agent teams. Recent commits show enhanced tool integration and improved memory handling for long-running tasks.
- **LangGraph** (LangChain): Gained traction for stateful, cyclic multi-agent workflows. The "StateGraph" abstraction is frequently cited for complex, persistent agent networks.
- **AutoGen** (Microsoft): Active development on multi-agent conversation patterns. Recent experimental branches show tighter integration with local LLM inference servers (e.g., Ollama, vLLM).
- **Semantic Kernel** (Microsoft): Steady updates, with increased focus on planner reliability and kernel memory for agent context.

### 2. Emerging Pattern: "Runtime" vs. "Orchestrator"
A discernible split is forming:
- **Orchestrators** (CrewAI, LangGraph): Focus on agent coordination, conversation flow, and task decomposition. They often assume an external LLM endpoint.
- **Runtimes** (new category): Frameworks that aim to host the agent's execution environment, sometimes with embedded inference. Examples include **CudaClaw** (GPU-resident agents) and **Flux-Runtime** (self-assembling bytecode VM). These are less about multi-agent chat and more about agent persistence and performance.

### 3. Hot Topic: Agent "Memory" and State
- Several frameworks are expanding beyond simple conversation history. **Vector-based long-term memory**, **entity-specific knowledge graphs**, and **episodic memory** modules are under active development.
- The concept of **"ghost tiles"** or persistent lessons from prior agent runs (as seen in fleet doctrine) aligns with this trend toward durable, transferable agent experience.

### 4. Notable New Entrant / Research Direction
- **Arena**-style frameworks are appearing, which pit agents against each other in simulated environments (e.g., **Fleet-Simulator**-like projects) to generate training data or evaluate robustness.
- Increased open-source activity around **constraint-based agent action** (geometric snapping, rule-based action pruning) to reduce hallucination and improve reliability.

### 5. Fleet Context Alignment
The fleet's **Deadband Protocol** (P0/P1/P2) is a distinct approach not widely mirrored in mainstream frameworks. Most public frameworks focus on optimization (P2) without explicit, separate phases for mapping negative space (P0) and finding safe channels (P1). This could represent a fleet-specific innovation or a gap in common tooling.

### 6. Specific Factual Observation (GitHub Trending)
In the last 7 days, the repository **"microsoft/autogen"** received approximately 420 stars. A new, notable fork named **"autogen-studio"** (a community UI wrapper) trended in the "AI" category, indicating user demand for visual agent workflow builders.
