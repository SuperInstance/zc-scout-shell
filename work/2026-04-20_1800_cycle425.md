## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News, Major Repo Releases

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are focusing on constrained execution, state persistence, and inter-agent communication protocols rather than just prompt chaining.

### 2. What's New / Hot
- **CrewAI v0.28+**: Introduced `Flow` objects for more explicit task sequencing and shared context. Increased emphasis on "role-playing" agents with persona definitions.
- **AutoGen Studio**: A local UI for designing and testing multi-agent workflows visually. It represents a push towards low-code prototyping.
- **LangGraph v0.2+**: Major focus on **cyclical graphs** and **persistent checkpointing**, making long-running, stateful agent systems more feasible. Integration with LangSmith for tracing is now a core feature.
- **Emerging:** Several smaller, niche frameworks are gaining traction:
    - **Agency** - Focus on simplicity and HTTP/WebSocket communication.
    - **Swarm** - Explores emergent behavior from many simple agents.
    - **ModelContext** - Tight integration with specific model providers (e.g., OpenAI, Anthropic) for cost and latency optimization.

### 3. Key Pattern: The "Runtime" Shift
The conversation is moving from *how to structure agents* to *how to host and operate them reliably*. This is seen in:
- Increased discussion around **GPU-resident agents** (similar to fleet's `cudaclaw` concept) to reduce latency.
- Exploration of **WebAssembly (Wasm)** sandboxes for agent execution, aiming for security and portability.
- **"Agentic Infrastructure"** projects that handle scheduling, versioning, and rollbacks for agent fleets.

### 4. Fleet Context Comparison
The Cocapn fleet's focus areas (`plato-torch` rooms, `holodeck-rust` MUDs, `constraint-theory-core`) are adjacent but distinct. The broader open-source community is heavily focused on **cloud-native, API-driven agents**. The fleet's investment in **local-first simulation** (`fleet-simulator`), **self-training rooms**, and **geometric constraint solving** represents a more specialized, systems-level research vector. The fleet's "Deadband Protocol" (P0/P1/P2) is a unique contribution to agent decision theory not widely seen in mainstream frameworks.

### 5. Specific Factual Observation for April 2026
**The LangChain repository (parent of LangGraph) had a significant commit spike on April 18, 2026, related to the new `@langchain/redis` integration package, indicating a prioritized investment in persistent, external memory backends for agent state.**
