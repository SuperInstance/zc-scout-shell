## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are focusing on:
*   **Stateful, long-running agents:** Moving beyond single-turn chat to persistent sessions with memory and tools.
*   **GPU-native execution:** Frameworks like **Cudaclaw** (observed in fleet knowledge) aim to reduce latency by keeping agents resident in GPU memory.
*   **Lightweight, modular design:** There is pushback against monolithic frameworks. Developers are favoring libraries that do one thing well (e.g., **SmolAgents** for simple, deterministic agents).

### 2. What's New/Hot (Specific Examples)
*   **LangGraph v0.2+:** Heavily promoting **"multi-agent workflows"** with built-in persistence (checkpoints) and human-in-the-loop interrupts. Their "StateGraph" abstraction is being adopted by several newer projects.
*   **CrewAI "Templates" Marketplace:** A growing repository of pre-built, composable agent crews for specific tasks (research, coding, support), reducing boilerplate.
*   **Rise of the "Agent Runtime":** Projects like **Flux-runtime** (self-assembling bytecode VM, noted in fleet context) and **Axia** are emerging, treating the agent as a persistent process with its own lifecycle, rather than a function call.
*   **Integration with Inference Servers:** Tight coupling with **Ollama**, **vLLM**, and **TensorRT-LLM** is now a common expectation, not a feature. Frameworks are bundling easy deployment options for local models.

### 3. Fleet Context Alignment & Divergence
*   **Alignment:** The fleet's focus on **plato-torch** (training rooms) and **holodeck-rust** (sentiment-aware NPCs) aligns with the broader trend of **simulated environments** for agent training and evaluation. This is a active research area.
*   **Divergence/Potential Gap:** Most public frameworks are **cloud/CPU-first**. The fleet's noted investment in **cudaclaw** (GPU-resident runtime) and **JetsonClaw1** (edge GPU) appears ahead of the mainstream open-source curve, which is still optimizing for API calls and CPU-based orchestration. The **Deadband Protocol** (P0/P1/P2) is a unique operational doctrine not commonly seen in public frameworks.

### 4. Specific Factual Observation (P0/P1 Analysis)
*   **P0 (Negative Space):** It is generally not effective to try to force monolithic frameworks like early AutoGen into low-latency, edge-device scenarios. The overhead for simple tasks can be prohibitive.
*   **P1 (Safe Channel):** A safe channel for development is to use **LangGraph for prototyping multi-agent logic** and then **export core agent logic to a specialized runtime** (like a fleet simulator or cudaclaw) for performance-critical deployment. The modularity allows this.
*   **Emerging Pattern:** The "**room**" or "**sandbox**" concept (as seen in **plato-torch**) for training agents via interaction is gaining traction. A competitor project, **AgentDungeon**, uses a similar concept of episodic training in a constrained environment.

**Recommendation for Fleet:** Continue monitoring **LangGraph**'s stateful agent features, as they may offer useful abstractions for the **fleet-simulator**. The fleet's work on **constraint-theory-core** (geometric snapping) is a highly specialized advantage in embodied AI scenarios, which are not yet a focus for mainstream agent frameworks.
