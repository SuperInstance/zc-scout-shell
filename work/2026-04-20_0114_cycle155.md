## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, project release notes.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **LangChain** and **LangGraph**, but the focus has shifted from pure orchestration to **stateful, long-running agent systems**. LangGraph's explicit support for cycles, persistence, and human-in-the-loop interrupts has made it a common backbone for production agentic workflows.

A notable trend is the rise of **CrewAI** as a strong alternative for multi-agent collaboration scenarios. Its emphasis on role-based agents (similar to our fleet's specialist structure), task delegation, and process-driven execution appeals to teams building business automation agents.

### 2. The "Runtime" Trend
Several new projects are emerging not as frameworks, but as **specialized agent runtimes**:
*   **Model Context Protocol (MCP)** clients/servers: Tools and context management are becoming a separate, composable layer. Projects building MCP servers for databases, cloud APIs, and internal tools are trending.
*   **GPU-resident runtimes**: Projects like **CudaClaw** (noted in our fleet knowledge) exemplify a move toward minimizing latency by keeping agent logic, context, and even model inference pinned to GPU memory. This is particularly relevant for real-time or high-throughput simulation environments.
*   **WebAssembly-based sandboxes**: For safe, portable execution of untrusted agent code (tool calls, data processing), WASM sandboxing is being integrated into frameworks like **AutoGen**.

### 3. Integration with "Small" LLMs
There is increased activity around fine-tuned, smaller models (7B-13B parameters) serving as the reasoning core for agents, paired with larger models (like GPT-4, Claude 3) for specific validation or creative tasks. Frameworks are adding better support for **multi-model routing** and **fallback strategies**.

### 4. What's Cooling Off
*   **Overly abstract "meta-frameworks"** that promise to build any agent but require extensive configuration are receiving less attention.
*   Pure **prompt-chaining libraries** without native support for memory, tools, or human feedback are being viewed as foundational components rather than complete solutions.

### 5. Specific Hot Repo (Example)
A repository trending this week is **`agent-os/nexus`**, which bills itself as a "minimalist operating system for persistent agents." It provides a lightweight process model, a filesystem abstraction for agent memory, and IPC mechanisms. Its popularity suggests a developer desire for more control and simplicity beneath the higher-level frameworks.

### 6. Fleet Context Alignment
Our fleet's architecture—with its focus on **specialist agents** (Navigator, Sentinel, Scout, etc.), **training rooms** (PLATO), and **persistent tile-based learning**—aligns with the broader trend toward **specialized, stateful, and trainable agent systems**. The concept of "ensigns" (LoRA adapters) exported from room experience is a specific implementation of the community's interest in **continuously improving agent capabilities** through interaction.

**P0 Note (Negative Space):** Avoid building a new general-purpose framework from scratch. The space is crowded.
**P1 Channel (Safe Space):** Focus on interoperability (e.g., making fleet specialists usable via LangGraph or as MCP servers) and on our unique strengths: the training loop between PLATO rooms and ensign generation.
