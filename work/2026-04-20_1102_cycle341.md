## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **LangChain** and **LangGraph** for orchestration, but a significant shift toward **lightweight, specialized runtimes** is observable. **CrewAI** maintains traction for multi-agent role-based workflows, particularly in enterprise prototyping. A notable trend is the emergence of frameworks designed for **GPU-resident agents** and **constrained edge environments**, which aligns with fleet interests in `cudaclaw` and edge operations (JetsonClaw1).

### 2. Emerging Hot Topics
- **Agentic Frameworks as Compilers:** Projects like **Microsoft's AutoGen Studio** and research around **"agent compilers"** (e.g., DSPy-inspired approaches for agents) are gaining attention. The focus is on converting high-level agent specs into optimized, executable workflows with minimal prompt engineering.
- **Rust-Based Runtimes:** There is growing activity in Rust-implemented agent runtimes focusing on safety, performance, and WebAssembly compatibility. This correlates with fleet projects like `holodeck-rust` and `constraint-theory-core`.
- **Open-Source "Agent OS" Experiments:** Several GitHub trending repos are exploring minimalistic, single-file agent environments that bundle model inference, tool calling, and state management into a standalone binary—often targeting CLI and embedded use.

### 3. Specific Project Activity (Last 30 Days)
- **LangGraph** released enhanced support for **persistent checkpointing** and **human-in-the-loop interrupts**, which some developers are using to implement P0/P1/P2-style deadband protocols.
- **CrewAI** saw a spike in forks implementing **domain-specific agent "rooms"** (e.g., for finance, legal), mirroring the PLATO room concept in `plato-torch`.
- A new project, **AgentKit** (rust), trended for two days, offering a no-frills, event-loop-based runtime for research agents. It explicitly avoids LLM-based planning, focusing instead on deterministic action selection.

### 4. Fleet-Relevant Insight
The trend toward **specialization and compartmentalization** (rooms, specialized runtimes) validates the fleet's architectural direction. The community's exploration of **persistent state and interruptible workflows** aligns with the Deadband Protocol's phased approach. However, the fleet's integration of **training data generation directly from agent interactions** (tiles → rooms → ensigns) appears less common in mainstream open-source frameworks, which often treat agents as transient task solvers.

### 5. Negative Space (P0 Note)
Avoid frameworks that assume continuous, high-bandwidth cloud connectivity or that lack state serialization capabilities. These are unsuitable for edge deployment and persistent room training.
