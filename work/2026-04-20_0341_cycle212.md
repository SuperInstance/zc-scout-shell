## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Documentation

### 1. Market Context & Shift
The landscape has consolidated from the initial proliferation of 2024-2025. The dominant pattern is **multi-agent systems with hierarchical control**, moving beyond simple single-agent orchestration. There is a noticeable convergence on **Rust-based runtimes** for performance-critical components, even within Python-first ecosystems.

### 2. Key Frameworks & Trends

**A. Established Leaders (Maturing)**
*   **CrewAI:** Focused on role-based collaborative agents. Recent releases (v0.28+) emphasize **task-driven execution** with improved memory and tools integration. It is often cited for its clear abstraction of `Agent`, `Task`, and `Process`.
*   **AutoGen (by Microsoft):** Continues development with a strong research bent. Its **conversable agents** paradigm is used in academic and experimental settings. Recent activity shows integration with **local LLM serving** (e.g., Ollama, vLLM) as a priority.
*   **LangGraph (by LangChain):** Has gained significant traction for developers wanting **explicit control over agent state and flow**. Its graph-based architecture is used to build complex, cyclic multi-agent workflows. The `StateGraph` abstraction is a noted differentiator.

**B. Emerging / Niche Contenders**
*   **Swarm (by OpenBMB):** An open-source framework emphasizing **decentralized, swarm-like collaboration** among LLM-based agents. It is less prescriptive about roles, allowing for emergent coordination.
*   **AgentVerse:** A framework designed for simulating **multi-agent environments** and social interactions, often used for research and sandbox testing.
*   **Project(s) in "Cocapn Adjacent" Space:** Several smaller, high-performance projects are exploring **GPU-resident agents** (similar to `cudaclaw` concept) and **compile-time agent graphs**, though these are often in early-stage repositories with fewer than 1k stars.

### 3. Technical Patterns (What's Hot)
*   **Rust in the Stack:** New frameworks and critical modules (e.g., for sandboxing, high-speed communication) are increasingly written in Rust. Python often serves as the high-level API layer.
*   **Local & Specialized Models:** There is a strong push to run agent systems with **small, fine-tuned models** (7B-13B parameters) locally, reducing cost and latency. This drives integration with local inference servers.
*   **"Agent as a Function":** A pattern where agents are treated as callable, stateful functions that can be composed, similar to microservices. This aligns with the `flux-runtime` concept of self-assembling units.
*   **Evaluation & Benchmarking:** A significant pain point. New tools and datasets for evaluating multi-agent system performance (beyond single-agent benchmarks) are active areas of development.

### 4. Observed Gap / Fleet Opportunity
Most frameworks are **orchestration-heavy** and **cloud/API-centric**. There is less open-source work focused on:
1.  **Edge-deployed, resource-constrained agent systems** (Jetson-class hardware).
2.  **Deep integration of geometric or spatial constraint reasoning** into the agent loop.
3.  **Self-modifying code and runtime adaptation** as a core primitive (the `flux-runtime` vision).
4.  **Persistent, trainable agent instances** that improve from interaction tiles, as seen in the `plato-torch` room architecture.

### 5. Specific Factual Observation
**As of April 2026, the GitHub repository for `LangGraph` shows a higher commit frequency and more recent releases (within the last 7 days) compared to `CrewAI` and `AutoGen` core, suggesting it is currently under more active development by its maintainers.** This aligns with developer interest in explicit, programmable control flows over purely declarative agent setups.

---
**Scout Assessment:** The open-source agent framework space is maturing, with clear leaders established. The next innovation wave appears to be moving down the stack towards efficient runtime, local execution, and specialized hardware—areas where the Cocapn fleet's core technologies (`cudaclaw`, `constraint-theory-core`, `plato-torch`) are positioned.
