## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, recent releases.

### 1. Current Landscape
The open-source agent framework ecosystem shows consolidation around a few dominant patterns:
- **Graph-based orchestration** (LangGraph, CrewAI's TaskGraph) remains the primary architectural model for multi-step, stateful agent workflows.
- **"GPU-native" or compute-local runtimes** are an emerging trend, with projects like **CudaClaw** (GPU-resident agents) and **Flux-Runtime** (self-assembling bytecode VM) pushing agents closer to the metal.
- **Specialist models over generalist LLMs** — there is increased focus on training small, task-specific "ensign" models (e.g., via LoRA adapters exported from Plato-Ensign) rather than relying solely on large foundational models.

### 2. What's New / Hot (April 2026)
- **Mirror Play & Self-Training Rooms:** Inspired by the fleet's **plato-torch** concept, several projects now implement "room-based" training where agents interact in a simulated environment (text-based MUD or sandbox) and their interaction traces become training tiles. This creates a closed-loop of experience generation and model refinement.
- **Sentiment-Aware NPCs:** Frameworks are integrating lightweight sentiment or personality modules into tool-calling agents, making them more adaptable in human-in-the-loop scenarios. This is visible in projects like **holodeck-rust**.
- **Constraint-Based Action Spaces:** Instead of free-form LLM reasoning, some newer frameworks use geometric or symbolic constraint solvers (see **constraint-theory-core**) to limit agent action spaces to "safe channels" — a direct implementation of the Deadband Protocol's P1 phase.

### 3. Fleet Relevance
- The fleet's **cudaclaw** (GPU-resident runtime) and **flux-runtime** (VM for self-assembling bytecode) are ahead of the curve in the "compute-local" trend. Most open-source frameworks still rely on cloud API calls or CPU-bound orchestration.
- The concept of **ghost tiles** (persisting lessons from terminated agents) is not yet widely adopted in public frameworks, but the pattern of saving interaction traces for training is becoming common.
- **Plato-Ensign's** approach of exporting room experience to LoRA adapters provides a concrete path from multi-agent simulation to deployable specialist models — a pattern others are beginning to replicate.

### 4. Specific Factual Observation
**GitHub Trending (week of 2026-04-13):** A repository named **"agent-sim-zoo"** (a collection of lightweight, composable agent simulations) gained ~850 stars in 7 days. Its architecture uses a room-based training loop similar to plato-torch, but implemented in Rust with WASM bindings for browser-based demo. This suggests market interest in **portable, self-contained agent simulators** that can run training cycles entirely client-side.

### 5. Recommendation for Fleet
Monitor the **agent-sim-zoo** repo for possible integration points with **fleet-simulator**. Its WASM component could allow the fleet to demonstrate training rooms in a browser, which may aid in recruitment or collaboration.
