## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, framework release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, a clear trend is the shift from pure "orchestration" to **specialized runtime layers**.

*   **CrewAI (v0.28+):** Focused on improving agent "crew" stability and adding more built-in tools for web search and data analysis. Recent commits show work on a long-term memory module using vector databases.
*   **AutoGen (Microsoft):** Development appears to have slowed in public commits, with the community focusing on stable deployments rather than major new features. The "Conversable Agent" pattern remains widely referenced.
*   **LangGraph (LangChain):** Has become the *de facto* standard for building stateful, multi-agent workflows due to its graph-based paradigm. Its integration with LangChain's tool ecosystem gives it a strong network effect.

### 2. Emerging Pattern: The "GPU-Resident" or "Compiled" Agent
The most notable trend is the exploration of agents that minimize Python interpreter overhead and live closer to the hardware, aligning with fleet technologies like **cudaclaw**.

*   **Example - `microsoft/agents` (Research):** Microsoft Research published a prototype for "Compiled Agents," using MLIR to compile agent logic (planning, tool use) into a more efficient, predictable executable. This is a research project, not a production framework.
*   **Implication:** The community is recognizing latency and cost bottlenecks in interpreter-heavy, LLM-loop architectures. The search is on for paradigms that enable faster, cheaper, and more deterministic agent cycles.

### 3. New Entrant / Notable Project: **Axiom**
A new framework, **Axiom**, has gained attention on GitHub trending in the last month. Its stated goal is "agents that reason with code first, LLMs second." It emphasizes:
*   A strong, Rust-based core for reliability.
*   A "planning-by-compilation" approach where agent tasks are decomposed into dependency graphs of pure functions where possible, only calling an LLM for ambiguous nodes.
*   Early benchmarks claim significant reductions in LLM token usage for deterministic sub-tasks.

**Scout's Assessment:** Axiom represents a concrete implementation of the "compiled agent" trend. Its philosophy of minimizing LLM calls for predictable work mirrors the fleet's **Deadband Protocol** (P0/P1/P2) in spirit, seeking efficient channels (P1) before optimization (P2). It is a project to monitor.

### 4. Fleet Contextual Observation
The fleet's architectural choices (**cudaclaw**, **flux-runtime**) are positioned ahead of the current mainstream open-source curve. While popular frameworks are grappling with Python/GIL limitations, the fleet is already investing in the next paradigm: agents with dedicated, efficient runtime environments.
