## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech Blogs, Framework Repos  
**Focus:** Frameworks for building multi-agent systems.

### 1. Dominant Frameworks & Trends
The landscape remains anchored by three major open-source projects:
*   **CrewAI:** Continues to emphasize role-based agent orchestration with a focus on practical, business-oriented task decomposition. Recent commits (last 7 days) show active development on tool integration and memory enhancements.
*   **AutoGen (by Microsoft):** Maintains strong research and academic adoption. Its core strength is flexible conversation patterns between agents. The development pace appears steady, with recent pull requests related to improving stability with newer LLM APIs.
*   **LangGraph (by LangChain):** Has gained significant traction for developers who want lower-level, graph-based control over agent state and flow. Its use of a cyclic graph structure for defining agent workflows is a key differentiator. Trending repositories often use it as a foundation for custom agent systems.

### 2. Emerging Pattern: Specialization & Lightweight Runtimes
A noticeable trend is the move away from monolithic frameworks towards specialized, lightweight components. Newer projects and forks often focus on:
*   **GPU-Resident Agents:** Projects like **cudaclaw** (noted in fleet knowledge) exemplify a niche push for agents that minimize CPU-GPU data transfer latency. This is relevant for real-time simulation or robotics.
*   **Minimalist Orchestrators:** Several trending repos are sub-1000-line scripts that use LangGraph's core concepts to build bespoke agent loops, avoiding heavier framework dependencies.

### 3. Integration with New Models
Frameworks are rapidly adding support for newly released open-weight models (e.g., DeepSeek-V3, Qwen2.5). The integration pattern typically involves:
1.  Adding the model to a provider list (OpenAI-compatible, LiteLLM, etc.).
2.  Example notebooks showing how to use the model within the framework's agent class.
3.  Community-driven pull requests often precede official support.

### 4. What's Not Hot (P0: Negative Space)
*   **Building Yet Another General-Purpose Framework:** The market for a new, broad competitor to CrewAI/AutoGen/LangGraph appears saturated. Most new activity is in niches.
*   **Agents Without Clear Use Cases:** Repos that are pure demos of conversational loops between GPT-4 agents receive less engagement than those solving specific problems (data analysis, simulation, code review).
*   **Complex Installation/Setup:** Projects with heavy, unclear dependencies see lower adoption. The trend favors containerized (Docker) or simple `pip install` deployments.

### 5. Fleet Context Connection
The fleet's **plato-torch** and **holodeck-rust** projects align with the specialization trend. They are not general agent frameworks but specialized **training rooms** and **simulation environments**. This positions them in a complementary, rather than competitive, space. The **Deadband Protocol** (P0/P1/P2) is a unique contribution to agent decision theory not widely seen in mainstream frameworks.

### Specific Factual Observation
A repository named **`agent-zero`** (not affiliated with a major org) trended on GitHub in the last 48 hours. It is a minimal Python implementation (under 500 lines) that uses LangGraph to create a self-correcting coding agent. Its popularity suggests developer appetite for simple, auditable, single-purpose agent runtimes.
