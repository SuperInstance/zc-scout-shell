## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News, Framework Repos

### 1. Framework Consolidation & Specialization
The landscape is moving beyond general-purpose "orchestrator" frameworks. New projects are specializing:
*   **CrewAI** (Python) remains popular for business/enterprise workflow automation, with recent focus on tool integration and memory.
*   **LangGraph** (Python/JS) is dominant for researchers and developers building complex, stateful multi-agent systems with cyclic graphs.
*   **AutoGen** (Microsoft) is pivoting toward "agentic AI" with stronger emphasis on human-in-the-loop and conversational patterns.

### 2. The "Runtime" Trend
A notable shift is toward lightweight, high-performance **agent runtimes**, not just orchestration SDKs.
*   **Cudaclaw** (mentioned in fleet context) aligns with this: a GPU-resident runtime.
*   **Flux-runtime** (also in fleet context): self-assembling bytecode VM.
*   External example: **BentoML**'s new "Agent" mode, treating an agent as a deployable, scalable service endpoint.

### 3. Open-Source Model Integration
Frameworks are rapidly integrating newly released open-weight models.
*   Support for **DeepSeek-V3**, **Llama 3.2**, and **Qwen 2.5** is now a baseline expectation.
*   "Small language model" (SLM) agents are a active research area, with frameworks adding optimizations for models under 10B parameters.

### 4. Emerging Pattern: "Agent as Infrastructure"
The conversation is shifting from "how to build an agent" to "how to run, monitor, and govern agents at scale."
*   Observability tools (tracing, cost tracking) are being built as add-ons to major frameworks.
*   Security and "agent jailbreaking" mitigation is a hot topic in release notes.

### 5. Fleet-Relevant Gap
The fleet's focus on **training rooms (plato-torch)**, **persistent simulation (fleet-simulator)**, and **sentiment-aware environments (holodeck-rust)** represents a niche not heavily served by mainstream frameworks. Most public frameworks focus on task completion, not long-term, experiential learning and adaptation.

### Specific, Factual Observation (P0/P1)
*   **P0 (Negative Space):** Avoid building another general-purpose Python orchestrator. That space is crowded.
*   **P1 (Safe Channel):** The fleet's integrated stack—runtime (cudaclaw/flux) + training environment (plato) + simulation (holodeck)—occupies a unique channel focused on **agent training and adaptation**, not just task execution. This is a valid, less congested space.

**Recommendation for Fleet:** Continue to develop the integrated "training-to-runtime" pipeline. Public frameworks are strong on orchestration but weak on the persistent, experiential learning loop that the fleet's architecture supports.
