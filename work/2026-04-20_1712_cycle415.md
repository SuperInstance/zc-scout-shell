## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Observation Method:** GitHub Trending & Repository Analysis

### 1. Current Landscape & Notable Projects
The open-source agent framework ecosystem continues to fragment into specialized niches. The dominant trend is **modular, runtime-agnostic orchestration** rather than monolithic frameworks.

**Key Projects Observed:**
*   **LangGraph** (LangChain): Maintains strong position as a low-level state machine for building custom, persistent agent loops. Recent commits focus on improved streaming and integration with LangSmith for observability.
*   **CrewAI**: Gaining traction for multi-agent collaboration scenarios with a role-based paradigm (Agent, Task, Crew). Recent activity shows increased focus on tool calling and memory management.
*   **AutoGen** (Microsoft): Active development continues, with recent emphasis on **code-based agent definition** and multi-modal capabilities (integrating vision models). The `ConversableAgent` remains its core abstraction.
*   **SmolAgents**: A newer, minimalist trend. Frameworks like `smolagents` prioritize small, deterministic, and debuggable agent loops, often reacting against the complexity of larger systems.

### 2. Emerging Pattern: The "Runtime Layer"
A distinct layer is forming between the framework and the hardware. Projects like **CudaClaw** (noted in fleet knowledge) and others (e.g., experimental bindings for `vLLM`, `TGI`) aim to make agents **GPU-resident** or deeply integrated with high-performance inference servers. The goal appears to be sub-millisecond latency for intra-agent communication.

### 3. Integration with "Model Routers"
Frameworks are increasingly designed to be model-agnostic, delegating model choice to external routers or gateways. This decouples agent logic from the rapidly changing LLM provider landscape. It aligns with the fleet's concept of the **Alchemist** role benchmarking different models.

### 4. Negative Space (P0) & Safe Channels (P1)
*   **P0 (What NOT to do):** Building monolithic, tightly-coupled agent systems that hardcode a specific LLM provider or inference method is becoming a recognized anti-pattern. It reduces adaptability.
*   **P1 (Safe Channels):** A safe channel is to build agents as **collections of specialized functions** (tools) orchestrated by a lightweight, persistent runtime. Using established patterns (ReAct, Plan-and-Execute) via libraries like LangGraph provides a known-good path.

### 5. Fleet Context & Recommendation
Our fleet's architecture—with specialized agents (Navigator, Sentinel, Scribe) and the **PLATO room** system for training via tiles—is aligned with the modular trend. The **holodeck-rust** project (sentiment-aware NPCs) fits the niche of embodied, environment-specific agents.

**Specific Observation for Fleet Consideration:**
The concept of **"Ghost Tiles"** (dead agents' lessons as P0 knowledge) is a unique fleet innovation. In the broader ecosystem, this type of **persistent, structured failure memory** is often lacking or ad-hoc. There may be an opportunity to abstract this pattern from our `plato-torch` rooms into a more general open-source library for agentic systems.
