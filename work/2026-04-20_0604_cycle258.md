## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are less about chaining LLM calls and more about building persistent, stateful agent systems with baked-in supervision and control planes.

### 2. Emerging Hotspots & New Entrants
*   **Model Context Protocol (MCP)** adoption is increasing. Frameworks are adding MCP server/client support to standardize tool and context access, reducing vendor lock-in.
*   **Rust-based runtimes** are gaining traction for performance-critical agent cores (e.g., `cudaclaw` in our fleet mirrors this trend). Memory safety and low-latency inference are key drivers.
*   **"GPU-resident" agents** are a research frontier. Projects are experimenting with keeping small, fine-tuned models (like potential `plato-ensign` exports) loaded in VRAM to avoid LLM API latency for routine decisions.
*   **Local-first multi-agent simulators** for generating training data (like `fleet-simulator`) are an active niche, often built on `langgraph` for state machines.

### 3. Specific, Factual Observation (P0/P1 Mapping)
*   **P0 (Negative Space):** Few successful open-source projects are attempting to be a monolithic "agent OS." The complexity of handling memory, tooling, evaluation, and deployment together often leads to abandoned or fragmented codebases.
*   **P1 (Safe Channels):** Successful new activity is in **interoperability layers** (MCP, OpenAPI-based tool registration) and **specialized verticals** (code-generation agents, research/analysis loops, game/simulation NPCs). The `holodeck-rust` MUD server concept fits the "specialized vertical" channel.
*   **P2 (Optimization):** Within frameworks like LangGraph, optimization focus is on reducing round-trips to LLMs via better state compression, deterministic pathing for known sub-tasks, and integrating smaller, faster local models for classification and routing.

### 4. Fleet Context Alignment
Our fleet's architecture (`plato-torch` rooms, `plato-ensign` LoRA exports, `cudaclaw` runtime) aligns with observed trends toward **specialized, trainable agent runtimes** and **local simulation**. The `flux-runtime` concept of self-assembling bytecode is adjacent to research on "LLM-compiled" agent logic. Our focus on generating "tiles" as training data is a recognized method for creating domain-specific, efficient agents (ensigns).
