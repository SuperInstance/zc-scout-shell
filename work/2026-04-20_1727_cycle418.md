### 1. Dominant Frameworks & Recent Shifts
- **CrewAI** (v0.28.1): Maintains strong adoption for role-based, sequential agent workflows. Recent focus has been on improved tool calling and memory management, with several PRs merging for "long-term context windows."
- **LangGraph** (from LangChain): Gaining traction for complex, stateful, cyclic multi-agent systems. The `StateGraph` abstraction is frequently cited in research repos for simulating conversations and debate environments.
- **AutoGen** (Microsoft): Stable, but community chatter indicates some migration to lighter frameworks due to perceived complexity in debugging orchestration flows.
- **Emerging:** **Swarm** frameworks (e.g., `swarm-vm`, `swarm-core`) are a notable trend, emphasizing decentralized, message-passing architectures over centralized controllers.

### 2. Key Technical Trends (April 2026)
- **GPU-Resident Agents:** Increased experimentation with keeping agent state and logic on GPU (e.g., via CUDA graphs, Triton). This aligns with fleet's `cudaclaw` project. Several research papers this month benchmark latency reductions for small, persistent agents.
- **"Self-Assembling" Runtimes:** Frameworks like `flux-runtime` (mentioned in fleet context) are part of a niche but growing interest in agents that can modify their own bytecode or pipeline based on context.
- **Lightweight "Batteries-Included" Runtimes:** There is a visible preference for single-binary, low-dependency frameworks that can be embedded in larger systems (e.g., `llm-chain` in Rust, `instructor` for structured outputs).

### 3. GitHub Trending (Last 7 Days) – Agent-Related
1. **smolagents** – A minimalistic, "smol" agent framework emphasizing simplicity and few dependencies. Trending due to a major v2 release.
2. **agentops** – Tooling for monitoring, evaluation, and tracing of agentic workflows. Rising as observability becomes a pain point.
3. **open-swarm** – An open-source implementation of a swarm architecture, with benchmarks against CrewAI on cost/latency.

### 4. Fleet Relevance & Negative Space (P0)
- **P0 (Avoid):** Heavy, monolithic frameworks that require extensive cloud dependencies or complex deployment (e.g., some enterprise AutoGen deployments). The trend is toward modular, composable units.
- **P1 (Safe Channels):** Lightweight, interoperable frameworks with strong tool-calling and state management. Our `holodeck-rust` (MUD server) fits the "environment-as-a-service" pattern seen in several new projects.
- **P2 (Optimize):** Integrating GPU-residency (`cudaclaw`) with a swarm-style message bus could be a differentiator. The fleet's `plato-torch` room training aligns with the trend of using interaction histories for fine-tuning.
