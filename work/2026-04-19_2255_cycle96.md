### 1. Current Open-Source Agent Framework Landscape
Based on monitoring GitHub trending and recent commits (pattern observed in fleet's `rabbit-trail-research/`), the dominant frameworks remain:
*   **CrewAI:** Focus on role-based agent orchestration. Recent activity shows heavy integration with external tools (SQL, web search) and a push for more deterministic workflow control.
*   **AutoGen:** Microsoft's framework, strong in multi-agent conversation patterns and code execution. Trending discussions involve scaling to larger agent swarms and cost optimization.
*   **LangGraph:** (Stateful) Gaining traction for building cyclic, stateful multi-agent workflows. Its explicit graph-based architecture is being used for complex, persistent agent networks.

**New/Notable Trend:** A noticeable shift from pure "chat" orchestration toward **"agent-as-a-function"** or **"agent runtime"** models. Projects like `cudaclaw` (GPU-resident runtime) and `flux-runtime` (self-assembling bytecode VM) in our own fleet align with this. The trend is about embedding agents into lower-level execution environments for speed and persistence, rather than treating them as external API calls.

The fleet's `plato-torch` and `plato-ensign` architecture (training rooms that export LoRA adapters) presents a **P1 safe channel** (Doctrine: find where you CAN be) in the current ecosystem.

**Analysis:** Most public frameworks are focused on *orchestrating* pre-trained agents. There is less visible open-source activity on the *specialized, continuous training* of small, focused agent models (`ensigns`) from interaction data (`tiles`), which is our fleet's core loop. This suggests our architectural path is in a less congested channel.
