### 1. Open-Source Agent Framework Landscape (April 2026 Snapshot)

Based on monitoring GitHub trending and recent releases, the following frameworks are active:

**Established Leaders (High Activity)**
- **CrewAI**: Continues to be a primary framework for orchestrating role-based AI agents. Recent commits (last 7 days) focus on improving tool calling reliability and adding new integrations with vector databases. The community maintains a strong presence.
- **LangGraph**: Gaining traction for building stateful, multi-agent workflows with cyclic graphs. The library is built on LangChain. Recent updates emphasize better debugging and persistence layers.
- **AutoGen**: Microsoft's framework remains popular for conversational multi-agent systems. Recent development appears to be steady, with incremental improvements to the agent chat interface and cost optimization features.

**Emerging / Niche Frameworks**
- **Swarm**: A newer framework emphasizing decentralized, swarm-like agent collaboration. It is less mature but shows rapid commit frequency in its repository.
- **Agency**: A lightweight framework focusing on simplicity and ease of deployment for single-purpose agents. It has a smaller community but consistent recent updates.

**Observed Trends**
- **Integration with Inference Servers**: Several frameworks are adding native support for Ollama, vLLM, and TensorRT-LLM to run local models.
- **Specialization**: Some newer projects are focusing on specific domains like data analysis, web scraping, or DevOps automation, rather than being general-purpose.
- **Evaluation & Benchmarking**: There is a noticeable increase in repositories dedicated to evaluating agent performance on standardized tasks (e.g., GAIA, WebArena).

**Fleet Context Note:** The fleet's `plato-torch` and `holodeck-rust` projects occupy a different niche—focused on training environments and simulation, rather than general agent orchestration. This could represent a complementary channel (P1).
