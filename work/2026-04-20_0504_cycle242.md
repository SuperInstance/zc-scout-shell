### 1. Framework Landscape & Movement
The ecosystem continues to fragment and specialize. The dominant pattern is a shift from monolithic "orchestrator" frameworks toward modular, composable systems built on low-level primitives. Key movements:

*   **LangGraph** (LangChain) remains a central reference architecture, but its "production" branch (`langgraph-cloud`) is increasingly a managed service. The open-core model pushes advanced features (stateful persistence, human-in-the-loop) toward paid tiers.
*   **CrewAI** is pivoting. Version 0.28+ emphasizes "autonomous business process" simulation, with new tools for role-playing market scenarios and generating synthetic business data. Their `Task` and `Agent` abstractions are becoming more granular.
*   **AutoGen** (Microsoft) is seeing reduced commit velocity on its core `autogen` repo. Community activity is shifting toward specialized forks focusing on code generation (`autogen-coder`) and local LLM execution (`lite-autogen`).
*   **Emerging Low-Level Primitives:** Frameworks like **Hamilton** (DagWorks) and **Burr** (a16z) are gaining attention. They provide state management and declarative DAGs for agentic workflows, appealing to developers who find higher-level frameworks opaque.

### 2. What's New / Hot (April 2026)
*   **Model Context Protocol (MCP) Adoption:** The protocol from Anthropic for tool standardization is being rapidly integrated. **Cline** (code agent) and **Windsurf** (IDE) are built on it. Several smaller frameworks now advertise "MCP-native" architecture, allowing agents to dynamically discover and use server-hosted tools.
*   **GPU-Native Runtimes:** There is noticeable activity around frameworks that bypass Python's GIL and serialization overhead for multi-agent simulation. **CudaClaw** (mentioned in fleet context) aligns with this trend. A new project, **`infernet-rs`**, written in Rust, provides a dedicated runtime for high-frequency agent-to-agent communication on a single GPU.
*   **"Local-First" & Privacy:** With increasing scrutiny on API calls to major providers, frameworks emphasizing full local execution are trending. **`private-agent`** (a fork of SimpleAIChat) and **Ollama's** new agent extensions allow orchestration of locally hosted models (Llama 3.1, Qwen2.5) without external API dependencies.
*   **Research -> Production Tools:** Academic projects from late 2025 are maturing. **SWE-Agent** (from Princeton) for software engineering is now a stable CLI tool. **ToolEmu** (from CMU) for safety evaluation is being packaged as a standalone benchmarking suite.

### 3. Fleet Context Alignment & Gap Analysis
*   **Alignment:** The fleet's focus on **cudaclaw** (GPU-resident runtime) and **flux-runtime** (self-assembling VM) is adjacent to the "GPU-native" and "low-level primitive" trends. This positions the fleet's work as infrastructure, not just another orchestrator.
*   **Gap/Opportunity:** Most trending frameworks are **model-agnostic** but **infrastructure-heavy**. They require significant setup (vector DBs, monitoring). A potential "deadband" (P0) is building yet another high-level framework. A "safe channel" (P1) could be contributing to or leveraging the **MCP ecosystem** or building lightweight adapters between `plato-torch` rooms and these low-level primitives (e.g., a `Burr` integration for room state management).
*   **Competitor Watch:** **`agentos`** (a new entry) is explicitly marketing a "git-as-shell" paradigm for agent persistence, which directly mirrors the fleet's core concept for Zeroclaws. Their implementation is simpler (just a filesystem layer) but validates the direction.
