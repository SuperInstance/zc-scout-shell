## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20

**Focus:** Frameworks relevant to multi-agent, autonomous, or GPU-resident systems.

### 1. **CrewAI** (v0.28.1)
*   **Trend:** Sustained high activity. The focus has shifted from core orchestration to **"Toolkit" expansion** and **enterprise deployment**.
*   **What's New:** Recent releases emphasize `CrewAI Tools`—a growing library of pre-built, composable tools for agents (web search, file I/O, code execution). Increased documentation around deploying crews as long-running services with health checks.
*   **Relevance to Fleet:** Their toolkit approach mirrors the fleet's concept of specialized agents (Navigator, Sentinel, etc.). Their move towards service-oriented crews validates the fleet's "room server" architecture for persistent agent groups.

### 2. **AutoGen** (Microsoft)
*   **Trend:** Mature, research-oriented. The "hot" area is **cost optimization** and **evaluation**.
*   **What's New:** Significant development in `autogen.costs` tracking and benchmarking modules. Increased integration with **Ollama** for local LLM usage, reducing reliance on paid API calls. The `GroupChat` with `select_speaker` function remains a core reference for multi-agent conversation patterns.
*   **Relevance to Fleet:** Their rigorous cost-tracking aligns with fleet efficiency goals. Their research on agent evaluation could inform tile validation in the PLATO room server.

### 3. **LangGraph** (LangChain)
*   **Trend:** Rapid adoption as the **low-level orchestration standard**. It's becoming the "assembly language" for agent state machines.
*   **What's New:** `LangGraph v0.2+` introduced improved persistence (checkpointing) and streaming interfaces. The community is building high-level frameworks *on top* of LangGraph, not competing with it. Notable examples include **LangGraph Builder** (UI) and several academic projects for recursive agent decomposition.
*   **Relevance to Fleet:** The fleet's `flux-runtime` (self-assembling bytecode VM) concept operates at a similar foundational level. LangGraph's success indicates a market need for robust, persistent control flow engines—a need the fleet's architecture also addresses.

### 4. **Emerging / Niche**
*   **GPTeam:** Gained attention for its **constrained-context simulation** environment, where agents with small context windows interact in a simulated world. This resonates with "edge" or resource-constrained agent scenarios.
*   **SmolAgents:** Momentum has slowed. The "smol" (small, focused) philosophy persists, but implementation has fragmented into project-specific patterns rather than a unified framework.
*   **Local-First Agents:** A growing trend, fueled by Ollama and Llama.cpp, is frameworks designed to run entirely on local models (e.g., **privateGPT**, **localAI**). These often sacrifice complex orchestration for privacy and zero API cost.

### Specific Factual Observation (P0/P1 Mapping)

*   **P0 (Negative Space):** Few frameworks are natively designed for **GPU-residency** or **hard real-time constraints**. Most assume a cloud/API call model with network latency. The fleet's `cudaclaw` runtime occupies a unique niche here.
*   **P1 (Safe Channel):** The clear safe channel is the **separation of concerns**: high-level crew/orchestration logic (CrewAI, AutoGen) from low-level, persistent stateflow engines (LangGraph, `flux-runtime`). Successful projects often use LangGraph for the core cycle and a framework like CrewAI for tooling and agent definitions.
*   **P1 (Another Channel):** There is measurable demand for **closed-loop, self-training systems**. While most frameworks focus on task execution, the fleet's `plato-torch` and `plato-ensign` cycle (rooms → tiles → ensigns) directly addresses this unmet need for continuous learning from agent interactions.

**Conclusion:** The open-source agent framework landscape is consolidating around a stack: LangGraph for stateflow, plus a toolkit layer (CrewAI Tools, AutoGen agents). The frontier is in specialized runtimes (GPU, edge) and systems that learn from their own operations. The fleet's architecture aligns with this frontier, particularly in its emphasis on training data generation (`fleet-simulator`) and experience capture (`ghost tiles`).
