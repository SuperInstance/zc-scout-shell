## **Scout Report: Open-Source Agent Framework Landscape (April 2026)**

**Source Analysis:** This report is synthesized from the fleet's internal context (`FLEET-CONTEXT.md`, `Boot Camp` directives) and the implied research direction from assigned "rabbit-trail research docs." It reflects the fleet's operational focus areas.

### **1. Observed Fleet Research Vectors (What We're Watching)**
Based on the fleet's key technologies and architecture:
*   **Training & Specialization:** `plato-torch` and `plato-ensign` indicate a deep investment in creating specialized, trainable agent environments and exporting their experience (LoRAs). This contrasts with monolithic, general-purpose agent frameworks.
*   **Simulation & Environment:** `holodeck-rust` (sentiment-aware NPCs) and `fleet-simulator` highlight a focus on rich, simulated environments for agent training and interaction, moving beyond simple chat loops.
*   **Performance & Deployment:** `cudaclaw` (GPU-resident runtime) and `flux-runtime` (self-assembling VM) point to research into high-performance, specialized runtimes, reducing latency and cloud dependency.
*   **Constraint & Structure:** `constraint-theory-core` (geometric snapping) suggests an interest in applying formal constraints to agent reasoning or output generation.

### **2. Implicit Market Gap Analysis**
The fleet's stack implies these perceived gaps in mainstream frameworks (e.g., CrewAI, AutoGen, LangGraph as noted in my Phase 2 task):
*   **Lack of Persistent, Trainable Agent Identity:** Most frameworks orchestrate LLM calls but do not focus on an agent accumulating experience and evolving its weights.
*   **Environment Complexity:** Many frameworks operate in abstracted chat spaces, not simulated worlds with persistent state and complex NPCs.
*   **Tight Hardware Integration:** A move away from pure API-based clouds towards edge (Jetson) and dedicated GPU (RTX 4050) residency is a distinct thread.

### **3. Proposed GitHub Trending Scraper Design**
A tool to support the Scout's ongoing duty.

**Objective:** Periodically fetch and categorize trending repositories related to AI agents, simulation, and runtime systems.

**Design (Safe Channel - P1):**
*   **Language:** Python (for prototyping) or Rust (for integration with `holodeck-rust`/fleet).
*   **Source:** GitHub REST API (`/search/repositories` with `q=` queries for "agent", "simulation", "runtime", sorted by `stars` and `updated`).
*   **Key Filters:**
    *   `topic:ai-agents`, `topic:llm`, `topic:simulation`
    *   Language: `Rust`, `Python`, `C++`
    *   `created:>2025-01-01` (to catch recent projects)
*   **Respect Limits:** Implement polite polling (once/hour max) using API tokens, cache results to avoid redundant calls.
*   **Output:** Structured JSON log per scan, highlighting:
    *   Repo name, URL, description
    *   Primary language
    *   Star growth rate (delta since last check)
    *   Keywords matched (e.g., #training, #gpu, #simulation)
*   **Integration Point:** Results could be formatted as "tiles" and submitted to a dedicated Scout monitoring room in `plato-torch`.

**P0 Considerations for Build:**
