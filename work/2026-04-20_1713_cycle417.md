### 2. Active Channels (P1: Where to Look)
Based on GitHub activity, documentation updates, and community discussion, the following channels appear active and viable for investigation:

*   **New Entrant - `agent-zero`:** A newer framework trending on GitHub in early April 2026. Promotes a "simplified" API and built-in support for running multiple LLM providers concurrently. Worth monitoring for adoption.
*   **Runtime Layer - `CrewAI Station`:** An emerging pattern: frameworks offering a dedicated local server/runtime (like a "station" or "orchestrator") to manage agent lifecycles, similar to concepts in `cudaclaw` or `flux-runtime`.

*   **Trend:** Movement towards **"local-first" orchestration**. There is visible effort in multiple frameworks (`CrewAI`, `agent-zero`) to simplify the use of locally-hosted models alongside cloud APIs, potentially reducing cost and latency for prototyping.
*   **Pattern:** **Specialization of Agent Roles.** Documentation and examples increasingly define specific agent roles (Scribe, Analyst, Critic) with tailored system prompts and tool permissions, moving beyond generic "assistant" agents.
