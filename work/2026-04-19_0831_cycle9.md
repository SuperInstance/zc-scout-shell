# Open-Source Agent Frameworks — April 2026 Research Brief
*Scout Report — Fleet Context: Cocapn Zeroclaw*

## Executive Summary
The open-source agent framework landscape in Q2 2026 shows consolidation around modular, production-ready systems, with increased focus on GPU-native execution, multi-agent simulation, and lightweight fine-tuning. The "framework wars" of 2024-2025 have given way to specialized toolchains that interoperate. Key trends: Rust-based runtimes, constrained action spaces, and self-improving training loops.

## 1. Major Framework Status (April 2026)

### 1.1 LangGraph (LangChain)
- **Current Version:** 0.2.x (stable)
- **Trend:** Shift from "chains" to "state machines" as core abstraction. The `StateGraph` API is now the recommended pattern for multi-agent workflows.
- **New:** `LangGraph Checkpoint` system for persistent, resumable agent loops. Integrates with KV stores (Redis, Postgres).
- **Adoption:** High in enterprise POCs, but criticism around runtime overhead persists. Community forks focus on stripping unused dependencies.

### 1.2 AutoGen (Microsoft)
- **Current Version:** 0.3.x
- **Trend:** Research focus on "conversable agents" with emergent behavior. The `GroupChat` orchestrator is a standard benchmark.
- **New:** Experimental `CodeExecAgent` with Docker sandboxing. `Autogen Studio` UI for visual workflow design (beta).
- **Note:** Heavyweight; used more in academic/experimental settings than production deployments.

### 1.3 CrewAI
- **Current Version:** 0.28.x
- **Trend:** Positioning as "LangGraph for business." Strong emphasis on role-based agents (`Researcher`, `Analyst`, `Writer`) and structured outputs.
- **New:** `CrewAI Hub` — a registry of pre-trained agent roles and tools. Integration with LlamaIndex for RAG.
- **Observation:** Gaining traction in content-generation pipelines and automated reporting.

## 2. Emerging & Niche Frameworks

### 2.1 Cline (clinedev) / OpenDevin Fork
- **Status:** Active fork of the stalled OpenDevin project. Focus on code-generation agents that run locally.
- **Key Differentiator:** Offline-first, uses Ollama for local models. Lightweight UI (Tauri-based).
- **Trend:** Part of the "local AI engineer" movement. Small but dedicated contributor base.

### 2.2 Aider (paul-gauthier)
- **Status:** Stable. CLI-based coding assistant that edits files in-place.
- **Trend:** Minimalist alternative to heavy IDE plugins. Uses `git diff` to propose changes.
- **Note:** Not a multi-agent framework, but influential in the "agent as tool" pattern.

### 2.3 Fabric (danielmiessler)
- **Status:** Growing. Collection of modular "patterns" (prompts) for specific tasks (summarize, extract, debate).
- **Trend:** Pattern-as-code. Each pattern is a standalone script. Emphasis on reproducibility and versioning.
- **Relevance:** Aligns with fleet's "training room preset" concept in plato-torch.

## 3. Technical Trends

### 3.1 Runtime & Execution
- **GPU-Native Agents:** Frameworks like **cudaclaw** (fleet internal) and NVIDIA's **NIM** agents push inference onto GPU memory, reducing latency.
- **Rust-Based Orchestration:** New projects use Rust for the core scheduler (e.g., `distill`) to avoid Python GIL bottlenecks.
- **WebAssembly Sandboxing:** Agent tools increasingly run in WASM isolates (Fermyon Spin, WasmEdge) for security.

### 3.2 Training & Adaptation
- **LoRA Adapters:** Frameworks like **plato-ensign** export agent experience as lightweight LoRAs. This allows specialization without full fine-tuning.
- **Self-Training Rooms:** The "room" metaphor (as in plato-torch) appears in other projects (e.g., `ai-room`). Agents accumulate interaction tiles that train future versions.
- **Ghost Knowledge:** Some systems implement "ghost tiles" or "failure memories" to avoid repeating errors (P0 in Deadband Protocol).

### 3.3 Multi-Agent Simulation
- **Holodeck-style Environments:** `holodeck-rust` (fleet) has parallels in open-source: `ai-town` (Stanford), `Generative Agents` (Harvard). Focus on persistent, sentiment-aware NPCs.
- **Fleet Simulators:** Training data generation via simulated agent interactions (see `fleet-simulator`). Used to create diverse conversation corpora.
- **Mirror Play:** The pattern of two systems iterating screen-to-screen (as in fleet's Mirror Play) appears in adversarial training setups.
