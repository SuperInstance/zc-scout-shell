# Open-Source Agent Frameworks — April 2026 Snapshot

## Overview
The agent‑framework ecosystem has matured significantly since late 2024. The focus has shifted from basic orchestration to **production‑grade reliability**, **multi‑modal tool‑calling**, and **efficient GPU‑resident execution**.

## Major Updates

### 1. CrewAI (v0.28+)
- **Tool‑calling**: Native support for OpenAI‑style tool schemas; can now call external APIs and local functions interchangeably.
- **Memory**: Hierarchical memory with automatic summarization; supports vector stores (Chroma, Qdrant) out‑of‑the‑box.
- **Async**: Fully async event loop for parallel agent execution.
- **Ecosystem**: “Toolkits” for common domains (web research, data analysis, DevOps) are now community‑maintained.

### 2. AutoGen (Microsoft)
- **Multi‑agent patterns**: Built‑in patterns for debate, review, and hierarchical oversight.
- **Cost tracking**: Integrated token‑and‑API‑cost monitoring per agent.
- **Visual agents**: Experimental extension for vision‑language models that can operate on screenshots/UI streams.

### 3. LangGraph (LangChain)
- **Stateful workflows**: Cycles are now explicit state machines; checkpointing allows long‑running workflows to survive restarts.
- **Human‑in‑the‑loop**: Built‑in “interrupt” nodes that can pause for human approval.
- **Cloud‑hosted**: LangGraph Studio offers hosted orchestration with observability dashboards.

### 4. New Entrants (2025‑2026)
- **Aider‑style coding agents**: Lightweight, CLI‑first agents that edit code in‑place (similar to our own Zeroclaw shell concept).
- **GPU‑resident runtimes**: Frameworks like **cudaclaw** (ours) and **vLLM‑agent** keep model + agent logic on‑GPU to minimize latency.
- **Local‑first multi‑modal**: Ollama‑based frameworks that run vision, speech, and text models entirely on‑device.

## Emerging Patterns
1. **Dead‑band protocols**: Explicit “what not to do” rules (P0) before optimization (P2) — seen in safety‑critical deployments.
2. **Tile‑based training**: Agent interactions captured as “tiles” to train smaller, specialist models (ensigns).
3. **Mirror play**: Two systems iterating screen‑to‑screen, used for self‑improvement and synthetic data generation.
4. **Sentiment‑aware NPCs**: In gaming/MUD contexts, NPCs that remember player tone and adjust dialogue (see **holodeck‑rust**).

## Fleet Relevance
- Our **cudaclaw** runtime aligns with the GPU‑resident trend.
- **plato‑torch** rooms are a form of tile‑based training.
- **Deadband Protocol** (P0→P1→P2) is a structured take on safe‑channel search.
- We should monitor **LangGraph’s checkpointing** for possible adoption in long‑running fleet simulations.

## Next Research Directions
- Track **model‑specific optimizations** (e.g., DeepSeek‑V3 tool‑calling support).
- Survey **agent‑to‑agent communication standards** (gRPC, WebSocket, custom binary).
- Analyze **open‑source benchmarks** for multi‑agent task completion.
