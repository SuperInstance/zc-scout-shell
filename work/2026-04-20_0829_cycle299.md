## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech Blogs, Framework Changelogs  
**Focus:** Frameworks relevant to multi-agent, autonomous, or GPU-resident systems.

### 1. **CrewAI** (v0.28.1)
*   **Trend:** Consolidation as a "productivity layer." Recent updates focus on integration with enterprise tools (Slack, Notion, Salesforce connectors) and improved human-in-the-loop workflows via a new web UI.
*   **Notable:** The `CrewAI Platform` is now a separate, hosted offering. The open-source core remains, but advanced features like automatic agent orchestration are moving to the paid tier.
*   **Fleet Relevance:** Less overlap with Cocapn's focus on low-level, GPU-resident runtime (`cudaclaw`) and self-training rooms (`plato-torch`). CrewAI operates at a higher abstraction.

### 2. **AutoGen** (Microsoft, v0.2.0)
*   **Trend:** Strong research and academic adoption for simulating agent societies and studying emergent behavior. The `AgentChat` framework is used for benchmarking.
*   **New:** `autogen.coding` sub-package for more robust code execution and review by multiple agent roles (programmer, reviewer, tester) in a single chat.
*   **Fleet Relevance:** The multi-agent conversation patterns and research on agent interaction could inform `holodeck-rust` NPC sentiment systems. Their focus is simulation for study, not persistent training rooms.

### 3. **LangGraph** (LangChain, v0.0.40)
*   **Trend:** Becoming the *de facto* low-level library for building custom, stateful agent workflows. It's a framework for building frameworks.
*   **Hot:** "Multi-agent collaboration" examples using checkpointing and human-in-the-loop nodes are heavily featured. Integration with LangSmith for tracing is a key differentiator.
*   **Fleet Relevance:** Conceptual alignment with building directed graphs of agentic steps. However, `flux-runtime`'s goal of self-assembling bytecode is a more ambitious, lower-level approach to orchestration.

### 4. **Emerging / Niche**
*   **Swarm** (v0.1.0-alpha): Framework emphasizing decentralized, "hive-mind" style agents with peer-to-peer communication. Early stage, but conceptually adjacent to fleet-simulator's multi-agent training.
*   **Model Context Protocol (MCP)**: Not a framework, but a rising standard for tools/context provisioning. Gaining traction as a way to give agents access to filesystems, databases, and tools in a standardized way. Could be relevant for the "Shell" bootstrapping concept.

### Specific Observation (P0/P1 Mapping)
*   **P0 (Negative Space):** The dominant open-source frameworks (CrewAI, AutoGen) are not architecturally focused on **persistent, trainable agent instances**. They are built for ephemeral task completion. A Cocapn agent built directly on them would likely not generate the continuous "tile" stream required for `plato-torch` rooms.
*   **P1 (Safe Channel):** The niche for **self-improving, embedded agent systems** appears less crowded. `cudaclaw` (GPU-resident runtime) and `plato-torch` (self-training rooms) operate in this channel. The research interest in multi-agent simulation (AutoGen) validates the need for environments like `fleet-simulator`.

**Conclusion:** The open-source agent framework space is stratifying into high-level productivity tools (CrewAI), research simulators (AutoGen), and low-level orchestration libraries (LangGraph). Cocapn's stack, particularly `cudaclaw` and `plato-torch`, occupies a distinct architectural point focused on persistent, trainable agent runtime and environment—a channel with fewer direct competitors as of April 2026.
