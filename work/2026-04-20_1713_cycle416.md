## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Evolution & Consolidation
The landscape shows a move from monolithic "orchestrator" frameworks toward modular, composable systems. Key projects reflect this:

*   **LangGraph** (LangChain) remains a dominant pattern for defining agentic workflows as state machines. Its adoption is driven by integration with the broader LangChain ecosystem (tools, retrievers) and support for persistent, long-running workflows.
*   **CrewAI** continues traction in multi-agent collaboration scenarios, particularly for structured tasks like research and content generation. Recent commits suggest increased focus on interoperability with other tools (e.g., LlamaIndex for RAG).
*   **AutoGen** (Microsoft) is seeing research and enterprise interest for complex multi-agent conversations, though some community feedback notes a steeper learning curve for deployment.

### 2. Emerging Pattern: "Runtime" Specialization
Newer projects are not just frameworks but specialized runtimes, aligning with fleet technologies like `cudaclaw` and `flux-runtime`.

*   **Model Context Protocol (MCP)** gaining attention as a standard for connecting agents to tools and data sources, promoting composability over tight coupling.
*   **Projects like `composio`** and `toolcall` focus on robust, schema-driven tool calling, reducing the "hallucinated function call" problem.
*   **GPU-native agent runtimes** are an active research area, aiming to reduce latency by keeping agent state on GPU, similar to the fleet's `cudaclaw` concept.

### 3. Integration with Open-Source Models
The release of increasingly capable open-source models (DeepSeek, Qwen, Llama 3) is shifting focus. Frameworks are competing on:

*   **Optimization for local inference** (Ollama, LM Studio integration).
*   **Cost-effective scaling** for multi-agent simulations, a direct parallel to the fleet's `fleet-simulator` objective.
*   **Fine-tuning and adaptation pipelines**, which relates to the fleet's `plato-ensign` (LoRA export) approach.

### 4. What's Not Hot (P0: Negative Space)
*   **Monolithic, closed-source agent platforms** with high per-query pricing are facing community skepticism.
*   **Agents without transparent tool/action schemas** are harder to debug and integrate.
*   **Pure "chat-over-docs" agents** are now considered a baseline capability, not a differentiator.

### 5. Fleet Relevance & Safe Channel (P1)
The fleet's architecture—`plato-torch` rooms, tile-based training, specialized ensigns—occupies a distinct channel not directly served by major OSS frameworks. The closest adjacent concepts are:
*   **Simulation-based training** (`fleet-simulator`): Similar to projects using generative agent simulations for data creation.
*   **Specialist model export** (`plato-ensign`): Aligns with the community's focus on fine-tuning small, task-specific models.
