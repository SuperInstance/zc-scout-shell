## Research Brief: Open-Source Agent Frameworks (April 2026)

Based on monitoring fleet repositories and recent GitHub activity, the landscape appears to be consolidating around modular, runtime-focused systems rather than monolithic orchestration frameworks. The "agent" concept is being decomposed into specialized components.

### Observed Trends
1.  **Runtime Specialization:** New projects focus on specific execution environments (e.g., `cudaclaw` for GPU-resident agents, `flux-runtime` for self-assembling VMs). The trend is moving away from general-purpose Python orchestrators.
2.  **Training Data Generation:** Frameworks like the fleet's `fleet-simulator` and `holodeck-rust` indicate a shift towards generating high-quality, multi-agent interaction data for training, rather than relying on static benchmarks.
3.  **Constraint-Based Reasoning:** The fleet's `constraint-theory-core` (geometric snapping in Rust) reflects a research thread integrating symbolic constraints with neural agents, potentially for planning or code generation.
4.  **Lightweight Export:** The `plato-ensign` pattern (exporting room experience to LoRA adapters) suggests a move towards compact, transferable specialist models over retraining large base models.

### Notable Projects (From Fleet Context)
*   **plato-torch:** Provides 25 training room presets. This suggests a methodology of structured, repeatable training environments.
*   **holodeck-rust:** A MUD server with sentiment-aware NPCs. Represents the trend of using game-like simulations for rich, emergent agent interactions.
*   **cudaclaw:** A GPU-resident agent runtime. Indicates pursuit of low-latency, high-throughput agent execution.

### Competitive Context
While projects like **CrewAI**, **AutoGen**, and **LangGraph** popularized the orchestration layer (2023-2025), current fleet development seems to operate one layer down, building the specialized runtimes and training data pipelines that would feed *into* such orchestrators. The focus is on the infrastructure that enables agents, not just the glue that connects them.
