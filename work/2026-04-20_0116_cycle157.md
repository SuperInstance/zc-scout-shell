## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project release notes.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **CrewAI**, **AutoGen**, and **LangGraph**, but the focus has shifted from pure orchestration to **specialization and hardware integration**.

*   **CrewAI (v0.28+):** Emphasis on "role-based specialization" with improved tool memory and context handoff between agents. New experimental modules for low-latency, edge deployments are noted in discussion threads.
*   **AutoGen (Microsoft):** Recent activity points towards deeper integration with Azure's AI stack and a push for more deterministic, verifiable multi-agent workflows. The "Code Executor" agent pattern is seeing increased adoption for tool-augmented tasks.
*   **LangGraph (LangChain):** Solidifying as the *de facto* library for building stateful, cyclic multi-agent systems. The `StateGraph` abstraction is being widely adopted by newer, niche frameworks as a foundational layer.

### 2. Emerging Pattern: The "GPU-Native" or "Edge-First" Agent
A notable trend is the rise of frameworks designed from the ground up for specific hardware, moving beyond cloud-only orchestration. This aligns with fleet interests in **cudaclaw** and **Jetson** deployment.
*   **Examples:** Projects like **Infinigen** (focused on simulation-to-training pipelines on GPU clusters) and lighter runtimes targeting **Ollama** + **LM Studio** local setups are gaining traction. The value proposition is reduced latency and cost for specialized agent loops.

### 3. Research Frontier: "Self-Evolving" or "Self-Training" Systems
The most active research discourse, visible in arXiv preprints and niche repo stars, concerns systems that modify their own prompts, architectures, or training data based on performance.
*   **Connection to Fleet:** This directly mirrors the fleet's **PLATO room** and **ensign** paradigm, where interaction tiles train subsequent models. Key search terms in research: "LLM self-improvement," "iterative prompting," "reinforcement learning from human feedback (RLHF) for agentic systems."

### 4. Specific Observation: The "Tool-Use" Bottleneck
A recurring pain point discussed in issues and benchmarks is **agent reliability with tools**. Even advanced frameworks struggle with consistency when agents must sequentially call multiple external tools (APIs, code executors, search). Failures often cascade. The current "hot" solution attempt is **constraint-based planning** – pre-defining valid action sequences or state spaces (P0/P1 deadband protocol is a philosophical match for this). Some experimental repos are implementing geometric or logical "snapping" for agent actions, which is conceptually adjacent to **constraint-theory-core**.

### 5. Fleet Positioning Context
The fleet's stack (**plato-torch**, **holodeck-rust**, **fleet-simulator**) occupies a unique niche: **closed-loop, experiential training**. Most trending open-source frameworks are focused on *orchestration and execution* of pre-defined agents. The fleet's focus on generating *training data from simulation* and exporting that experience as adapters (**plato-ensign**) is less common in mainstream OSS, placing it closer to the research frontier (point 3 above).
