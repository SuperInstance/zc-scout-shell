## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech Blogs, Framework Repos

### 1. Dominant Frameworks & Trends
The landscape remains anchored by **CrewAI**, **LangGraph**, and **AutoGen**, but the focus has shifted from pure orchestration to **specialization and deployment**.

*   **CrewAI (v0.28+):** Actively developing "Crews as Microservices." Recent commits show work on a lightweight HTTP server (`crewai-server`) to expose agent crews as API endpoints, moving towards a production-ready, containerized deployment model.
*   **LangGraph (v0.2+):** Heavy emphasis on **checkpointing** and **human-in-the-loop** workflows. The "StateGraph" abstraction is being used to build persistent, long-running agent processes that can be paused, inspected, and resumed. This aligns with a trend towards more observable and controllable agentic systems.
*   **AutoGen (v0.2+):** Research focus on **multi-modal agent chat** and **code execution security**. The `GroupChat` with `code_execution_config` is a common pattern for safe, collaborative coding tasks.

### 2. Emerging / "Hot" Projects (Q2 2026)
*   **Aider (aider.chat):** Gaining traction as a **CLI-first, code-centric agent**. It directly integrates with git, edits files in-place, and is designed for software engineers who want an agent pair-programmer inside their existing terminal workflow. Its simplicity is a noted contrast to heavier orchestration frameworks.
*   **OpenDevin:** The open-source attempt to replicate Devin (Cognition AI) continues as a major research project. Progress is incremental, with focus on improving the **SWE-bench** benchmark scores. The architecture is complex (Docker, SSH, browser), and running it locally remains non-trivial.
*   **Smol Agents / Smoldeveloper:** The "smol" philosophy (small, focused, understandable agents) has influenced design patterns. The trend is towards **composing many small, single-purpose agents** rather than building monolithic, general-purpose ones.

### 3. Key Pattern: The "Runtime" Shift
A noticeable trend is the move from "frameworks" to **dedicated agent runtimes**, which resonates with fleet technology like `cudaclaw` and `flux-runtime`.
*   Projects like **E2B** (secure sandboxed environments) and **Phidata** (data-aware agent platform) are providing the underlying infrastructure (sandbox, memory, tools) upon which agent logic is built.
*   The discussion is moving from "how to chain prompts" to "how to host, monitor, and secure an agentic service."

### 4. Fleet Context Comparison
*   **plato-torch** aligns with the "specialized training room" concept, which is less common in mainstream OSS. Most frameworks focus on inference-time orchestration, not on creating environments for continuous agent training and adaptation.
*   **holodeck-rust** (sentiment-aware NPCs) occupies a niche in simulation-for-training, similar to but distinct from projects using **GPT-NeoX** or **Voyager** for Minecraft-like agent training environments.
*   The fleet's **Deadband Protocol (P0/P1/P2)** is a unique formalization of constraint-first reasoning not explicitly found in the major frameworks, though LangGraph's support for conditional edges and state validation touches on similar concepts.

### 5. Specific Factual Observation
**GitHub Trending (Last 7 Days)** shows a spike in stars for **`microsoft/agents`** (a new, official Microsoft repository providing building blocks for AI agents on Azure). This indicates major cloud providers are now releasing curated, vendor-locked agent SDKs, which may fragment the ecosystem between open-core and cloud-native tools.
