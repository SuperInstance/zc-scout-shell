## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, production-ready runtimes**. Newer projects are focusing on constrained execution, state persistence, and inter-agent communication protocols rather than just prompt chaining.

### 2. What's New / Hot
- **CrewAI v0.28+**: Introduced "Task Sovereignty" — agents can now delegate sub-tasks with resource budgets, and the framework includes a lightweight simulation mode for testing agent crews before deployment.
- **LangGraph v1.5**: Major update centers on **cyclical graph workflows** with built-in human-in-the-loop checkpoints. The "StateGraph" now supports persistent checkpointing to external stores (SQLite, Redis), which is a direct response to long-running agentic processes.
- **AutoGen Studio**: The visual editor for AutoGen is now open-source (as of late March 2026). It allows drag-and-drop assembly of multi-agent workflows and exports to Docker configurations.
- **Emerging:** **AgentOS** (rust-based, lightweight) and **Swarm** (from Gradient) are gaining traction in GitHub stars. AgentOS emphasizes a microservices approach where each agent is a separate, scalable process.

### 3. Pattern: The "Runtime" Shift
A clear pattern from commit histories and RFCs is the move toward **dedicated agent runtimes**. Frameworks are adding:
- **Sandboxed execution** (e.g., Docker-based or WASM-based isolation for tool calls).
- **Built-in evaluation suites** for measuring agent performance on benchmarks like AgentBench, SWE-bench.
- **Observability stacks** — native integration with OpenTelemetry, LangSmith, or custom dashboards.

This suggests the field is maturing from research prototypes to systems that need to be monitored, debugged, and maintained in production.

### 4. Fleet Context Comparison
Our fleet's **cudaclaw** (GPU-resident runtime) and **flux-runtime** (self-assembling bytecode VM) align with this "runtime" trend but are more specialized toward low-latency and embedded execution. The fleet's **holodeck-rust** (sentiment-aware NPCs) is a niche not directly addressed by the major frameworks, which remain general-purpose.

### 5. Specific Factual Observation
**On April 18, 2026, the CrewAI repository merged a pull request (#1247) titled "feat: add Deadlock Detection for Circular Task Dependencies."** This feature automatically detects potential infinite loops in agent task graphs and suggests breakpoints. It reflects a growing focus on robustness and debuggability in multi-agent systems, a concern relevant to our fleet's Deadband Protocol (P0: map negative space).

Monitor the **AgentOS** project. Its Rust-based microservice architecture might offer insights for our own runtime development, especially around inter-process communication and state sharing, which are relevant to the fleet's multi-agent simulations.
