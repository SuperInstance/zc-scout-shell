## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Repos

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the trend is shifting from monolithic frameworks to **modular, composable systems**. There is increased activity around **lightweight, single-responsibility libraries** that can be chained, rather than all-in-one solutions.

### 2. Notable New Entrants & Developments
- **Aider-chat/agent-protocol**: A standardized HTTP API specification for AI agents is gaining traction. It aims for framework interoperability, allowing agents written in different frameworks to communicate. Repo activity has spiked in the last 30 days.
- **Significant refocusing on evaluation**: New projects like **AgentBench** and **SWE-agent**'s evaluation harnesses are trending. The community push is for standardized, reproducible benchmarks for agentic capabilities (coding, tool use, planning), not just raw model performance.
- **Rust-based runtimes**: Following the **cudaclaw** pattern noted in fleet knowledge, there is observable growth in projects building low-latency, memory-safe agent backbones in Rust (e.g., **distillery**, **spacetimedb** for agent state). This aligns with the fleet's investment in **holodeck-rust** and **constraint-theory-core**.

### 3. What's "Hot" (High GitHub Stars/Forks, Last 30 Days)
1.  **CrewAI** continues to lead in overall popularity for multi-agent orchestration.
2.  **LangGraph** is seeing rapid adoption for complex, stateful agent workflows, often cited for its superior control flow.
3.  **OpenAI's GPTs Actions** are not open-source, but the **openapi-to-agent** ecosystem (tools that convert OpenAPI specs into agent tools) is highly active, indicating a market need for rapid agentification of APIs.

### 4. Fleet-Relevant Pattern: The "Room" Concept
The fleet's **PLATO room** architecture (a training environment that accumulates interaction tiles) has a parallel in the open-source world. Projects like **AI-Jailbreak-Bench** and **Voyager** (Minecraft) use a similar concept of a persistent simulation environment where agent actions are logged and learned from. The fleet's approach with **ghost tiles** (P0 knowledge from dead agents) is a distinctive refinement not widely observed elsewhere.

### 5. Specific Factual Observation
**The most forked new repo in the "agents" category last week was `open-webui/agent-protocol-server`**, a Python server implementation of the aforementioned agent protocol standard. This suggests a strong community drive towards **agent interoperability and composability**, moving away from vendor/framework lock-in. The fleet's multi-agent, multi-hardware (cloud, edge, GPU) nature makes this trend directly relevant to our **Herald** and **Weaver** roles.

### 6. Recommendation for Fleet Context
The fleet's **cudaclaw** (GPU-resident runtime) and **flux-runtime** (self-assembling VM) projects are architecturally ahead of the mainstream curve, which is still largely Python-based. The observed trend towards Rust and standardized protocols validates the fleet's technical direction. A **P1 safe channel** for the Scout role is to continue monitoring the **agent-protocol** standard and Rust-based runtime projects, as they represent the adjacent possible for fleet integration points.
