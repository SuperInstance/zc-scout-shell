### 1. Dominant Frameworks & Trends
The landscape remains anchored by three major open-source projects, each evolving distinct niches:

*   **LangChain/LangGraph:** Continues to be the most starred and forked framework. The focus has shifted from basic chains to **production-grade, stateful multi-agent systems** via LangGraph. Recent commits show heavy investment in **observability tools** (LangSmith integration) and **optimized runtimes** for serverless deployments. The "LangGraph Studio" local dev tool is gaining traction.
*   **CrewAI:** Has solidified its position as the framework for **role-based, collaborative agent swarms**. Its primary differentiator is a strong emphasis on **process-driven task execution** (sequential, hierarchical, consensual). The April 2026 trend shows increased adoption for **business process automation** use cases, with new community templates for sales prospecting and competitive analysis.
*   **AutoGen (by Microsoft):** Maintains strength in **research-oriented and highly customizable conversational agent** scenarios. Its pattern of using **code-executing "User Proxy" agents** for tool use remains unique. Recent activity indicates work on **lightweight, specialized model fine-tuning** workflows integrated directly into agent teams.

### 2. Emerging Pattern: The "Runtime" Layer
A new layer of abstraction is emerging, separate from the orchestration frameworks above. Projects like **`flux-runtime`** (noted in fleet knowledge) exemplify this. These are minimal, self-assembling execution environments designed for **portability and embedded deployment**. The trend suggests a move away from monolithic frameworks towards **composable, lightweight runtimes** that an orchestrator can manage.

### 3. Hot New Entrant (Q1 2026): **Swarm**
A framework called **Swarm** (by OpenAI alumni) has rapidly gained GitHub stars. Its core thesis is **emergent behavior from simple, massively parallel agents**. Unlike CrewAI's structured roles, Swarm agents have minimal predefined logic and rely on **local environment signals and stigmergy** (digital pheromone trails) for coordination. It's experimental but is being watched for novel approaches to complex, dynamic problem-solving.

### 4. Integration Shift: **Model-Agnostic Tooling**
There is a noticeable push for tools that work across any LLM provider. Frameworks are deprecating hard-coded OpenAI integrations in favor of **standardized LiteLLM or OpenRouter-compatible interfaces**. This aligns with the fleet's observed use of multiple models (DeepSeek, Groq).

### 5. Fleet Context Observation
The fleet's architecture, with **PLATO rooms generating training tiles** and **ensigns as exported LoRAs**, operates at a different layer than these frameworks. The frameworks (CrewAI, LangGraph) are for *orchestration*, while the fleet's `plato-torch` and `cudaclaw` focus on *agent training and GPU-resident execution*. This suggests a potential **P1 safe channel**: using an orchestrator like CrewAI to manage a swarm of specialized fleet ensigns, each trained in a specific PLATO room.
