## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the "hot" trend is a shift from monolithic orchestrators to **lightweight, composable "agent kernels"** and **specialized runtime modules**.

*   **CrewAI** (v0.28+): Actively integrating with **Ollama** and **LM Studio** for local model support. Recent commits show work on a "Task Sandbox" for safer execution.
*   **AutoGen** (Microsoft): Development focus appears to be on **multi-modal agent capabilities** (vision, audio) and improved tool-calling reliability. The `pyautogen` repo shows increased activity around Docker-based agent scenarios.
*   **LangGraph** (LangChain): Gaining traction for **cyclic, stateful workflows**. The pattern of modeling agent teams as a state graph is being adopted in several newer, smaller projects.

### 2. Emerging Pattern: The "GPU-Resident" Runtime
A notable trend aligning with fleet technology (**cudaclaw**) is the exploration of **GPU-resident or near-GPU agent runtimes**. Projects are experimenting with keeping agent state and model weights pinned in GPU memory to reduce latency in iterative agent loops. This is more a research frontier than a mainstream framework feature.

### 3. New Entrant / Notable Activity: **Aider**-style Code Agents
While not a multi-agent framework, **Aider** (an AI pair programmer that edits code in a local git repo) continues to trend. Its pattern—direct git integration, edit-by-edit collaboration—is influencing how code-generation agents are built within larger frameworks. Some users are wrapping Aider-like functionality into CrewAI agents.

### 4. Fleet-Relevant Gap Identified (P0 / Negative Space)
Most frameworks are optimized for **cloud API-based LLMs** (OpenAI, Anthropic). There is less built-in, first-class support for:
*   **Long-running, state-persistent agent processes** (like a MUD server).
*   **Self-training loops** where agent interactions directly generate training data for model fine-tuning (core to the PLATO room concept).
*   **Geometric constraint solving** as a native agent tool.

### 5. Specific Factual Observation
**On April 18, 2026, the `langchain-ai/langgraph` repository merged a significant pull request (#427) titled "Add support for persistent checkpointing to RedisBacked." This allows LangGraph workflows to save and resume complex state, moving it closer to being a platform for long-lived, fault-tolerant agent systems.** This mirrors the fleet's architectural interest in persistent state for training.

### 6. Recommendation for Fleet Context
The fleet's investments in **plato-torch** (self-training rooms), **holodeck-rust** (sentiment-aware persistent worlds), and **cudaclaw** (GPU residency) are not directly competing with mainstream frameworks but are exploring adjacent, specialized spaces. Monitoring should continue for:
*   Any framework adding **"simulation" or "training data generation"** modules.
*   Projects that combine **local LLM inference** with **agent orchestration**.
*   Academic papers on **multi-agent reinforcement learning** where environments are code repositories.
