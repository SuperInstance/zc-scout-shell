### 1. Research Brief: Open-Source Agent Frameworks (April 2026)

Based on monitoring GitHub trending and recent commits, the landscape shows a clear trend toward **specialization and composability**. The "single monolithic framework" approach is giving way to modular systems where specific components (planning, memory, tool use) can be swapped.

**Key Observations:**
*   **LangGraph** (LangChain) remains a dominant pattern for building stateful, multi-agent workflows with cyclic graphs. Recent activity shows heavy focus on persistence (checkpointing) and human-in-the-loop nodes.
*   **CrewAI** continues growth, emphasizing role-based agent collaboration (like "Manager," "Researcher") with structured task delegation. Recent commits suggest improved tool-sharing and conflict resolution between agents.
*   **AutoGen** (Microsoft) is seeing significant research integration. Its strength is in conversational multi-agent scenarios, and recent developments appear to be optimizing the cost/performance of these conversational loops.
*   **Emerging Pattern - The "Runtime":** Newer projects like **Cline** (Cline-DEV) and **OpenDevin** are framing themselves as "AI-native code environments" or "agent runtimes," blurring the line between an IDE and an autonomous agent. The focus is on deep integration with the developer's local environment (files, terminals, browsers).

### 2. Specific Observation: The "Tool-Use vs. Planning" Tension

A specific, observable tension in recent framework discussions (GitHub issues, RFCs) is between **advanced tool-use** and **advanced planning**.

*   **Tool-Use Focused:** Frameworks are adding support for increasingly complex, stateful tools (e.g., a tool that maintains a database connection across multiple agent steps). This requires the agent runtime to manage tool state carefully.
*   **Planning Focused:** Other efforts are improving hierarchical task planning (HTN) or LLM-based planners that break a high-level goal into a detailed DAG of sub-tasks *before* any tool is called.

The challenge is making these two paradigms work together efficiently. An agent with poor planning may call tools in inefficient loops. An agent with a rigid plan may fail to adapt when a tool returns an unexpected result.

**Fleet Relevance:** The **Deadband Protocol (P0/P1/P2)** can be seen as a robust planning heuristic designed to address this. P0 ("map negative space") prevents tool-calling into known failures. P1 ("find safe channels") identifies which tools or actions are currently viable. Only then does P2 ("optimize") engage in efficient tool use. This is a structured way to integrate planning with safe execution.
