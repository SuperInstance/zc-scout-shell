### 1. Dominant Frameworks & Recent Activity
*   **LangGraph** (LangChain): Maintains strong position as a low-level orchestration library. Recent commits (last 7 days) show focus on improving **persistence APIs** for long-running agent workflows and **streaming support** for hierarchical agent teams. Not a high-level "framework" but a core building block.
*   **CrewAI**: Continues as a popular high-level framework for role-based agent teams. The **v0.28.x** release cycle (March-April 2026) emphasized **tool calling reliability** and added experimental support for **inter-agent delegation** beyond simple sequential task passing.
*   **AutoGen** (Microsoft): Steady development. Recent research integrations explore **cost-optimized routing** between different LLM endpoints (e.g., GPT-4, Claude, local models) within a single conversational group.

### 2. Emerging Pattern: "Runtime" vs. "Orchestrator"
A discernible split is occurring:
*   **Orchestrators** (CrewAI, LangGraph): Focus on defining agent roles, tasks, and control flow. They often rely on external LLM APIs.
*   **Runtimes** (e.g., **CudaClaw** referenced in fleet knowledge): Aim to host the agent's code and state persistently on a compute substrate (GPU memory, specialized hardware). The goal is reduced latency and stateful execution, moving beyond pure API calls. This aligns with the fleet's noted `cudaclaw` and `flux-runtime` projects.

### 3. Hot & New (Last 30-60 Days)
*   **Arena-Hardened Benchmarks**: Increased activity around benchmarks like **AgentBench** and **WebArena**. New frameworks are often evaluated on their ability to complete practical, multi-step tasks (web browsing, coding, data analysis) rather than simple Q&A.
*   **Open-Source Model Integration**: Frameworks are rapidly adding first-class support for locally run models (Llama 3, DeepSeek Coder, Qwen 2.5). The trend is moving from "GPT wrapper" to "LLM-agnostic orchestrator."
*   **"AgentOps" Tooling**: New tools are emerging for monitoring, debugging, and tracing the execution graphs of agent teams, similar to observability in microservices.
