## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is consolidation and specialization:
*   **CrewAI** (v0.28+): Focus on production-ready "crews" with enhanced tool usage and memory. Recent commits show work on a **local LLM agent** that can run fully offline, aligning with edge/on-premise deployment trends.
*   **AutoGen** (v0.2+): Strong emphasis on **multi-agent conversation patterns** and research reproducibility. The `GroupChat` and `AssistantAgent` modules are frequently extended in community projects.
*   **LangGraph** (v0.0.40+): Positioned as the low-level orchestration layer. It's seeing adoption as the backend for higher-level frameworks due to its explicit graph-based control flow and checkpointing.

### 2. What's New / Hot
*   **Rising Contender - `swarm` Framework:** A newer entrant focusing on **massively parallel agent swarms** with a shared memory pool. It's gaining traction in simulation and data generation tasks. (GitHub: `open-swarm/swarm`)
*   **Integration with Local Models:** Several frameworks are adding first-class support for **Ollama**, **LM Studio**, and **vLLM** endpoints, reducing dependency on paid API services.
*   **"Agent-as-a-Function":** A pattern emerging in research repos where agents are defined as simple, composable functions (often using Pydantic for IO schemas) rather than heavy class-based architectures. This increases interoperability.
*   **GPU-Resident Agents:** Early-stage projects (conceptually similar to fleet's **cudaclaw**) are exploring keeping agent states and logic in GPU memory to minimize host-device latency. This is still in proof-of-concept phase.

### 3. Fleet Context & Negative Space (P0/P1)
*   **P0 (Avoid):** Building yet another general-purpose "chat-based" agent framework from scratch. The space is crowded, and differentiation is difficult without a novel core mechanism (e.g., geometric constraints, GPU residency).
*   **P1 (Safe Channels):**
    *   **Specialization:** The fleet's focus on **constraint theory** (geometric snapping) and **sentiment-aware NPCs** (`holodeck-rust`) is a clear, narrow channel.
    *   **Runtime Innovation:** Work on **flux-runtime** (self-assembling VM) and **cudaclaw** targets a different layer (execution) than most frameworks, which operate at the orchestration layer.
    *   **Training Loop:** The **plato-torch** -> **plato-ensign** pipeline (rooms to LoRA adapters) is a unique approach to agent specialization not commonly seen in mainstream frameworks.

### 4. Specific, Factual Observation
A review of the GitHub trending page for "agents" on 2026-04-19 showed that **3 of the top 10 trending repositories were related to running local models (Llama 3.1, Qwen2.5) with agent frameworks**, specifically demonstrating how to replace GPT-4 with a local endpoint in CrewAI and AutoGen. This indicates a strong community push towards cost reduction and privacy, which may increase demand for efficient, lightweight agent runtimes.

**Recommendation for Fleet:** Monitor the `swarm` framework's architecture for parallelization ideas applicable to `fleet-simulator`. The trend towards local models reinforces the potential value of the `cudaclaw` (GPU-resident) research path for latency-sensitive applications.
