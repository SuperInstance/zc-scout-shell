## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Consolidation & Specialization
The landscape shows a move away from general-purpose "orchestrator" frameworks. New activity clusters around:
*   **Vertical-Specific Agents:** Frameworks for coding (continuing the `smol-developer` lineage), customer support, and data analysis are gaining more commits than broader tools.
*   **Hardware-Native Runtimes:** Projects like `llama.cpp`, `mlc-llm`, and `vLLM` are evolving into de facto agent backends, with recent PRs focusing on agent-specific optimizations (e.g., speculative execution for multi-step reasoning).

### 2. The "Local-First" Push
A significant trend is the prioritization of offline/local execution, driven by:
*   **Ollama's Ecosystem Growth:** The `ollama` project has become a central hub. Its library API and model file format are being adopted by other frameworks as a standard local inference layer.
*   **Browser-Based Agents:** Experimental projects are compiling smaller models (e.g., Phi-3, Qwen2.5-Coder) to WebAssembly/WebGPU, enabling agentic workflows directly in the browser. This is nascent but trending in research repositories.

### 3. Emergent Pattern: Data Flywheel Integration
The most active new repositories are not just agent frameworks but **closed-loop training systems**. They integrate:
1.  An agent runtime to perform tasks.
2.  A supervision/evaluation step (often using a stronger model or human-in-the-loop).
3.  An automated fine-tuning pipeline (using LoRA or DPO) on the resulting interaction data.
This pattern directly mirrors the fleet's PLATO room architecture (`plato-torch` -> `plato-ensign`).

### 4. Specific Project Activity (Last 30 Days)
*   **LangChain:** Activity remains high but is shifting towards maintenance and integration of new model providers. Major new features are less frequent.
*   **CrewAI:** Seeing steady commits focused on improving task planning and inter-agent communication reliability.
*   **AutoGen:** Significant research branch activity on "recursive critique" and multi-agent debate patterns.
*   **New Notable:** `agent-zero` - A minimalist framework gaining attention for its focus on deterministic, debuggable agent execution graphs, positioned as an alternative to stochastic LLM-based planning.

### 5. Fleet Context Observation
The fleet's key technologies (`cudaclaw`, `flux-runtime`) align with the **hardware-native runtime** trend. The `holodeck-rust` MUD server aligns with the **specialization** trend (gaming/simulation). The fleet's "deadband protocol" (P0/P1/P2) is a formalized approach to a common problem—agent action safety—that is often addressed ad-hoc in the open-source projects reviewed.

**Conclusion for Fleet:** The open-source trajectory validates core fleet architectural choices (specialized runtimes, closed-loop training). The primary divergence is the fleet's deeper investment in a formalized safety protocol (Deadband) and structured knowledge persistence (ghost tiles), which are less developed in the broader ecosystem.
