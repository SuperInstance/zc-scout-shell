## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Dominant Frameworks & Trends
The landscape remains anchored by **CrewAI**, **AutoGen**, and **LangGraph**, but the focus has shifted from pure orchestration to **specialization, observability, and cost control**.

*   **CrewAI** continues to lead in structured, role-based multi-agent workflows. Recent commits (last 30 days) show heavy investment in **integrated tool calling** (native support for Tavily, Exa, Brave Search) and **streaming intermediate steps** to frontends. The "Planning" module is being extended with LLM-based constraint solvers.
*   **LangGraph** (LangChain) is seeing adoption for **stateful, long-running agentic processes**. The pattern of modeling agent teams as a cyclic graph with persistence (checkpointing) is prominent in production use cases. Their "StateGraph" abstraction is a key differentiator.
*   **AutoGen** from Microsoft maintains strong research and enterprise integration ties. Recent activity highlights **conversation patterns for code generation & review** and improved **cost tracking** per agent/dialogue turn.

### 2. Emerging & Niche Players
*   **Model Context Protocol (MCP)** is gaining traction as a **standard for tool/context provisioning** to agents, decoupling the agent runtime from the data sources. Servers for PostgreSQL, Notion, and file systems are popular.
*   **Smol Agents / SmolLM** trend continues: ultra-small, focused models (e.g., 2B-7B parameters) fine-tuned for specific agentic tasks (SQL generation, bash control) are being bundled with minimalist frameworks to reduce latency and cost.
*   **Rust-based runtimes** are appearing for performance-critical layers. Projects like **`crat`** (not `cudaclaw`) focus on deterministic, sandboxed execution of tool calls, reflecting a concern for safety and reproducibility.

### 3. Observed "Hot" Repos & Topics (Last 7 Days)
1.  **`open-webui/open-webui`**: Continues trending. Its agent extension framework allows users to visually chain models and tools, making advanced agentic workflows accessible.
2.  **`microsoft/autogen`**: A new release candidate (`v0.3.0rc`) featuring enhanced **WebSocket support for real-time collaboration** between human and agent participants.
3.  **`langchain-ai/langgraph`**: Increased discussion around **"hierarchical agents"** where a supervisor agent spawns and manages sub-graphs for sub-tasks.
4.  **`ToolBench` / `OpenTool`**: Repositories providing **standardized benchmarks for agent tool-use accuracy** are receiving more stars, indicating a community push for evaluation.

### 4. Fleet-Relevant Insight (P1/P2 Mapping)
*   **P0 (Negative Space)**: Monolithic, opaque agent systems with poor cost visibility. Frameworks that don't support partial execution or state saving.
*   **P1 (Safe Channels)**: Frameworks with strong observability hooks (token usage, step logs), support for multiple LLM backends, and modular tool integration (like MCP). The trend towards **specialized, small models** aligns with the fleet's "ensign" concept.
*   **P2 (Optimization)**: The fleet's `plato-torch` and `cudaclaw` projects could find integration points with the **Rust-based runtime** trend for performance, and the **MCP standard** for tooling. The "ghost tiles" concept is a unique form of persistent knowledge not yet seen in mainstream frameworks.

### 5. Specific Factual Observation
On April 18, 2026, the **CrewAI** repository merged a pull request (#1247) titled *"feat: Add persistent memory backend for agents using SQLite."* This allows agent roles to maintain memory across sessions, a direct move towards more persistent, stateful agents—a functionality previously more associated with LangGraph's checkpointing system.

**Conclusion:** The open-source agent framework ecosystem is maturing beyond basic orchestration. The competitive axes are now **specialization, operational control (cost/state), and interoperability**. The fleet's architectural concepts (ensigns, tiles, specialized rooms) are adjacent to but distinct from these trends, potentially occupying a unique niche in **cumulative, experience-based training**.
