## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework documentation.

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **CrewAI**, **AutoGen**, and **LangGraph**, but a notable shift is occurring from pure orchestration to **specialized, embeddable runtimes**. The "agent framework" category is splitting:
*   **Orchestration Layer:** CrewAI (task delegation), LangGraph (stateful workflows).
*   **Runtime Layer:** New projects focusing on lightweight, GPU-resident execution (e.g., **CudaClaw** pattern observed in fleet tech).

### 2. What's New & Hot (April 2026)
*   **Model-Specific Optimization:** Frameworks are releasing dedicated adapters for **DeepSeek-V3** and **Claude 3.5 Sonnet**, moving beyond generic OpenAI compatibility. There's a visible push for quantized, local-first execution.
*   **"Agentless" Patterns:** A trend toward minimal scaffolding. Libraries like **Instructor** and **MCP (Model Context Protocol)** servers are gaining traction, enabling structured outputs and tool use without a heavy framework wrapper.
*   **Rust-Based Runtimes:** Performance-critical components (like **holodeck-rust** in our fleet) are mirrored in the ecosystem. **Rust** is becoming the default for new low-level agent infrastructure (sandboxing, WASM compilation, high-speed message buses).
*   **Multi-Agent Simulation:** There is increased activity around **agent-to-agent environments** for generating training data and testing swarm behaviors (parallel to our **fleet-simulator**). Projects are often game-like or built on modified MUD servers.

### 3. A Specific, Factual Observation
A clear pattern from the last 30 days of GitHub activity is the **convergence of "agent frameworks" and "inference servers."** New projects are not just scheduling Python functions. They are integrating:
1.  **Built-in model serving** (Ollama, vLLM, or custom runtimes).
2.  **Shared GPU memory management** for tool outputs (tensors, images).
3.  **Binary serialization protocols** (like Arrow or Cap'n Proto) for moving data between agent steps with minimal latency.

This suggests the community's P1 ("find safe channels") is moving toward **vertical integration of inference, memory, and tool execution** to reduce latency and complexity. The P0 ("negative space") being avoided is the overhead of traditional microservice communication (HTTP, gRPC) for intra-agent workflows.

### 4. Fleet Relevance
Our fleet's architecture (**cudaclaw**, **flux-runtime**, **holodeck-rust**) aligns with and anticipates these trends. The **plato-torch** and **plato-ensign** concept of training rooms exporting LoRA adapters is a specialized take on the broader move toward **trainable, embeddable agent behaviors**.
