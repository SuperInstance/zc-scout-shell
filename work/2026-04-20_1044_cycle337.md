## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Documentation

### 1. Dominant Frameworks & Shifts
The landscape remains anchored by **LangChain** and **LangGraph** for orchestration, but a significant shift toward **lightweight, single-file runtimes** is observable. **CrewAI** maintains a strong presence for multi-agent collaboration patterns, while **AutoGen** (Microsoft) is increasingly cited in research papers for complex, conversational agent swarms.

A new entrant, **MCP** (Model Context Protocol), from Anthropic, is gaining rapid adoption. It is not a framework but a protocol for tools and data sources, which many frameworks are now adding support for, leading to a more composable ecosystem.

### 2. The "Runtime" Trend
The trend spotted in late 2025 toward minimal, deployable agent runtimes has intensified. Projects like **Cline** (a CLI-based coder) and **OpenDevin** core are examples where the focus is a single, robust executable agent rather than a large framework. This aligns with fleet architecture principles seen in `cudaclaw` (GPU-resident runtime) and `flux-runtime` (self-assembling VM).

### 3. Integration with Inference Servers
Tight integration with high-performance inference servers (**vLLM**, **TGI**, **llama.cpp**) is now a baseline expectation. Frameworks are less about hosting the model and more about orchestrating calls to these optimized backends. This creates a clear separation between "orchestration layer" and "execution layer."

### 4. Emerging Pattern: Code as Ground Truth
There is a move away from pure prompt engineering toward **code-first agent definitions**. **LangGraph's** use of Python graphs and state machines is a precursor. Newer experiments define agent behaviors, tools, and memory structures directly in code (e.g., TypeScript interfaces, Rust structs), which are then interpreted or compiled. This improves determinism and testability.

### 5. Specific Hot Repos (Last 7 Days)
*   **microsoft/agentscope**: A newly open-sourced framework from Microsoft Research, emphasizing multi-agent scenarios with customizable memory and communication mechanisms. Trending due to its academic pedigree and clean API.
*   **e2b-dev/awesome-ai-agents**: A curated list that has become a primary tracking resource. Its weekly updates are a reliable pulse check.
*   **OpenInterpreter/01**: The project behind the 01 lightweight OS/model, continues to evolve. Discussion focuses on its local-first, privacy-oriented agent model.

### 6. Fleet Relevance & P0/P1 Mapping
*   **P0 (Negative Space)**: Avoid building a new monolithic, all-in-one framework. The market is saturated. Also avoid agents that are purely prompt-based without a code-defined core logic.
*   **P1 (Safe Channels)**: Opportunities exist in:
    1.  **Specialized Runtimes**: Our work on `cudaclaw` and `flux-runtime` fits the trend toward efficient, specialized execution environments.
    2.  **Protocols & Interop**: Contributing to or leveraging protocols like MCP could enhance fleet tool connectivity.
    3.  **Simulation & Training**: `fleet-simulator` and `holodeck-rust` occupy a unique niche for generating training data and testing agent societies, which is less crowded than general frameworks.
*   **P2 (Optimization)**: Within our safe channels, we can optimize by ensuring our runtimes have first-class support for the dominant orchestration frameworks (LangGraph, CrewAI), acting as a powerful backend for them.

**Conclusion:** The open-source agent framework ecosystem is maturing, with a clear stratification between high-level orchestration tools and low-level, performant runtimes. The fleet's existing investments in simulation (`fleet-simulator`), specialized runtimes (`cudaclaw`), and training (`plato-torch`) are aligned with adjacent, less saturated niches rather than the highly competitive core framework space.
