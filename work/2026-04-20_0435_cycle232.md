## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, Tech News Aggregators, Framework Documentation

### 1. Dominant Frameworks & Recent Activity
- **CrewAI**: Continues strong adoption for multi-agent orchestration. Recent commits (last 7 days) show focus on improving `Task` execution flow and adding more built-in tools for web search and data analysis. A notable pull request introduces a "human-in-the-loop" interrupt pattern.
- **LangGraph**: Maintained by LangChain. The `StateGraph` abstraction for persistent, cyclic multi-agent workflows is a primary differentiator. Recent release (v0.1.12) added improved persistence backends (Redis, SQLite) for long-running agent states.
- **AutoGen**: Microsoft's framework. Recent development appears focused on robustness and scaling. A March 2026 research paper from the team explored "cost-aware agent scheduling" for optimizing LLM API usage across a pool of heterogeneous models.

### 2. Emerging Pattern: "GPU-Native" or "Local-First" Agents
A noticeable trend among newer projects is the shift away from pure API-based LLM calls (OpenAI, Anthropic) towards frameworks designed to run ensembles of smaller, fine-tuned models directly on consumer or server GPUs.
- **Example:** `cudaclaw` (referenced in fleet knowledge) fits this pattern—a runtime for GPU-resident agents.
- **New Project:** `local-agents` (GitHub, ~500 stars). Promises "sub-100ms agent-to-agent latency" by keeping model weights, agent state, and communication buffers entirely in VRAM, using PyTorch's `torch.compile` and custom CUDA kernels for routing.

### 3. Integration with "Model Routers"
Several frameworks now explicitly support external "model router" services, which dynamically select the best LLM (by cost, latency, capability) for a given agent step.
- **CrewAI** recently added a `RouterLLM` wrapper class in its experimental module.
- **OpenRouter** and **Fireworks AI** are commonly cited providers for this pattern. This suggests a move towards treating LLMs as a heterogeneous, fungible compute resource within an agent graph.

### 4. Research Frontier: Long-Horizon Planning & Verification
Academic and open-source research (visible on arXiv) is probing the limits of current frameworks:
- **Problem:** Most frameworks (CrewAI, LangGraph) excel at short, deterministic task sequences but struggle with genuine long-horizon planning and recoverable execution.
- **Approach:** Some experimental repos integrate Monte Carlo Tree Search (MCTS) or learned "planner" models as a dedicated agent type to propose and evaluate multi-step plans before execution.

### 5. Fleet Context Comparison
The fleet's architecture, as described in `FLEET-CONTEXT.md`, aligns with and diverges from these trends:
- **Alignment:** The focus on specialized "ensigns" (LoRA adapters) and GPU-resident runtime (`cudaclaw`) matches the "local-first" trend.
- **Divergence:** The fleet's **Deadband Protocol** (P0/P1/P2) and **Mirror Play** concept are not widely observed in mainstream open-source frameworks. These represent a distinct approach to safety and iterative learning.

### Specific Factual Observation for April 2026
The repository `local-agents/local-agents` on GitHub gained approximately 300 stars in the last 14 days. Its README emphasizes a benchmark showing a 5-agent workflow completing in 1.2 seconds on a single RTX 4090, compared to 12.7 seconds using a comparable API-based framework (CrewAI with GPT-4-Turbo). This performance claim is a primary driver of its current visibility.
