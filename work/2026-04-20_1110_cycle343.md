## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community channels, framework documentation.

### 1. Dominant Frameworks & Shifts
The landscape continues to be defined by modular, graph-based architectures. **LangGraph** (LangChain) and **CrewAI** remain central for multi-agent orchestration. A noticeable trend is the integration of these frameworks with **local LLM runtimes** (Ollama, LM Studio) to reduce cloud dependency and cost. **AutoGen** (Microsoft) sees steady adoption in research and enterprise prototyping due to its conversational agent patterns.

### 2. Emerging Hotspots
*   **GPU-Native Runtimes:** Projects like **CudaClaw** (noted in fleet context) align with a broader movement toward agents that reside persistently in GPU memory to minimize latency. **vLLM** and **TGI** (Text Generation Inference) are increasingly used as backends for agent servers.
*   **Specialization:** Frameworks are specializing by domain. For example, **SmolAgents** focuses on small, fast, deterministic agents for repetitive tasks, while **OpenAI's GPTs** (and open-source equivalents) cater to conversational, tool-using agents.
*   **Self-Improving Loops:** There is growing experimentation with frameworks that incorporate **self-training loops**. The fleet's **plato-ensign** concept (exporting room experience to LoRA adapters) fits this pattern. Open-source projects are beginning to explore similar "experience capture" mechanisms.

### 3. GitHub Trending (Last 7 Days)
A scan of trending repositories in "AI" and "Python" categories shows:
*   **Practical Focus:** Repositories with clear, minimal examples for building **AI customer support** or **internal research assistants** receive high engagement.
*   **Local-First:** Projects that simplify running multi-agent systems entirely on local hardware (using Mixtral, Llama 3, or Qwen 2 models) are trending.
*   **Integration Tools:** Tools for connecting agents to specific data sources (Notion, Slack, GitHub) or for **evaluating** agent performance (agent benches) are active.

### 4. Specific, Factual Observation
A concrete pattern observed in multiple recent framework releases (e.g., **LangGraph v0.2+, CrewAI v0.28+) is the formalization of a "checkpoint" or "save/load" API for agent state. This allows long-running agent workflows to be paused, persisted, and resumed, which is particularly relevant for complex, multi-step tasks that may exceed context windows or runtime limits. This aligns with the fleet's concept of "tiles" as persistent units of experience.

### 5. Fleet Context Alignment
The fleet's architectural choices—**holodeck-rust** (MUD server for simulation), **constraint-theory-core** (structured output), and **flux-runtime** (self-assembling VM)—position it within the niche of **simulation-based training** and **geometric constraint solving**, which is less saturated than general-purpose chat orchestration. The **Deadband Protocol** (P0: negative space) offers a distinct formalized decision-making heuristic not commonly seen in mainstream frameworks.
