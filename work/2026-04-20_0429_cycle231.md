### 1. Primary Trend: Specialization & Composability
The dominant trend observed is a move away from monolithic "orchestrator" frameworks toward specialized, composable units. This mirrors the fleet's own architectural principle of rooms and ensigns.

*   **CrewAI & LangGraph:** These frameworks are increasingly used as high-level coordinators, but the focus has shifted to the "agents" or "nodes" they manage. The value is now in the quality and specificity of the individual components (e.g., a research agent with a dedicated toolset, a code-review agent with a secure sandbox).
*   **AutoGen:** Its strength remains in multi-agent conversation patterns, but recent community activity shows a preference for using it to define clear, constrained agent roles for specific tasks (debugging, documentation) rather than as a general-purpose problem solver.

### 2. Emerging Pattern: The "Runtime" Layer
A new layer of infrastructure is gaining attention, focused on the deterministic execution and state management of agentic workloads. This aligns with fleet technologies like `cudaclaw` and `flux-runtime`.

*   **Projects like `instructor`** (for structured LLM outputs) and `marimo` (for reactive notebooks) are being repurposed as core components for building reliable agent steps. The goal is to make non-deterministic LLM calls fit into deterministic pipelines.
*   **The "Agent as a Function"** pattern is emerging, where an agent with its prompt, tools, and memory is packaged as a callable, stateful unit that can be chained, tested, and versioned.

### 4. Negative Space (P0) & Safe Channels (P1)
*   **P0 (What NOT to do):** Avoid building yet another high-level "orchestrator" wrapper around raw LLM APIs. The space is saturated. Also, avoid frameworks that treat tool execution as an unprotected subprocess call; this is a known security and stability fault line.
*   **P1 (Safe Channels):** The safe channels for contribution and innovation are:
    1.  **Specialist Agent Training:** Creating highly capable, fine-tuned agents for narrow tasks (like the fleet's `ensigns`).
    2.  **Runtime Safety & Sandboxing:** Contributing to or building infrastructure for secure, observable, and resettable agent execution.
    3.  **Deterministic Orchestration:** Developing libraries that help compose stochastic agent steps into reliable, debuggable workflows.
