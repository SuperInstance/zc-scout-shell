### 1. Framework Landscape & Movement
The ecosystem remains dynamic, with established frameworks consolidating features and newer entrants focusing on niche capabilities. The dominant pattern is the shift from simple orchestration to **stateful, long-running agent systems** with built-in memory, tool use, and human-in-the-loop controls.

*   **CrewAI** continues to gain traction for its role-based orchestration and ease of defining sequential, hierarchical agent workflows. Recent commits (last 30 days) show increased focus on integrating with external vector databases and improving the `crew.kickoff()` execution model for more deterministic outputs.
*   **LangGraph** (LangChain) is being adopted for building complex, cyclic, stateful agent systems. Its `StateGraph` model, which explicitly manages a shared state object between nodes, is cited in several new research repos as a method for implementing planning and reflection loops. The `persistence` API for saving/loading graph state is a noted feature for long-running applications.
*   **AutoGen** (Microsoft) maintains a strong presence in academic and research-oriented multi-agent scenarios. Recent discussion threads indicate users are combining it with **TypeChat** for stronger schema validation of agent outputs.

### 2. Emerging Pattern: "GPU-Native" & Local-First Runtimes
A notable trend is the exploration of agents designed to reside entirely on a GPU or edge device, minimizing CPU-hosted orchestration overhead. This aligns with fleet technology **cudaclaw**.

*   Projects like **Tabby** (self-hosted GitHub Copilot) and **Continue.dev** emphasize local LLM inference. While not full agent frameworks, they signal a demand for low-latency, private tooling.
*   The **flux-runtime** concept (self-assembling bytecode VM) mentioned in fleet knowledge appears to be part of this broader research direction—exploring agents that operate closer to the metal than Python-based interpreter loops.

### 3. Hot & New (Last 90 Days)
*   **Smol Agents / smol-developer**: A minimalist trend advocating for small, focused, single-file agent scripts (often < 500 lines of Python) over large frameworks. The philosophy is "do one thing well" and compose via shell. This is sometimes seen as a reaction to framework complexity.
*   **Aider.chat**: Gaining visibility as a CLI agent that directly edits code in a local git repo. Its tight integration with git (making commits, viewing diffs) is a specific implementation of the "agent with a shell" concept relevant to the Zeroclaw role.
*   **OpenAI DevDay Tool Updates**: While not open-source, the new `file_search` and `structured_outputs` features in the OpenAI API are influencing OSS project roadmaps, with several libraries adding similar retrieval or Pydantic-native output parsers.
