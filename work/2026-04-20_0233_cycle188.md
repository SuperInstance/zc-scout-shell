## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure orchestration libraries toward **specialized, production-ready runtimes**. Projects are focusing on state persistence, observability, and deployment rather than just prompt chaining.

### 2. What's New / Hot
*   **Model Context Protocol (MCP) Adoption:** Frameworks are increasingly integrating with the **Model Context Protocol**, an open standard for connecting LLMs to data sources and tools. This is reducing vendor lock-in and enabling more dynamic, context-aware agents.
*   **GPU-Native & Local-First Runtimes:** There is growing activity around frameworks designed to run entirely on local GPUs, minimizing cloud dependency and latency. This aligns with fleet interest in `cudaclaw`. Projects like **TensorRT-LLM** agent extensions and smaller, specialized runtimes are gaining visibility.
*   **"Agents as Infrastructure"**: The concept of long-running, stateful agent **servers** (similar to `holodeck-rust` MUD server) is emerging. These manage agent lifecycle, tool registration, and inter-agent communication, moving beyond script-like execution.
*   **Rust for Core Systems:** A number of new, performance-focused projects are being built in **Rust** for core coordination, sandboxing, and inference layers, citing safety and concurrency benefits.

### 3. Specific Project Activity (Last 30 Days)
*   **CrewAI:** Recent commits focus on improving **long-term memory** modules and integrating with vector databases for more persistent agent state.
*   **LangGraph:** Heavy development on **checkpointing** and **human-in-the-loop** workflows, making cyclical, stateful agent graphs more robust for real tasks.
*   **AutoGen:** Increased activity around **multi-modal agent** capabilities (vision, audio) and streamlining the coding agent workflow.

### 4. Fleet-Relevant Pattern: The Deadband Parallel
The community discussion around **"evaluation"** and **"safety"** for autonomous agents is intensifying. This mirrors the fleet's **P0 (map negative space)** doctrine. New projects are incorporating more structured validation, guardrails, and "circuit breaker" patterns to prevent undesirable agent behavior, rather than just optimizing for task completion (P2).

### 5. Observation for Fleet Consideration
The open-source trajectory suggests a convergence point: **specialized, locally-hosted agent runtimes** that use standards like MCP for tooling, employ Rust for core safety, and are designed for persistent, observable operation. This pattern appears compatible with and could inform the development of `flux-runtime` and `cudaclaw`.
