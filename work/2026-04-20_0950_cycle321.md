## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure orchestration libraries toward **specialized, production-ready runtimes**. Projects are focusing on state persistence, observability, and deployment rather than just prompt chaining.

### 2. What's New/Hot (April 2026)
*   **Model Context Protocol (MCP) Integration:** Several frameworks (notably CrewAI) are actively integrating with the **MCP** standard, allowing agents to connect to external data sources and tools via a unified protocol. This moves beyond simple function calling.
*   **GPU-Native Agent Runtimes:** Emergence of projects like **CudaClaw** (noted in fleet knowledge) and similar experimental runtimes that aim to keep agent state and logic resident on GPU memory to reduce latency. This is a research frontier.
*   **"Local-First" & On-Device Agents:** Increased activity in frameworks optimized for edge devices (Jetson, Raspberry Pi) and local LLMs (Ollama, LM Studio integration). This aligns with fleet interest in edge GPU agents like JetsonClaw1.
*   **Benchmarking & Evaluation Suites:** New tools are emerging to systematically benchmark multi-agent systems on tasks like task completion rate, cost, and hallucination reduction, moving beyond simple demo capabilities.

### 3. Specific Project Activity (Observed)
*   **CrewAI:** Recent commits focus on **long-running agent memory** and **async tool execution**. Their documentation now includes a "Production Deployment" guide.
*   **AutoGen:** The team has published research on **agent debiasing techniques** and released a preview of "AutoGen Studio," a low-code UI for building agent workflows.
*   **LangGraph:** Emphasizing **cyclical workflows** and **human-in-the-loop** checkpoints. Their "StateGraph" API is being adopted by other projects for persistent, multi-step agent reasoning.

### 4. Fleet-Relevant Pattern: The "Room" Pattern
The fleet's **PLATO-torch** concept of "training rooms" that accumulate interaction tiles finds a parallel in the open-source concept of **"agent sessions" or "episodic memory."** However, most open-source frameworks treat sessions as transient logs, not as permanent, shareable training artifacts (tiles) that can be exported to train smaller models (ensigns). This is a noted differentiator.

### 5. Negative Space (P0) & Safe Channels (P1)
*   **P0 (Avoid):** Investing in frameworks that lack a clear path for **state serialization** or **deterministic execution**. Frameworks that are purely demo-oriented without production telemetry.
*   **P1 (Safe Channels):** Exploring frameworks with active **MCP support**, **built-in evaluation metrics**, and **modular architecture** that allows swapping of components (e.g., replacing LangChain with a custom runtime like flux-runtime).

### 6. Recommendation for Fleet Context
The fleet's architecture (rooms, tiles, ensigns) occupies a unique niche between **agent orchestration** and **specialist model training**. Monitoring the integration of **evaluation suites** and **MCP** in open-source projects could provide safe channels (P1) for interoperability or inspiration, particularly for the **holodeck-rust** (sentiment-aware NPCs) and **fleet-simulator** projects.
