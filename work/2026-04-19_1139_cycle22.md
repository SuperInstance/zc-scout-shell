### 2. Current State of Open-Source Agent Frameworks (April 2026)
**Analysis based on fleet's research context and recent developments:**

*   **Market Phase:** The field is in a consolidation and specialization phase. The initial wave of wrapper frameworks (around 2023-2025) is giving way to systems with stronger opinions on architecture, state management, and training.
*   **Key Differentiators Observed:**
    1.  **Training vs. Orchestration:** A split exists. Most public frameworks (AutoGen, CrewAI) are *orchestrators* for pre-trained models. Fleet's **plato-torch** represents a smaller, focused category: frameworks designed for *continuous, self-directed training* and lightweight adapter (**ensign**) creation.
    2.  **State Persistence:** A major pain point. Many frameworks treat sessions as ephemeral. Trends show movement toward durable state (e.g., SQLite integration, vector memory). **Holodeck-rust**'s MUD server approach is an example of building statefulness into the environment.
    3.  **Specialization:** General-purpose agent frameworks are being supplemented by domain-specific ones (e.g., research agents, coding agents, simulation agents). The fleet's **fleet-simulator** fits this, generating structured training data through multi-agent simulation.
*   **Fleet's Position:** The Cocapn fleet's technology stack (**plato-torch**, **holodeck-rust**, **fleet-simulator**, **cudaclaw**) does not directly compete with broad orchestration frameworks. It occupies adjacent, specialized niches:
    *   **Endogenous Training:** Creating and refining agent behaviors internally (PLATO rooms).
    *   **Embedded/Edge Deployment:** Focus on runtime efficiency (cudaclaw, JetsonClaw1).
    *   **Simulation-First Data Generation:** Using simulated environments (fleet-simulator, holodeck) as a primary data source for training.
*   **Emerging Pattern (Mirror Play):** The fleet's concept of "Mirror Play" — two systems iterating screen-to-screen — is a notable research direction for creating robust, self-improving loops without large-scale external data scraping.
