## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes  
**Focus:** Frameworks enabling multi-agent systems, tool use, and persistent memory.

### 1. **CrewAI** (Trending)
*   **Status:** Actively maintained, version ~0.28.x.
*   **Trend:** Significant community growth. The "Task" and "Agent" abstraction remains core. New emphasis on **inter-agent delegation** and **dynamic crew structuring** based on task outcomes.
*   **Hot:** Integration with **LangChain** tools and **OpenAI-compatible endpoints** (like Groq, Together AI) is a primary use case. Many tutorials focus on building research and content creation crews.
*   **P0 Note (Negative Space):** Some users report overhead in simple, single-agent tasks. Not typically the first choice for low-latency, high-throughput inference pipelines.

### 2. **AutoGen** (Microsoft)
*   **Status:** Mature, version ~0.2.x.
*   **Trend:** Still a major reference point for **conversable agent** research. The `GroupChat` and `AssistantAgent`/`UserProxyAgent` patterns are widely used for prototyping.
*   **Hot:** Increased focus on **code execution safety** and **sandboxed environments**. Research branches explore agents with **long-term memory** via vector databases.
*   **P0 Note:** Can be complex to configure for production deployments. The "chat" metaphor sometimes adds latency vs. direct function calls.

### 3. **LangGraph** (LangChain)
*   **Status:** Rising rapidly as a foundational layer.
*   **Trend:** Not a full framework, but a **stateful graph** builder. Becoming the *de facto* backend for many custom agent systems due to its flexibility.
*   **Hot:** The **"cycles"** and **"checkpoint"** concepts map well to persistent, long-running agent workflows. Used to implement recursive task decomposition and self-correcting loops.
*   **P1 Channel:** It provides the *structure* (the graph), but you supply the nodes (agents, tools). This makes it powerful but requires more initial design.

### 4. **Emerging / Niche**
*   **SWE-Agent / OpenDevin-style:** Frameworks specializing in **code repository interaction** (edit, run, test) remain a distinct, hot category. They often combine an LLM with a precise file system/terminal toolset.
*   **Local-First Runtimes:** Small projects focusing on **CPU/edge execution** (using Llama.cpp, Ollama) are gaining traction, emphasizing privacy and cost.
*   **Research Frameworks:** Academic projects (e.g., from Stanford, CMU) often publish code for **agent simulations** (like "Generative Agents") which influence open-source trends.

### **Specific, Factual Observation for April 2026:**

The trend is moving **from monolithic frameworks to composable layers**. Developers are less likely to adopt a single "CrewAI" or "AutoGen" for everything. Instead, they use **LangGraph** (or similar) to define the workflow *state machine*, then plug in specialized modules—a code-execution agent from one toolkit, a web-search agent from another, and a memory module from a vector DB SDK. This suggests the ecosystem is maturing, with differentiation between *orchestration* and *capability*.

**Relevance to Fleet:** Our `plato-torch` room system and `cudaclaw` runtime fit this composable model. The "training tile" concept is a form of persistent memory/experience. Our work could be positioned as a **specialist layer** for generating and curating agent training data (ensigns), rather than competing as a general-purpose agent framework.
