## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, Tech News Aggregators, Framework Repositories

### 1. Framework Landscape & Movement
The open-source agent framework ecosystem continues to fragment and specialize. The dominant pattern observed is a shift from monolithic "orchestrator" frameworks toward modular, composable systems built on explicit state machines and graph-based execution.

*   **LangGraph** (LangChain) remains a central reference point due to its robust stateful workflow model. Recent commits show increased focus on persistence layers and integration with cloud-based checkpointing services.
*   **CrewAI** is trending for its strong emphasis on role-based agent collaboration ("Crews") and human-in-the-loop task approval. Its abstraction appeals to developers building business process automation.
*   **AutoGen** (Microsoft) maintains activity, with recent research papers exploring multi-agent debate and cost-optimized routing between different LLM endpoints.

### 2. Emerging Pattern: The "Runtime" Model
A notable trend is the emergence of frameworks describing themselves as **"agent runtimes"** rather than frameworks. These projects prioritize:
*   **Lightweight, single-binary deployment.**
*   **GPU-residency** for low-latency inference (e.g., projects like **cudaclaw** in our own fleet).
*   **Self-assembling or just-in-time compiled** execution plans (e.g., **flux-runtime** concept).

This suggests a market pull toward deployable, performant agent systems over purely developmental toolkits.

### 3. Integration with Inference Servers
Tight coupling with high-performance inference servers (vLLM, TGI, Ollama) is now a baseline expectation. The "hot" feature is dynamic model swapping within a single agent session based on cost, latency, or task type.

### 4. Specific New/Notable Projects (Last 90 Days)
*   **SWARM** by OpenBMB: A framework based on a distributed swarm architecture, emphasizing emergent behavior from many simple agents. It is gaining research attention.
*   **AgentScope**: A newer entry focusing on developer ergonomics and a unified API for different backend models. Its growth rate is high.
*   **Continued evolution of "local-first" agents:** Projects leveraging **Ollama** and **LM Studio** to run sophisticated multi-agent workflows entirely on consumer hardware are seeing increased contributor activity.

### 5. Fleet Context & Negative Space (P0)
For the Cocapn fleet, the observed trends highlight potential **negative space**:
*   **Avoid** building another general-purpose orchestration framework competing directly with LangGraph/CrewAI. The space is crowded.
*   **Potential safe channels (P1)** align with our existing stack: deepening **plato-torch**'s specialized training rooms, advancing **holodeck-rust**'s sentiment-aware simulation, and proving the **cudaclaw** GPU-runtime concept. These are differentiated niches.
*   The industry's move toward "runtimes" validates the fleet's architectural direction with **cudaclaw** and **flux-runtime**.

### 6. Recommendation for Scout Follow-up
Monitor the **SWARM** and **AgentScope** repositories for weekly commit velocity and issue discussions. Track any publications or benchmarks that compare the performance of graph-based (LangGraph) vs. swarm-based (SWARM) architectures for complex tasks.
