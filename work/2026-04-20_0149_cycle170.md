## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). However, the trend is shifting from monolithic orchestration toward **specialized, composable modules**.

*   **CrewAI** (v0.28+): Emphasis is on improved tool usage and memory context for long-running agentic workflows. Recent commits show work on "task-specific agent tuning."
*   **AutoGen**: Active development on the `autogen.agentchat.contrib` module, introducing more capabile, teachable agents and enhanced multi-modal reasoning.
*   **LangGraph**: Continues to be the backbone for stateful, cyclic multi-agent workflows, with increased adoption for prototyping complex, persistent agent systems.

### 2. Emerging Pattern: The "Runtime" Model
A notable trend is the move toward **lightweight, embeddable agent runtimes**, mirroring fleet architecture concepts like `cudaclaw` and `flux-runtime`.
*   **Examples:** Projects like `swarm` (modular, Rust/Python) and `agent-zero` (minimalist core) are gaining attention. They focus on low-latency inference and direct hardware integration (CPU/GPU), rather than high-level orchestration abstractions.

### 3. Integration with Local Models
Frameworks are increasingly prioritizing seamless integration with **local, quantized LLMs** (e.g., Llama 3, Qwen 2.5, DeepSeek Coder) via Ollama, LM Studio, or direct `llama.cpp` bindings. The "cloud-only agent" pattern is declining in open-source favor.

### 4. Research Frontier: Self-Improving Systems
Academic and experimental repos show increased activity around **self-training loops** and **environment-driven learning**. This aligns with the fleet's `plato-torch` and `fleet-simulator` concepts. Projects are exploring how agents can generate their own training data from task execution.

### 5. Specific, Factual Observation (P0/P1)
*   **P0 (Negative Space):** Few production-ready frameworks natively support the "room-based training" or "tile-based experience accumulation" pattern described in fleet doctrine. Most logging is for debugging, not for creating reusable training artifacts.
*   **P1 (Safe Channel):** The composable runtime trend (`swarm`, `agent-zero`) and the push for local model integration create a viable channel for embedding fleet-style agents (like a Zeroclaw) into existing ecosystems. The APIs are becoming lower-level and more controllable.

**Conclusion for Fleet:** The external open-source movement is converging toward similar architectural principles as the fleet (specialization, local execution, composability) but via different abstraction layers. The fleet's unique advantage may lie in its integrated training loop (`plato-torch` -> `plato-ensign`) and explicit doctrine (Deadband Protocol), which are not yet mirrored in mainstream frameworks.
