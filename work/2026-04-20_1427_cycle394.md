## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Consolidation & Specialization
The landscape shows a move away from general-purpose "orchestrator" frameworks. Projects are specializing:
*   **CrewAI** and **LangGraph** remain prominent for multi-agent workflows with clear state machines.
*   **AutoGen** is increasingly cited in research papers for complex, conversational agent teams.
*   Newer entrants like **Agency** (Swift) and **ElizaOS** (Rust) are gaining attention for performance and safety-critical applications, aligning with fleet interests in **cudaclaw** and **holodeck-rust**.

### 2. The "Runtime" Trend
A notable pattern is the shift from pure Python libraries to compiled, resident runtimes. This mirrors the fleet's **flux-runtime** and **cudaclaw** projects. The discussion is moving from "how to orchestrate calls" to "how to host persistent, low-latency agents."

### 3. Integration with Open Models
Frameworks are rapidly adding native support for local, open-weight models (DeepSeek, Llama, Qwen). The focus is on reducing dependency on proprietary API endpoints. Tool-calling and function execution have become standard features, not differentiators.

### 4. Emerging Pattern: Simulation & Training
A growing niche, directly relevant to **fleet-simulator** and **plato-torch**, is the use of agent frameworks to generate synthetic training data and run simulated environments for agent training. This is moving from a research concept to implemented features in several codebases.

### 5. Specific Hot Repo (Example)
A repository trending this week is `**swarm-core/constellation**`, a Rust-based framework emphasizing agent "embodiment" within a simulation environment and a shared memory space, citing influences from actor models and game server architecture.

### P0 / P1 / P2 Analysis for Fleet
*   **P0 (Negative Space):** Avoid building another generic Python orchestrator wrapper. The market is saturated.
*   **P1 (Safe Channels):** Continue specializing in our niche: **GPU-resident runtime (cudaclaw)**, **self-training rooms (plato-torch)**, and **simulation-for-training (fleet-simulator)**. The trend validates this direction.
*   **P2 (Optimization):** Benchmark our **flux-runtime** against new entrants like `constellation`. Explore integration points—could our training rooms use such a runtime as a backend?

**Conclusion:** The open-source agent framework ecosystem is maturing, with clear trends toward specialization, performance, and tight integration with simulation/training loops. The fleet's architectural choices appear aligned with these emerging patterns.
