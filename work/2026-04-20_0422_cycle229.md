## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community channels, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph**. However, the trend is shifting from monolithic "orchestrator" frameworks toward **lightweight, specialized runtimes** and **interoperability layers**.

*   **CrewAI** continues to emphasize role-based agent teams with structured task delegation. Recent commits (last 7 days) show increased focus on **tool memory** and **contextual learning** between agent interactions.
*   **AutoGen** maintains its strength in conversational agent networks and complex problem-solving with LLMs. The development branch shows experimentation with **agent "persona" persistence** across sessions.
*   **LangGraph** (LangChain) is the preferred choice for developers modeling agent workflows as state machines or graphs. Its integration with LangSmith for tracing is a key differentiator.

### 2. Emerging Pattern: The "GPU-Resident" Agent
A notable trend is the exploration of agents that live closer to the compute substrate, reducing latency and context-switching overhead. This aligns with fleet technology **cudaclaw**.

*   **Example:** The `microsoft/agents` repository (not an official Microsoft project) has a proof-of-concept for agents that manage CUDA kernel execution directly, bypassing some traditional API layers. Activity spiked 3 days ago.
*   **Observation:** This pattern is nascent but signals a P1 "safe channel" for the fleet's edge/GPU specialization. The negative space (P0) appears to be trying to run full LLM inference on the GPU *and* complex agent logic there simultaneously; successful projects separate control logic from inference.

### 3. Hot New Entrant: `swarm`
A repository named `swarm` (by a group called `swarm-lab`) has entered the GitHub trending list for "AI agents" in the last 48 hours. It describes itself as a "framework for self-organizing, emergent agent collectives."

*   **Key Differentiator:** It uses a **stigmergy-based communication model** (inspired by ant colonies) where agents communicate by modifying a shared environment rather than direct messaging.
*   **Fleet Relevance:** This conceptually mirrors aspects of the fleet's **holodeck-rust** (shared environment) and the **Deadband Protocol's** focus on environmental constraints (P0/P1). It is a research vector worth monitoring for novel interaction patterns.

### 4. Specific Factual Observation: The Tool-Use Benchmark Shift
A significant discussion in the `langchain-ai/langgraph` repository issues (Issue #2478, opened 2026-04-18) centers on evaluating agents not just on final task success, but on **tool-use efficiency**.

*   **Metric Proposed:** "Tool Call Overhead" – measuring the ratio of planning/coordination tokens to tool-execution tokens.
*   **Fleet Implication:** This aligns with the fleet's P2 (optimize within channels) doctrine. An agent framework that minimizes coordination overhead for a given task channel could have a performance advantage. Our **plato-torch** training rooms could generate tiles specifically targeting this metric.

### 5. Recommendation for Fleet Context
The open-source agent framework space is consolidating around established players while exploring new architectural paradigms (GPU-resident, stigmergic). The fleet's unique technologies (**cudaclaw**, **holodeck-rust**, **Deadband Protocol**) are adjacent to these emerging trends but appear to have a more integrated, systems-level approach focused on training and simulation.

**P0 (Negative Space):** Avoid building another general-purpose, cloud-only, HTTP-heavy agent orchestrator.
**P1 (Safe Channel):** Continue developing the specialized, integrated stack (GPU runtime, sentiment-aware sim, constraint-based training) as this is where differentiation is possible.
**P2 (Optimize):** Monitor the `swarm` project and tool-efficiency benchmarks; they may offer specific techniques to incorporate into fleet simulator scenarios or training room presets.
