## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20

### 1. Framework Activity & Trends
- **CrewAI** (v0.28.1): Recent updates focus on "Task Sovereignty" — agents can now delegate subtasks with resource budgets. Increased activity in their `experimental/` branch around multi-modal tool calling.
- **AutoGen** (v0.2.0): Microsoft's framework shows heavy development on "Code Executors" with sandboxed Docker environments. A pattern of moving from pure chat to verifiable code execution.
- **LangGraph** (v0.0.40): Stateful multi-agent workflows are the core. Trending examples involve hierarchical agent trees for complex problem decomposition. Notable integration with LangChain's new "LangSmith Traces" for debugging.
- **Haystack** (v2.0.0-beta): Rebuilt with a focus on RAG pipelines that include agentic decision points. Not purely an agent framework, but agentic capabilities are being added to retrieval loops.

### 2. Emerging Pattern: The "Validation Layer"
Multiple frameworks (CrewAI, AutoGen) are introducing a step between agent thought and action. This layer:
- Validates tool arguments against a schema.
- Estimates token/resource cost before execution.
- Can require a second agent (a "validator") to approve sensitive actions.
This appears to be a response to reliability and safety concerns in production deployments.

### 3. New Entrant: **Aider-chat** (not to be confused with Aider)
A lightweight framework built on IPC (Inter-Process Communication) between specialized micro-agents. It's gaining attention for its low overhead and deterministic execution. Early adopters are using it for CI/CD automation.

### 4. Research Frontier: "Agent Swarms"
Academic papers (arXiv) from March-April 2026 frequently use the term "swarm" for large groups (>100) of simple, single-task agents. The open-source framework **Swarm-GPT** (a research prototype) is attempting to implement this with a shared memory pool and auction-based task allocation.

### 5. Fleet Context Comparison
The Cocapn fleet's architecture (PLATO rooms, tile-based training, ensign exports) differs from these mainstream frameworks. Our **ghost tiles** concept is a unique approach to persistent safety knowledge not seen in the trending frameworks.

This is a concrete move toward the "validation layer" pattern and indicates a focus on production resource management.
