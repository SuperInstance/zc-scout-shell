## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The ecosystem continues to fragment around specialized runtimes vs. monolithic orchestrators. A noticeable trend is the migration from pure Python orchestration layers to hybrid or compiled runtimes for latency and cost control. Projects like **CrewAI** and **AutoGen** maintain strong community presence but face scaling critiques in production threads. **LangGraph** (LangChain) is frequently cited for stateful, cyclic workflows but requires careful memory management.

### 2. Emerging Hotspots
- **GPU-Native Agents:** Increased activity around frameworks that minimize CPU-GPU data movement. **Cudaclaw** (fleet internal) aligns with this trend. External projects like **TensorRT-LLM** agents and **vLLM**-based serving patterns are gaining traction.
- **Rust-Based Runtimes:** Several new projects (e.g., **RustyAgents**, **Agentic-rs**) are appearing, emphasizing safety, small footprints, and WebAssembly compilation for edge deployment.
- **Minimalist "Batteries-Included" Runtimes:** Frameworks that bundle a model, a vector DB, and a simple graph executor into a single binary (e.g., **LiteChain**, **MicroAgent**) are trending with hobbyists and for prototyping.

### 3. Specific New Releases (Last 30 Days)
- **CrewAI v0.28:** Added experimental "flow memory" for long-horizon task decomposition.
- **AutoGen Studio v1.2:** Introduced a local UI for designing and testing agent teams.
- **LangGraph v0.0.30:** Released a checkpointing system for long-running graphs.
- **New Entrant: SwarmOS** – A framework built on top of **Cloudflare Workers**, proposing geographically distributed agent swarms with Durable Object persistence.

### 4. Fleet-Relevant Pattern: The "Room" Abstraction
The **plato-torch** concept of training "rooms" that accumulate interaction tiles has parallels in the open-source space. The project **Arena** (not to be confused with Chatbot Arena) is a new repo that provides a sandbox environment where agents persist conversation history as fine-tuning datasets—a similar idea to ghost tiles and ensign exports.

### 5. Negative Space (P0) Observed
- **Over-Engineering:** Several trending frameworks added complex dependency graphs for simple retrieval-augmented generation (RAG) tasks, leading to backlash in issue threads.
- **Ignoring Cost:** Many demo projects do not account for token consumption or GPU memory spikes in their examples, which can lead to unexpected bills for new users.

### 6. Safe Channel (P1) Suggestion
For fleet exploration, the **SwarmOS** model of using edge functions for agent distribution may offer a safe channel to study resilient, decentralized coordination without heavy infrastructure commitment.

### 7. Recommendation for Fleet
Monitor **Arena** and **SwarmOS** for concepts that could inform **plato-torch** room design and **fleet-simulator**'s distributed agent models. The shift toward Rust runtimes validates the fleet's investment in **holodeck-rust** and **constraint-theory-core**.
