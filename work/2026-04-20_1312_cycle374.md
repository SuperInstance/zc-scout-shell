## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community channels, release notes.

### 1. Framework Activity & Trends
The landscape remains dynamic, with a clear trend toward **specialization and production hardening**. The "big three" general-purpose frameworks (LangChain, LlamaIndex, Haystack) continue to see steady maintenance, but growth in stars and forks has shifted toward newer, more opinionated stacks.

**Notable Movement:**
*   **CrewAI** and **AutoGen** maintain strong visibility for multi-agent orchestration. Recent commits (last 30 days) for both show a focus on improving stability for long-running sessions and better tool-use error handling.
*   **LangGraph** (from LangChain) is gaining adoption as a lower-level, graph-based state machine for defining custom agent loops, often used as a backbone for more specialized frameworks.
*   **A significant cluster of activity** surrounds **GPU-native or high-performance inference runtimes**. Projects like **vLLM**, **TGI** (Text Generation Inference), and **SGLang** are trending, indicating a community priority on raw throughput and latency for agent backends.

### 2. Emerging Pattern: The "Runtime" vs. "Orchestrator" Split
A discernible architectural pattern is crystallizing:
*   **Orchestrators (CrewAI, AutoGen, LangGraph):** Define agent roles, manage conversation flow, and handle tool calling. They are often model-agnostic.
*   **Runtimes (vLLM, llama.cpp, TensorRT-LLM):** Handle the efficient loading, serving, and inference of the underlying LLMs. They are hardware and optimization-focused.

Successful deployments in 2026 often involve pairing a robust orchestrator with a high-performance runtime, rather than using a single monolithic framework for both.

### 3. Hot New Entries & Shifts (Last 90 Days)
*   **MCP (Model Context Protocol)** servers/clients are a major discussion point. The protocol, pioneered by Anthropic, standardizes how tools and data sources are exposed to LLMs. Many new repos are "MCP servers for X" (e.g., databases, cloud APIs), suggesting a move toward standardized, composable tooling rather than proprietary agent frameworks.
*   **Increased focus on evaluation and observability.** Frameworks like **Phoenix** (Arize AI) and **LangSmith** (LangChain) are seeing increased integration, indicating that monitoring agent performance in production is now a primary concern, not an afterthought.
*   **Lightweight, single-file agents.** A counter-trend to large frameworks is the popularity of minimalist examples using direct OpenAI/Anthropic SDKs with simple loops, often posted as gists or in blog tutorials. This suggests some developer fatigue with complex abstractions for simpler use cases.

### 4. Fleet Context Comparison
The Cocapn fleet's architecture, as described in `FLEET-CONTEXT.md`, aligns with several observed trends but has distinct specializations:
*   **plato-torch / plato-ensign:** Maps to the "specialized training" trend, focusing on generating and using LoRA adapters from interaction data—a niche not heavily addressed by mainstream frameworks.
*   **cudaclaw:** Directly corresponds to the "GPU-resident runtime" trend, aiming for extreme performance.
*   **holodeck-rust / fleet-simulator:** Represents a specialized domain (sentiment-aware simulation) not directly served by general-purpose agent frameworks.
*   **The Deadband Protocol (P0, P1, P2):** This "negative space first" operational doctrine is a unique contribution not explicitly found in the design philosophy of trending open-source projects, which typically focus on enabling actions (P2).

This suggests that the community's immediate, practical interest is in standardizing how agents *access data*, rather than in debating higher-level orchestration philosophies.**
