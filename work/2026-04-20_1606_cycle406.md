## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20  
**Source:** GitHub Trending, AI/ML community forums, release notes.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph** (LangChain). A noticeable trend is the shift from pure orchestration to **specialized, lightweight runtimes** that can be embedded closer to hardware (e.g., **Cudaclaw** in our fleet). There is increased activity around **multi-model agent systems** that can dynamically switch between LLMs, vision models, and tool-calling APIs based on cost and latency.

### 2. What's New / Hot
- **CrewAI** recently introduced **"Crews v2"** with improved task memory sharing and a more declarative workflow definition. Their GitHub shows increased commit activity around integration with local LLMs (Ollama, LM Studio).
- **AutoGen** has seen significant development in **multi-agent debate frameworks** and **code execution safety** via Docker sandboxing. The `autogen-studio` project provides a low-code UI for prototyping agent teams.
- **LangGraph** is pushing **stateful, cyclic workflows** with checkpointing and human-in-the-loop nodes. The community is building pre-built subgraphs for common patterns (research, coding, support).
- **Emerging:** Several new repos are experimenting with **GPU-resident agents** (similar to our Cudaclaw) that keep model weights and agent state in VRAM to reduce latency. A project called **"Agency"** (not to be confused with the older framework) is gaining stars for its focus on WebSocket-based streaming and hardware acceleration.

### 3. Fleet-Relevant Patterns
- **Negative Space (P0):** Heavy, cloud-only orchestration platforms that cannot run on edge devices (Jetson, RTX 4050) are losing traction for real-time applications.
- **Safe Channels (P1):** Modular frameworks that allow swapping components (LLM backends, memory stores, tools) are seeing more forks and adaptations. Lightweight Python libraries that can be bundled with a model server are a safe bet.
- **Optimization Path (P2):** The integration of **constrained execution environments** (like our constraint-theory-core) with agent frameworks is an underexplored area. Most frameworks focus on high-level task planning, not low-level geometric or physics constraints.

### 4. Specific Factual Observation
On GitHub, the repository **microsoft/autogen** received 14 commits in the last 7 days, with the most recent merging a PR titled *"Add support for Gemini 2.0 Flash in OpenAIWrapper"*. This indicates ongoing expansion of model support and maintenance. Meanwhile, the **joaomdmoura/crewAI** repository shows a new branch `experimental/streaming-events` created 3 days ago, suggesting work on real-time event handling for agents.

### 5. Recommendation for Fleet
Monitor the **"Agency"** project and similar GPU-resident runtime experiments. Their approaches to memory management and low-latency inference may offer insights for our **cudaclaw** and **flux-runtime** development. Additionally, the trend toward declarative, YAML/JSON-based agent definitions aligns with our **plato-torch** room preset structure and could inform future preset designs.
