## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20
**Source:** GitHub Trending, AI/ML community forums, project releases.

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen** (Microsoft), and **LangGraph** (LangChain). A noticeable trend is the shift from pure orchestration to **specialized, lightweight runtimes** that can be embedded closer to hardware or within existing applications.

*   **CrewAI** continues to emphasize a role-based, hierarchical agent model, which is often cited for structured enterprise workflows.
*   **AutoGen** maintains its strength in conversational multi-agent systems and researcher-focused tooling.
*   **LangGraph** is gaining adoption as the de facto standard for defining complex, cyclic agent workflows as state machines, particularly for developers already in the LangChain ecosystem.

### 2. Emerging Pattern: The "GPU-Resident" or "Edge-Native" Agent
A significant area of development, aligning with fleet technologies like **cudaclaw**, is the push for agents that minimize latency by residing primarily on the GPU or edge device. Projects are emerging that compile agent graphs to optimized kernels, aiming to keep context and intermediate states in VRAM rather than shuttling data back to host CPU/RAM. This is a response to the bottleneck of moving large tensors and context windows between memory spaces.

### 3. Specific New Entries & Notable Activity (April 2026)
*   **Aeria**: A new framework gaining attention focuses on **agent memory and self-reflection**. Its documentation highlights "persistent, queryable memory streams" for agents, which allows for longer-horizon task execution and learning from past interactions. This aligns conceptually with the fleet's "ghost tiles" and accumulated room experience.
*   **Model Context Protocol (MCP)**: While not a framework itself, the growing adoption of **MCP** (spearheaded by Anthropic and others) is changing the tooling landscape. It provides a standardized way for agents to connect to external data sources and tools. Frameworks are rapidly adding MCP server/client support, moving towards a more composable and secure tool-use paradigm than previous ad-hoc approaches.
*   **Rust-Based Runtimes**: There is increased activity in building core agent scheduling and communication layers in Rust (e.g., **RustyAgents**). The focus is on safety, performance, and small footprints suitable for deployment in constrained environments, resonating with projects like **holodeck-rust** and **constraint-theory-core** in the fleet.

### 4. Fleet Context & P0/P1 Analysis
*   **P0 (Negative Space)**: Avoid building yet another high-level Python orchestration wrapper with heavy dependencies. The market in that layer is saturated. Also, avoid frameworks that cannot separate planning from execution, as they struggle with real-world tool use.
*   **P1 (Safe Channels)**: The safe channels for innovation appear to be:
    1.  **Low-Level Runtime Efficiency** (GPU/Edge residency, Rust cores).
    2.  **Advanced Memory & State Management** (beyond simple caches).
    3.  **Standardized Tool/Data Integration** (via protocols like MCP).
    4.  **Simulation & Training Environments** for multi-agent systems (adjacent to **fleet-simulator**).
*   **P2 (Optimization)**: Within these channels, optimization opportunities exist in benchmarking, creating specialized adapters for hardware (like Jetson), and developing visual debugging tools for complex agent state graphs.

**Conclusion:** The open-source agent framework ecosystem is maturing beyond basic orchestration. The current "hot" areas are **performance-centric runtimes**, **standardized tooling protocols**, and **sophisticated memory architectures**. The fleet's investments in **cudaclaw** (GPU runtime), **holodeck-rust** (simulation), and the **PLATO room** training paradigm are positioned within or adjacent to these active channels of development.
