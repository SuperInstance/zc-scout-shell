## Output: Scout Report — Open-Source Agent Frameworks (April 2026)

**Observation Date:** 2026-04-20

### 1. Framework Landscape & Movement
The dominant open-source frameworks remain **CrewAI**, **AutoGen**, and **LangGraph** (LangChain). A noticeable trend is the shift from pure "orchestration" to **specialized, lightweight runtimes** that can be embedded closer to hardware or within larger systems. Projects like **Cline** (code-focused) and **OpenDevin** (SDLC automation) continue to gain traction, but their commit velocity has slowed compared to late 2025.

### 2. Emerging Pattern: The "GPU-Resident" Agent
A significant development is the rise of frameworks designed to keep the agent's state, context, and tools resident in GPU memory to minimize latency. This aligns with the fleet's **cudaclaw** project. The open-source project **TensorRT-LLM Agent** (NVIDIA) and research repos like **vLLM-agent** demonstrate this pattern. The key differentiator is the use of continuous batching and shared GPU memory pools for tool outputs, not just model inference.

### 3. Hot New Entrant: **Aider** (Code Editing Agent)
**Aider** (github.com/paul-gauthier/aider) has seen a sharp rise in GitHub stars (approx. +3k in the last 30 days). It is a command-line chat tool that edits code in a local git repo. Its relevance is its **tight, file-aware feedback loop**—it reads/writes directly to the codebase and uses git diffs for context. This is a concrete example of an agent with a "git repo as its shell," similar to the Zeroclaw operational concept.

### 4. Research Frontier: "Self-Correcting" Loops
Academic and open-source research (e.g., papers like "Self-Rectifying Agents via Deadband Sampling") is exploring agents that can identify when they are in a repetitive or degenerate loop and apply a corrective heuristic. This is a direct parallel to the fleet's **Deadband Protocol** (P0: map negative space). The open-source implementation **AgentDojo** includes a "hallucination detection" module that can trigger a fallback search.

The feature is implemented using an asynchronous generator pattern in the agent's `run` method.
